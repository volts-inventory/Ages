## Q102 — Recognition templates for Q90-Q101 physics
**Status:** Decided
**Context:** Q90-Q101 added a tonne of new physics — radiative
balance, atmospheric advection, hydrologic cycle, lunar tides,
magnetic field as a vector, latent-heat coupling, real
trigonometry. But almost none of it was *observable* to civs
through the recognition layer. The pre-Q102 templates were
authored before any of these laws existed and keyed on
charge / temperature / water_depth alone. New physics was
silent backdrop with no observation surface.

**Decision:** Add four new recognition templates that exploit
the Q90-Q101 fields, plus a new `Field::WindMagnitude` selector
so templates can key on wind speed:

- **id 36 — `tropical_moist`**: high vapour density (Q92) in
  the `Hot` climate band (Q90/Q97). Recognisable via
  chemical-taste (humid air), visual light (haze), and
  tactile channels.
- **id 37 — `dry_zone`**: low water_depth + low vapour density
  in the `ProductiveBand` (mid-latitude). Captures rain
  shadows, continental interiors, trade-wind deserts. Tactile,
  visual, chemical-taste.
- **id 38 — `storm_cell`**: high vapour density in `Cold` band
  cells (where Q92 condensation actively drops liquid).
  Recognisable via electric field (cloud charge), acoustic
  (thunder), and visual light (lightning).
- **id 39 — `windy_strait`**: `WindMagnitude > 0.5`. Captures
  jet streams, mountain passes, pressure-gradient gales.
  Tactile + acoustic-air.

`Field::WindMagnitude` is computed on demand as
`sqrt(v_q² + v_r²)` — same shape as Q95's `MagneticMagnitude`
but without a per-cell cache (only one template currently
reads it, so the recompute cost is bounded).

**Why this respects the project goals.**
- *Closes the observation gap:* Q90-Q101 physics is now
  recognisable. Civs that perceive any of `ChemicalTaste`,
  `Tactile`, `ElectricField`, `AcousticAir`, `VisualLight`
  channels will accumulate samples on the new templates and
  their hypothesisers will fit forms against them.
- *Each template uses multiple Q-decision outputs:*
  `tropical_moist` reads Q92 vapour + Q90/Q97 climate band;
  `storm_cell` similar; `windy_strait` reads Q91 wind. Each
  template's firing demands the underlying physics to be
  working, so accidental regressions in Q90-Q101 will surface
  as recognition firings disappearing.
- *Form-vocabulary coverage:* the four templates contribute
  `Threshold`, `Polynomial`, `DistanceDecay`,
  `ExponentialChange`, `Periodic` tags. No new tags needed —
  the existing form vocabulary already covers the shapes.
- *Determinism:* templates use the existing `Signature`
  combinators (`All`, `Above`, `Below`, `InClimateBand`); no
  new signature variants. Iteration order unchanged.

**Numerical care.**
- `WindMagnitude`'s `sqrt(v_q² + v_r²)` adds a per-evaluation
  sqrt for the one template that uses it (`windy_strait`).
  Under Q91's friction-damped velocity (~0.5 typical, capped
  by friction), `v_q² + v_r²` ≤ 0.5 — well inside Q32.32.
- Magnitude threshold `0.5` for `windy_strait` is calibrated
  against the post-friction steady-state velocity Q91
  produces; templates fire when the velocity field is in the
  upper quartile of typical magnitudes.
- All four templates use existing climate-band selectors so
  per-planet calibration applies automatically (Q74).

**Lives in:** `sim/recognition/src/lib.rs` (new
`Field::WindMagnitude` variant + `Field::read` arm + 4 new
template entries with ids 36–39).

**Follow-ups (not blocking).**
- *Direction-aware signatures.* `windy_strait` only sees
  magnitude. Templates like "downwind shadow" or "leeward dry"
  need a separate signature variant that reads
  `(v_q, v_r)`. Lands when civs build navigation tools that
  need direction.
- *Tidal-period recognition.* Real intertidal-zone templates
  fire on water_depth oscillating at the tidal period. Needs
  cross-tick signature support (currently signatures are
  per-tick).
- *Latent-heat-zone template.* Q100 latent-heat coupling cools
  source cells and warms receivers; a template that fires on
  cells that are *anomalously cool given their nominal
  insolation* would catch evaporative-cooling fingerprints
  without needing humidity tracking.
- *Magnetic-pole / aurora template refinement.* Q98 charge
  drift toward high-|B| cells will eventually populate auroral
  zones; a template that fires on `MagneticMagnitude > T1 AND
  Charge > T2` would distinguish auroral cells from generic
  high-charge weather. (Existing `aurora_polar` id 35 does
  some of this but uses charge alone; a Q102 follow-up
  refines it.)
