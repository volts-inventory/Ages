**Context:** Q55 ships M3's sensorium-extending tools with a

shouldn't be inventing instruments.

1. **Observation pressure** â€” should a tool's unlock be gated on

2. **Workforce capacity** â€” should unlock additionally require a

**Decision:** Hybrid observation-pressure + literacy-floor gate.
The wall-clock `time_gate_open` from Q55 retires. Each tool
declares:

- `observation_threshold: u64` â€” accumulated firings of templates
  the tool's `prereq_channels` natively sense (or universal
  templates for tools without channel prereqs). Earth-like and
  sub-surface-ocean civs accrue this differently because their
  perceivable templates differ.
- `literacy_floor: Real` â€” the civ's `literacy_score()` (Q58)
  must clear this to support specialised instrument-makers.

Plus the existing Q55 prereq stack: manipulation prereq
(`ToolExtension`), native-channel prereq, planet-feature prereq
(magnetosphere / EM medium).

The tier ordering preserves Q55's intended progression (T2 â†’
T3 â†’ T4) but makes it earned, not clock-driven.

**Why hybrid not pure observation:** isolated genius outpacing
the civ would let a primitive society invent thermal sensors
just because lightning struck a lot. The literacy gate enforces
"the civ has the durable knowledge base to make and reproduce
the instrument."

**Why hybrid not pure literacy:** a civ that's never observed
lightning shouldn't invent a field sensor purely from
literature. Observation pressure ties tools to physical
opportunity.

**Wiring:** `Civ::observed_count_for(tool)` returns accumulated
firings on `prereq_channels`-matching templates (or the union
of all templates' firings for prereq-less tools). `tech::is_unlocked`
takes `(civ, species_channels, species_manipulations,
has_magnetosphere, has_em_medium)` and returns `true` when all
gates pass. sim/core's CapabilityEvaluation phase no longer
calls `time_gate_open`.
