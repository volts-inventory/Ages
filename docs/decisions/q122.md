## Q122 — Phone-fit viewport defaults
**Status:** Decided
**Context:** After Q114-Q121 the live viewport carried four
planet-card lines + caption + grid + log tail, with each addition
optionally enabled by CLI flags. Default invocation (`ages
--seed N --years M --cli viewport`) still produced a 12×8 grid in
*standard* render mode (2-char-per-cell + hex-row offset, 28-col
body), `· `-bullet-separated planet-card lines that on some seeds
exceeded 32 cols, and a `[year     N] msg` log prefix that was
14 chars before the message ever started. On a portrait phone
terminal (~30 cols) the title and log wrapped, the grid wrapped,
and the user had to opt into a half-dozen flags to make it fit.

**Decision:** Three coordinated changes that ship together because
they only make sense as a set — the new defaults rely on each
other to fit.

1. **Default to compact + 24×16.** `ViewportConfig::compact = true`
   and the CLI now defaults `grid_width = 24`, `grid_height = 16`,
   `viewport_compact = true`. The `--viewport-compact` flag is kept
   as an explicit-on for back-compat, and a new
   `--no-viewport-compact` opts back into the standard layout for
   users who prefer the hex offset on wider terminals. Compact body
   width: 3 (row-prefix) + 24 (1-char cells) = 27 cols. Fits ~30-col
   phone terminals with margin.

2. **Dense 2-row column header (compact mode).** Q117's sparse
   "marker every 5 cols" header was readable but coarse — a viewer
   tracking a civ at column 17 had to count from the q=15 marker.
   Q122 replaces it with a 2-row dense header: top row = tens
   digit (blank for `q < 10`), bottom row = ones digit. Every
   column gets a label, every digit sits directly above its cell.
   ```
                11111111112222
      012345678901234567890123
    0 ........................
   ```
   Total header rows: 2 (same as Q117). Visual density: 5× higher.

3. **Tighten the planet card + log lines.** The card's data lines
   (substrate/atm/moons; mag/temp/day/tilt/year; species/cog/mod/
   manip) drop the ` · ` bullet for a single space — the bullet
   was 3 chars per gap, and with 4 gaps per line that was 12 chars
   of ornament. The log line drops `[year {n:>5}]` (14 chars
   including padding) for `y{n}` (≤ 5 chars). The `recent events:`
   divider becomes `-- log --`. Title line 1 (`{name} · {badge}`)
   keeps the bullet because it's a title, not a data row.

Sample card width before / after (Lyra-a, seed 1):
```
before:                                  after:
Lyra-a · boiling-off              (20)   Lyra-a · boiling-off              (20)
aqueous · reducing · 2 moons      (28)   aqueous reducing 2 moons          (24)
strong mag · 241F · 129h · 17° · 11mo (37)   strong mag 241F 129h 17° 11mo (29)
Cyranites · med-cog · touch/tent  (32)   Cyranites med-cog touch/tent      (28)
```
Worst-line drops from 37 → 29 cols. Log line drops from
`[year     0] civ 1 founded (10 cells)` (37) →
`y0 civ 1 founded (10 cells)` (27).

**Why this respects the project goals.**
- *Phone-fit by default:* the user's primary surface (live ASCII
  viewport on a portrait terminal) now works without any flags.
  Q114-Q121 added information; Q122 makes the default presentation
  of that information actually viewable.
- *4× spatial resolution at the same on-screen footprint:*
  pre-Q122 the default 12×8 grid in standard mode was 28 cols
  body; Q122's 24×16 grid in compact mode is 27 cols body. The
  user gets 4× the cells in the same width because each cell is
  1 char instead of 2, and the height grew at no width cost.
- *Determinism unchanged:* sim state and event stream are
  byte-identical. Q122 is purely a viewport-default change plus
  a render-helper upgrade.
- *Opt-out preserved:* `--no-viewport-compact`, `--grid-width`,
  `--grid-height` all still work — power users who pipe to a
  wide terminal or custom dashboard can pick their own layout.

**Numerical care.**
- Grid 24×16 = 384 cells. Per-tick recognition cost stayed within
  budget (no test-suite slowdown observed).
- Tens-digit header for `w >= 10` requires the
  `std::char::from_digit` fallback to never panic on
  `(tens % 10)` — bounded by construction since `w` fits in
  `u32` and we've already cast to `usize`.

**Lives in:** `sim/report/src/viewport.rs`
(`ViewportConfig::default()` flips compact, planet-card format
strings, log-line format string, divider string),
`sim/report/src/frame.rs` (compact column-header replacement),
`ages/src/main.rs` (CLI defaults + new `--no-viewport-compact`
flag + help-text update).

**Follow-ups (not blocking).**
- *Pressure-aware host-species badge.* Still open from Q121. Not
  a Q122 concern.
- *Standard-mode header alignment.* Q122 only fixes the compact
  header. Standard mode (opt-in via `--no-viewport-compact`) keeps
  its existing single-row dense header where the ones digit sits
  one char right of the cell. Low priority since standard mode is
  no longer the default.
- *Per-substrate name pools.* Still open from Q120.
