## Q36 — Determinism strategy for arithmetic
**Status:** Decided
**Context:** Both the physics engine (Q40) and the hypothesis-fitting
pipeline (Q32) need real arithmetic. Naive `f64` breaks byte-for-byte
determinism across platforms (denormals, transcendental rounding,
parallel reductions, FMA, SIMD reordering). Five candidates audited:
(A) fixed-point library, (B) rational arithmetic, (C) `f64` with
strict ordering rules, (D) software-emulated IEEE 754 (soft-float),
(E) hybrid fixed-point + f64-strict.
**Decision:** **A — fixed-point library.** Standard pattern for
deterministic-physics engines (lockstep RTS, multiplayer sims,
fighting games). Bit-exact by construction; fast on native integer
hardware; risk is local (overflow / precision) rather than
intermittent cross-platform divergence; encapsulates cleanly behind
a `Real` newtype API.
B (rational) too slow at our op count. C (f64-strict) brittle —
compiler updates can break determinism years later; not worth the
maintenance burden. D (soft-float) too slow. E (hybrid) doubles the
surface area without sufficient gain.

**Concrete spec for M0:**
- `sim/arith` crate exposing a `Real` newtype wrapping a Q-format
  integer.
- Q-format: Q32.32 default (`i64` underlying, ±~2.1×10⁹ range,
  ~2.3×10⁻¹⁰ precision). Tunable per-module if a region needs
  different precision.
- Standard ops (`+`, `-`, `*`, `/`, comparison) deterministic by
  construction.
- Transcendentals (`sin`, `cos`, `exp`, `ln`, `sqrt`, etc.)
  implemented via fixed-iteration CORDIC or Taylor with table lookup.
  Same input bits → same output bits, every platform, every run.
- All physics, fit, recognition, and any other real-valued code
  routes through `Real`. Lint / test discipline enforces no direct
  `f64` use outside `sim/arith` itself.
- Choice of underlying crate (e.g. `fixed`, `cordic`) settled at
  scaffold time; the API in `sim/arith` is the stable surface, the
  underlying crate can be swapped later.

**Note to revisit:** If M1 surfaces precision or dynamic-range issues
in physics integration, the first move is widening Q-format
(Q24.40 / Q16.48) per-module rather than abandoning fixed-point.
Switching arithmetic systems entirely would be a major undertaking
and is the last resort.
