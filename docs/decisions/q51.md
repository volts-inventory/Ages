## Q51 — Drop LLM from the project entirely
**Status:** Decided
**Context:** LLM narration was held as an optional M6 layer (Q46
draft) — a post-run pass producing in-world prose for the report.
Pressure-tested honestly, the LLM brings: dependency on an external
service / token budget, partial determinism (LLM responses vary
without aggressive caching), uncertain output quality (risk of
generic-AI feel), scope creep in M6 (integration, prompts, error
handling). What it adds — story-prose flavour — can be approximated
with templated prose (Mad-Libs-style fills from event data) for the
post-run report. The simulation's value is the real discoveries
(fitted equations from real physics) and the real civ-history arcs;
those are interesting on their own.
**Decision:** Drop LLM entirely from the project. No LLM in the
simulation loop, in the post-processing pipeline, or in the report
generation. The report uses templated prose only; the live CLI
stream uses formatted event lines. If a user wants LLM narration,
they can pipe `events.ndjson` into their own tooling separately —
the project does not integrate or maintain that. Replace the
narration capability with a **live CLI event stream** during the run
(see `docs/architecture.md`) so users can watch runs unfold without
waiting for the report.
**Lives in:** `PLANNING.md` Non-goals + Output goals,
`docs/architecture.md` Process layout (LLM out-of-scope note) + Live
CLI event stream, `AGENTS.md` Project intent #5.
