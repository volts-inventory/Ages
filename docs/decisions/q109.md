## Q109 — Vertical atmosphere stack (1.5D MVP)
**Status:** Decided
**Context:** Q90-Q108 give the planet a rich 2D atmosphere —
radiation, advection, hydrology, tides, magnetism, Coriolis,
seasons, eccentricity, diurnal cycling. But every cell is a
single column with one temperature value: there's no vertical
structure, no lapse rate, no separation between surface and
upper-atmosphere. Real atmospheres have a strong vertical
temperature gradient (~6.5 K/km on Earth) that drives cloud
formation, Hadley circulation, and convective instability.

A full 3D atmospheric stack is a multi-PR architectural
overhaul (vertical advection, density profiles, multi-level
state, vertical velocity field, …). Q109 ships the
**1.5D** minimum viable: one upper-atmosphere temperature
field per cell, coupled to surface temperature by a single
convective heat-exchange step.

**Decision:**

1. **`PhysicsState::upper_temperature: Vec<Real>`** — the
   upper-layer temperature per cell. Initialised to zero;
   bootstrapped on first integrate to `T_surface - 30 K`
   (a sensible Earth-tropopause-ish lapse).
2. **`VerticalConvection` law** runs after `HeatConduction`
   each macro-step:
   ```
   gap        = T_surface - T_upper
   xfer       = exchange_k · dt · gap
   T_surface -= xfer
   T_upper   += xfer

   space_gap  = T_upper - space_temperature
   T_upper   -= radiative_loss_k · dt · space_gap
   ```
3. **Earth-like defaults**:
   - `exchange_k = 0.05` (~20-tick equilibration)
   - `space_temperature = 100 K` (effective radiative-loss
     reservoir; not the cosmic background but the tropopause-
     emission temperature for our cooling budget)
   - `radiative_loss_k = 0.01` (~100-tick radiative timescale,
     matching real stratospheric cooling rates)
4. **Wired** through `build_laws` → `integrate_civ_step`
   takes a 14th `vertical: Option<&VerticalConvection>`
   parameter.

**Why this respects the project goals.**
- *Physics as close to real as possible:* the steady state of
  the two-layer system has `T_upper < T_surface` by a margin
  set by the radiative-loss / convective-exchange ratio.
  That's the *real* lapse rate emerging from the energy
  budget, not a hardcoded gradient. Tropical surface heating
  drives upward heat flux; polar stratospheric cooling drives
  downward heat flux. Both fall out of one law.
- *Queryable upper-atmosphere temperature:* future Q-decisions
  can read `state.upper_temperature()` for cloud altitude,
  upper-atmosphere wind, satellite-orbit drag, etc. The
  simulation now has a vertical dimension that didn't exist
  before.
- *Determinism:* per-cell read of `(T_surface, T_upper)`,
  per-cell write of both. No interior mutability, no per-tick
  allocation beyond two reusable buffers. Trivially
  deterministic across parallel state instances sharing a
  single law.
- *Scope-bounded:* the 1.5D MVP doesn't pretend to be full 3D
  Hadley. Real vertical advection (warm air *moving* between
  layers) is deferred. But the minimum scaffolding is now in
  place.

**Numerical care.**
- `exchange_k · dt = 0.05` per tick is well inside Euler
  stability for the convective gap closure.
- `radiative_loss_k · dt = 0.01` keeps upper-layer cooling
  smooth.
- The bootstrap (`T_upper = 0` → `T_upper = T_surface - 30`)
  triggers exactly once per cell on its first integrate.
  Pre-Q109 the upper-temperature field didn't exist, so
  there's no risk of clobbering meaningful data.
- Per-tick cost: O(N_cells) — same scale as `HeatConduction`
  and `Radiation::integrate`.

**Lives in:** `sim/physics/src/vertical.rs` (the law),
`sim/physics/src/state.rs` (`upper_temperature` field +
accessors),
`sim/physics/src/lib.rs` (re-export),
`sim/physics/src/orchestration.rs` (wired after Heat),
`sim/core/src/lib.rs` (`build_laws` returns 12-tuple).

**Follow-ups (not blocking).**
- *Real vertical advection.* Couple the upper layer to wind
  velocity for horizontal transport (currently the upper
  layer is per-cell only). Lands when the upper layer needs
  its own dynamics rather than just its own temperature.
- *Multi-level stack.* Surface, mid-troposphere,
  upper-troposphere, stratosphere — at least 3 layers for
  realistic Hadley. Big architectural change.
- *Vertical velocity field.* Real updrafts/downdrafts. Lands
  alongside multi-level stack.
- *Cloud-altitude tracking.* Q92 vapour condenses at `T <
  boil`; with Q109 the upper-layer T is now queryable, so
  Q92 follow-ups can route condensation to the upper layer
  (cloud formation) when surface is too warm.
- *Latent heat coupling at the upper layer.* Q100 latent
  heat currently warms the surface on condensation. Q109
  upper-layer condensation should warm the upper layer.
- *Q109-aware recognition templates.* "Tropopause", "high
  cloud cover", "stratospheric warming event" — all
  recognisable now that the upper layer is queryable.
