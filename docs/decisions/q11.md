## Q11 — Snapshot retention
**Status:** Decided
**Context:** Snapshots are full-state JSON written at fixed tick
intervals (now M1 onward, since physics-state replay benefits from them
from the start). Pure rolling window loses bookend states; pure
keep-all grows disk linearly with run length.
**Decision:** Lean hybrid. Rolling window of N=50 most recent
(configurable), plus permanent pins at run start and run end. Events
are kept indefinitely regardless. Later milestones can layer in
additional pin types (age transitions in M4, interesting-moment pins in
M6 alongside the post-run report once Q13 resolves).
**Lives in:** `docs/architecture.md` Snapshot retention.
