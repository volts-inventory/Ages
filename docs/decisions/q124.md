## Q124 — Box-section viewport layout
**Status:** Decided
**Context:** Q123 added a 1-line legend between the planet card
and the grid, but two pieces of feedback came back:

1. The 26-col legend was missing glyphs the renderer actually
   emits. `≈` (deep water), `△` (low mountain / hill), `▒`
   (inland land), `░` (coastal land) all appear in the grid
   on most seeds, but the Q123 key only listed `▲mt` and `~sea`.
2. The viewport sections (planet card, grid, log) ran together
   visually — the only divider was `-- log --`, the grid carried
   leftover markdown ` ```text ` fences from the post-run report
   path, and the legend was wedged between the card and the grid
   where it competed with both for attention.

**Decision:** Three coordinated changes.

1. **Box-section layout.** Each section gets a `-- {name} --`
   divider so the four parts of the viewport are visually
   addressable:
   ```
   -- planet --
   {4 card lines}
   -- map --
   {caption + grid}
   -- key --
   {2 legend lines}
   -- log --
   {log lines}
   ```
   Dividers appear only when their section content does — the
   `-- planet --` and `-- key --` rows are paired with the
   `--no-planet-card` flag (both vanish when the card is
   suppressed), and `-- log --` is paired with
   `--viewport-log-lines 0`. The `-- map --` divider is
   always shown because the map is always rendered.

2. **Move the legend above the log (was above the grid).** Q123
   put the legend between the card and the grid, on the theory
   it would prepare the reader for the upcoming glyphs. In
   practice that placement competed with the planet card for
   the reader's attention before the map was even visible.
   Q124 moves it after the map so it reads as a glossary you
   consult while staring at the grid — the placement matches
   typical map-key conventions.

3. **Expand the legend + strip markdown fences.** Two lines
   instead of one so every glyph the renderer emits gets a
   label:
   ```
   A=cap 1=civ #=war
   ~sea ≈deep ▲pk △hl ▒in ░cst
   ```
   Line widths: 17 + 27 cols. Both fit portrait phone terminals.
   Symbols match `render::terrain_symbol` (water depths +
   elevation tiers + coastal/inland flag) and
   `frame::centroid_symbol` / `claim_symbol` (civ overlays).

   Concurrently, the live viewport now strips the leading
   ` ```text\n ` and trailing ` ```\n ` fences from the body
   string returned by `render_world_frame_*`. Those fences are
   markdown ornament for post-run-report rendering; in the
   live viewport they read as ASCII noise. Stripping is a pure
   `str::strip_prefix` / `str::strip_suffix` chain — no
   allocation, no body mutation.

**Why this respects the project goals.**
- *Visual addressability:* a viewer asking "where's the year /
  the species / the legend?" can scan four divider rows
  instead of pattern-matching against content. Particularly
  useful on a portrait phone where the whole layout barely fits
  in one screenful.
- *No information loss:* the expanded legend documents every
  glyph the renderer can emit. The `*` (civ id ≥ 10) and ` `
  (gaseous-shell empty cell) markers stay implicit since they
  appear only on edge-case planets.
- *Determinism preserved:* divider strings are static
  constants; legend text is a static constant; fence-stripping
  is a pure string operation. No state, no per-tick branching.
- *Opt-out preserved:* `--no-planet-card` still hides the card
  + legend pair; `--viewport-log-lines 0` still hides the log.
  Power users running the viewport in a non-terminal sink keep
  full control.

**Numerical care.**
- `str::strip_prefix` returns `Option<&str>`; the Q124 chain
  uses `unwrap_or(&body)` / `unwrap_or(stripped)` so a body
  without the expected fences (e.g. an empty-grid edge case)
  passes through untouched instead of panicking.
- Legend symbols are valid UTF-8 in the source file
  (`as_bytes()` on `&str` is the same byte sequence the
  renderer emits per cell), so terminals with wide-char
  rendering land each glyph in 1 column — same width as in
  the body.

**Lives in:** `sim/report/src/viewport.rs` — `render()` writes
the four section dividers and the new 2-line legend; the body
gets fence-stripped before write.

**Follow-ups (not blocking).**
- *Variable-width dividers.* `-- planet --` (12 cols) and
  `-- map --` (9 cols) have different visual heft. Padding to a
  fixed grid-width (`-- planet ----------` etc.) would look
  more "boxy" but adds rendering complexity for marginal value.
- *Caption in its own section.* `Y{n} M{n} · {civ} civ · F/C`
  currently rides at the top of the `-- map --` section. A
  separate `-- now --` section would highlight current state
  but adds another divider row.
- *Color the dividers.* ANSI dim grey or muted blue would make
  the dividers recede so they delineate without competing.
  Pure cosmetic; deferred until a user asks.
