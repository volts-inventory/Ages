## Q16 — Catalog generation approach (static vs dynamic vs hybrid)
**Status:** Superseded by Q40
**Context:** Could phenomena be generated dynamically from planet
physics rather than from a curated catalog?
**Decision:** Originally hybrid — archetype kinds authored, parameters
computed from planet/species. Superseded by Q40, which goes further:
phenomena emerge from a real physics engine. Authoring shifts from
phenomena to physics laws + recognition templates. The "hybrid" framing
holds in spirit (laws + templates authored, behaviour emerges) but the
execution is fundamentally different.
**Lives in:** `docs/world.md` Recognized phenomena section,
`docs/architecture.md` Physics engine section.
