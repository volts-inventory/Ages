## Q123 — Cognition topology on species line + map legend
**Status:** Decided
**Context:** Q122 fit the planet card in 30 cols and made compact
the default, but two small follow-ups remained:

1. The Q120 species line `{name} {tier}-cog {mod}/{manip}` had
   measurable slack on most seeds — `Cyranites med-cog touch/tent`
   is 28 chars, leaving 4 chars of trailing whitespace on a 32-col
   phone terminal. Q120's follow-ups list called this out:
   *"Topology + sociality on the species line. Currently we show
   cog + modality + manip but not `cognition_topology`
   (centralised vs distributed) or `sociality`."* The slack was
   the natural place to land that.
2. The grid uses a glyph alphabet (`A`, `1`, `#`, `~`, `≈`, `▲`,
   `△`, `▒`, `░`) that a reader has to learn from out-of-band docs.
   No legend in the live viewport meant the user had to remember
   what each glyph stood for, which is the kind of friction
   compact-mode is supposed to eliminate.

**Decision:** Two changes, shipped together because they both
trade against the same vertical-space budget:

1. **Cognition topology on the species line.** The `{tier}-cog`
   token (7 chars) becomes `{topo3}-{tier}` (5-7 chars), where
   `topo3 ∈ {cnt, dst}`. The cog tier is also tightened from
   `low/med/hi` to `lo/med/hi` so the new token never grows. The
   `-cog` suffix drops because the position is now self-evident
   (it's the trait token between name and modality/manipulation).
   ```
   Q120: Cyranites med-cog touch/tent          (28 cols)
   Q123: Cyranites cnt-med touch/tent          (28 cols)
   ```
   Same width, +1 trait dimension. Reads as
   "centralized-medium-cognition, touch sensorium, tentacle
   manipulation" — a 4-axis at-a-glance description of the natives.

2. **1-line legend above the grid.** A static 26-col legend
   appears between the planet card and the frame body whenever
   the planet card is shown. Symbol set picked for the
   highest-frequency / lowest-discoverability glyphs:
   ```
   A=cap 1=civ #=war ~sea ▲mt
   ```
   - `A` = civ centroid (Q57 territory focus)
   - `1` = single-civ claim (digit cycles per civ id)
   - `#` = disputed / overlapping claims
   - `~` = water (shallow + deep collapse to the same icon for
     legend brevity; deep water `≈` and inland/coastal land
     `▒`/`░` left for the reader to infer from context)
   - `▲` = mountain (also stands in for low-mountain `△`)

   Hidden together with the card under `--no-planet-card` so
   minimal-render setups stay minimal. Drawn once per frame; sim
   state and event stream are byte-identical to Q122.

**Why this respects the project goals.**
- *Information density without width inflation:* the species line
  picks up `cognition_topology` at the same width as before. The
  legend adds 1 row that pays for itself the first time a viewer
  asks "what's the `▲`?".
- *Pure presentation change:* no sim-state mutation, no event
  changes, no protocol changes. `cognition_topology` was already
  in `SpeciesDerived` (Q47); Q123 just surfaces it.
- *Determinism preserved:* the legend is a static const string
  and the topology mapping is a pure 2-arm match; same seed →
  identical output bytes.
- *Opt-out preserved:* `--no-planet-card` hides both the card and
  the legend; the legend is part of the "planet identification"
  pre-roll, not core content.

**Numerical care.**
- `cog_tier` boundaries unchanged (< 0.34 → lo, < 0.67 → med,
  else hi). String labels swapped only.
- `cognition_topology` field is a String in the protocol; Q123's
  match falls through to `"?"` for any unknown variant so the
  viewport never panics if `sim_species` introduces a third
  topology in the future.

**Lives in:** `sim/report/src/viewport.rs`
(`planet_card`'s species-line format; `render`'s legend write
between card and body).

**Follow-ups (not blocking).**
- *Sociality on the species line.* Still open from Q120. The
  species line is now near saturation on long names + long
  modality/manip combos (`Sepharites cnt-med vis-pol/burrow`
  ≈ 33 cols). Adding sociality would require dropping another
  token or moving to a 5th card line.
- *Topology-aware naming.* Distributed-cognition species feel
  like they should draw from a different name pool than
  centralized ones. Pure flavour; deferred.
