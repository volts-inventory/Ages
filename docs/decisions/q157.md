# Q157 — Experiment apparatus: civs run controlled-conditions interventions

**Status:** Decided.

## What

Add a tier-2 capability tool, `ToolKind::ExperimentApparatus`, that
turns a civ from a passive observer into an active experimenter.
Each unlocked apparatus clamps a physics channel at one of four
ladder values pre-physics, then samples the apparatus cell's
`measure_channel` post-physics, feeding `(clamp_value, response)`
pairs into the civ's hypothesizer measurement track. The clamp
cycles through four values keyed on `tick % 4` so a 4-tick window
spans the full ladder.

## Why

Pre-Q157 the discovery pipeline derives every law from passive
observation: civs sample whatever the planet's heterogeneity
provides and curve-fit the result. Real science isn't only that —
it's also intervention. Galileo dropped weights from controlled
heights; Faraday clamped current and measured deflection;
Lavoisier sealed jars and weighed combustion products. The pure-
observation civ in this sim was a structurally-Aristotelian one:
it could correlate but never cleanly isolate.

Apparatus cells unlock the Galileo move: hold one channel fixed,
measure another, recover the underlying coefficient cleanly.
Heat-diffusion `α` from a clean source-sink experiment converges
in dozens of ticks; from passive planetary gradient observation
it can take thousands. The same fit machinery (Q33/Q34) processes
both kinds of samples — apparatus contributions just feed the
existing measurement-relation buffer with controlled `(x, y)`
distributions instead of whatever the planet happens to provide.

The tool gate also reinforces the project's "different sciences"
goal. Gated on `MANIPULATION_PREREQ = ToolExtension`, a tactile-
chemical drift species without tool-using manipulation modes can
*never* build the apparatus. Its science stays observational
forever — Aristotelian by virtue of its body plan, not by
authorial choice.

## How

### Apparatus

A civ-level `Apparatus { cell, clamp_channel, measure_channel }`
record. One apparatus per civ at unlock. The cell is picked
deterministically from `Civ::claimed_cells` (lowest-population,
ties broken by `cell_id`) so the experiment lives on a sparsely-
occupied frontier rather than the capital.

The clamp ladder is a fixed per-channel constant in raw physics
units. Temperature spans freeze→ignition (250 / 300 / 400 / 500 K);
charge stays below the EM discharge threshold (0 / 10 / 25 / 45)
so the apparatus never fires artificial lightning. `Elevation` is
not clampable — geology doesn't change in one tick.

### Tick cycle

Pre-physics (in `physics_phase`), each civ's apparatus writes its
clamp value into the physics state at the apparatus cell. The Q75
prev-state snapshot then captures the clamped state, so this
tick's `TemporalDelta` reads the relaxation response.

Post-physics (in `cohort_and_figure_phase`, after
`observe_per_figure`), each civ samples its apparatus cell's
`measure_channel` and feeds `(clamp_value, post_phys_value)` into
the first active figure's hypothesizer via
`Hypothesizer::record_experimental_measurement`. That helper:

- Pushes the sample twice into the buffer for the relation
  `measurement_relation_id(Direct(measure_ch), Direct(clamp_ch))`
  — the 2× weight expresses the information-density advantage of
  controlled samples.
- Lazily inserts a `MeasurementCandidate` if the
  `(measure_ch, clamp_ch)` pair isn't already in the Q73 default
  catalogue, so apparatus-only relations still flow through
  `step_with_cosmology_and_doubt`'s fit pipeline.
- Bumps `experimental_count_by_relation[relation_id]` by 2; this
  count is read at confirm time so
  `ConfirmedMeasurement.is_experimental` flags any relation that
  received apparatus contributions.

### Tech gate

Tier 2 (same as `ThermalSensor` / `RemoteAcoustic`):

- `MANIPULATION_PREREQ = ToolExtension` (universal Q55 gate;
  enforced in `is_buildable`)
- `observation_threshold = 30_000`
- `literacy_floor = 0.30`
- `relation_prereqs = [(template_id 1, InfraredThermal)]` — the
  civ has confirmed a fire / heat-equivalent law, demonstrating
  it's done some real fitting before it builds a controlled-
  conditions device
- `tool_prereqs = []`
- `species_maturity_floor = 0`
- `prereq_channels = []` (synthetic substitute, no native-channel
  prereq)
- `granted_channels = []` (effect on discovery layer, not perception)

### Protocol

`MeasurementConfirmed` gains `is_experimental: bool`
(`#[serde(default)]` so Pre-Q157 event logs decode cleanly). The
flag persists with the relation (`ConfirmedMeasurement`) so
successor civs and the post-run report can distinguish
experimentally-derived laws from passively-observed ones.

### Determinism

Apparatus cell selection is deterministic at unlock. The clamp
ladder is constant. The per-tick step index is `tick % 4`. No
RNG is threaded through the apparatus path. Apparatus-cell
iteration over `civs: &[Civ]` walks the slice in order; each
civ's `apparatus_cells: Vec<Apparatus>` walks its `Vec` in order.
No `HashMap`. Byte-replay holds.

## Trade-offs considered

- **Selection-only (no intervention).** A `MeasurementChannel::Within`
  variant filtering samples by another channel's band would have
  been ~100 lines. Cheap, no protocol, no new physics. Rejected
  because selection bias isn't intervention; it doesn't deliver
  the meaning of "experiment."
- **Tool-as-multiplier.** Adding the tool as a confidence-bonus
  multiplier on certain forms (~50 lines). Rejected against the
  project's "no fudge constants where derivation is possible"
  principle — a multiplier hides the absence of the underlying
  mechanic.
- **More apparatus per civ.** One per civ at unlock keeps the
  cost bounded. Scaling to 3 with literacy is a tuning lever for
  later if the rate of apparatus-confirmed relations is too low.

## Acceptance

- `Channel::Temperature.clamp_ladder()` returns four values
  spanning freeze→ignition; `Channel::Elevation.clamp_ladder()`
  returns `None`.
- A no-tool species (no `ToolExtension`) cannot build the
  apparatus regardless of any other gate.
- `Hypothesizer::record_experimental_measurement` pushes the
  sample twice and bumps the count; on confirm the
  `ConfirmedMeasurement.is_experimental` flag is `true`.
- Pre-Q157 NDJSON event logs continue to decode (the new
  `is_experimental` field defaults to `false`).
