## Q125 вЂ” Full-width centered viewport layout
**Status:** Decided
**Context:** Q124 added `-- planet --` / `-- map --` / `-- key --`
/ `-- log --` dividers, but they hugged the labels (12 chars max)
while the planet card and grid stretched to ~28-30 chars. The
result felt visually unbalanced вЂ” short divider rows floating
above wide content rows. Two related observations:

1. *Available width was wider than Q122 assumed.* Q122 had
   targeted в‰¤ 30 cols to fit a portrait phone terminal at the
   smallest font. The user pointed out that on actual phone
   code-block displays the rendered output had visible right-edge
   slack вЂ” multiple cards had 4-8 chars of trailing whitespace
   before the visible boundary.
2. *Some card lines lost their `В·` separators in Q122.* Lines 2,
   3, 4 of the planet card dropped the bullet-separator to fit
   30 cols; the resulting `aqueous reducing 2 moons` reads less
   clearly than `aqueous В· reducing В· 2 moons` because the
   reader has to mentally re-tokenize.

**Decision:** Pick a single visual width and align everything to
it.

1. **`VIEWPORT_WIDTH = 40`.** The dividers stretch to 40 chars
   with the section label centered:
   ```
   ---------------- planet ----------------
   ----------------- map ------------------
   ----------------- key ------------------
   ----------------- log ------------------
   ```
   Width chosen to comfortably hold the longest realistic card
   line вЂ” `strong mag В· 241F В· 129h В· 17В° В· 11mo` в‰€ 36 chars вЂ”
   with 4 chars of margin. A small helper
   `divider(label) -> String` does the math:
   `lhs = (W - len(" label ")) / 2`,
   `rhs = (W - len(" label ")) - lhs`.

2. **Restore `В·` on every multi-field card line.** Lines 2-4
   of the card go back to bullet-separated form, matching
   line 1's `{name} В· {badge}` and the caption's
   `Y{n} M{n} В· {civ} civ В· {F}F/{C}C`. The legend gains the
   same separator (`A=cap В· 1=civ В· #=war В· ~sea В· в‰€deep`) so
   every list-of-fields in the viewport reads with the same
   delimiter.

3. **Expand the species-line abbreviations.** With 40 cols of
   width budget, the Q123 `cnt-med` token expands to `cent-med`
   (centralized vs. distributed cognition), and the cog tier
   restores from `lo/hi` to `low/high`:
   ```
   Q123: Cyranites cnt-med touch/tent
   Q125: Cyranites В· cent-med В· touch/tent
   ```
   Same trait dimensions, fewer cryptic abbreviations.

4. **Center the map within 40 cols.** The compact grid is
   `3 + grid_width` chars wide (3-char row prefix + 1 char
   per cell); on the default 24-wide grid that's 27 chars,
   leaving 13 chars of slack. Center pad =
   `(VIEWPORT_WIDTH - max_line_width) / 2` is computed once
   per frame from the widest body line, then prepended to every
   non-empty body line. Header rows + grid rows + caption all
   shift by the same offset so the grid stays column-aligned.

**Why this respects the project goals.**
- *Visual harmony:* one width, one separator (` В· `), four
  visually identical dividers, one centering offset for the map.
  The viewport stops looking like a stack of independently sized
  fragments.
- *Information clarity:* `cent-med` reads as
  "centralized-medium" and `dist-high` as "distributed-high"
  without requiring the reader to remember a 3-letter
  abbreviation table. The bullet-separated card lines tokenize
  on their own.
- *Determinism preserved:* `divider()` is a pure function; the
  centering pad is computed from a deterministic body string;
  the map output is byte-stable for a given seed + tick.
- *Phone-fit preserved:* 40 cols still fits portrait phone
  code-block displays. The viewport tests run with
  `use_alt_screen: false` and the standard test rig is
  unaffected вЂ” only sample seed output now sits in the
  40-col layout.

**Numerical care.**
- `divider("planet")` produces 40 chars exactly:
  `lhs (16) + " planet " (8) + rhs (16) = 40`. For odd-length
  labels (`map`, `key`, `log`), `lhs (17) + " map " (5) + rhs (18) = 40`
  вЂ” the extra dash lands on the right.
- `chars().count()` is used for body-line widths instead of
  `len()` because the legend includes terrain glyphs (`в–І в–і в–’ в–‘ ~ в‰€`)
  that are multi-byte UTF-8 but display 1 col wide; byte count
  would overcount and skew the centering.
- The map-pad computation uses
  `VIEWPORT_WIDTH.saturating_sub(max_line_width)` so a wider-
  than-`VIEWPORT_WIDTH` grid (e.g. user-configured
  `--grid-width 50`) doesn't underflow вЂ” pad falls to 0 and
  the grid prints flush-left.

**Lives in:** `sim/report/src/viewport.rs`
(`VIEWPORT_WIDTH` const, `divider()` helper, `planet_card()`'s
restored bullets + expanded species line, `render()`'s
divider calls + map centering loop).

**Follow-ups (not blocking).**
- *Configurable width.* `VIEWPORT_WIDTH` is a const; users on
  desktop terminals could comfortably hold 60-80 cols. Adding a
  `--viewport-width` CLI flag is straightforward but waits for
  a user signal.
- *Color the dividers.* Same Q124 follow-up вЂ” ANSI dim grey on
  dividers would make them recede. Pure cosmetic.
