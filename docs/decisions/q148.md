## Q148 — Log polish: legend glyphs, cosmology axis, conflict suppression
**Status:** Decided
**Context:** Three small but unrelated nicks in the live viewport's
log + legend, surfaced while watching seed 495 (Lumen-h) play out:

1. **Legend missed two of Q139's fallback glyphs.** Q139 had added
   `≡` for gas-giant cloud bands and `·` for low-relief / sub-
   surface-ocean / oceanic-basin floors, replacing the bare blank-
   space fallback. The viewport's key section still listed only
   the pre-Q139 glyphs (`A=cap · 1=civ · #=war · ~sea · ≈deep` +
   `▲peak · △hill · ▒land · ░coast`), so a viewer staring at a
   `·`-or-`≡`-textured map saw glyphs the legend never named.

2. **Cosmology shift was anonymous.** Q145 surfaced
   `CosmologyShifted` events in the log as `civ N cosmology
   shift` — useful as a beat marker but uninformative about
   *which* axis moved. The event already carries the full 5-axis
   `axes_q32: Vec<i64>` payload (`empirical, communitarian,
   reformist, mystical, hierarchical`), so the viewport had the
   data to name the dominant axis but threw it away.

3. **Conflict resolved fired per skirmish, not per war.** Q66's
   multi-tick war model emits a `ConflictResolved` event for each
   tick a war runs, with `loser_defeated: bool` — true only on
   the last tick when the loser's pop fell below floor and they
   surrendered. Surfacing every per-skirmish event in the log
   flooded the small (3-line default) log buffer with
   `conflict resolved` lines while the war was still being
   actively fought; the viewer wanted to see the *outcome*
   (`civ X defeated`), not the running tally.

**Decision:** Three coordinated viewport-only edits in
`sim/report/src/viewport.rs`. None touch sim state, the event
schema, or determinism — pure presentation polish.

1. **Extend the legend.** Split the two lines into the sidebar
   (Q149 lands the side-by-side layout in the same PR), 4 short
   lines each ≤ 28 cols:
   ```
   A=cap · 1=civ · #=war
   ~sea · ≈deep · ▲peak
   △hill · ▒land · ░coast
   ·=plain · ≡=gas
   ```
   Every glyph the renderer can emit now appears in the legend.

2. **Track previous cosmology per civ + name the dominant axis.**
   Add `civ_cosmology: BTreeMap<u32, [f64; 5]>` to
   `ViewportEmitter`. On each `CosmologyShifted` event, the log
   formatter computes the delta between the new `axes_q32` (Q32-
   decoded to f64) and the prior snapshot, finds the axis with
   the largest absolute change, and emits:
   ```
   civ N cosmology shift: {axis_name}{+/-}{|delta|:.2}
   ```
   Axis names are indexed by `axes_q32` vector position — `[0]`
   empirical, `[1]` communitarian, `[2]` reformist, `[3]`
   mystical, `[4]` hierarchical. First shift on a civ has no
   prior snapshot — treat the prior as zero so the line still
   names the dominant absolute-position axis. Snapshot is
   updated *after* `log_message` reads it, so the next shift's
   delta is computed against the just-arrived vector. Cleared on
   `CivCollapsed` so a refound civ_id starts fresh.

3. **Suppress mid-war skirmishes from the log.** Match
   `Event::ConflictResolved(c)` and early-return `None` when
   `c.loser_defeated == false`. The final skirmish (the only
   `ConflictResolved` with `loser_defeated = true` per war)
   surfaces as:
   ```
   conflict resolved (civ {winner} vs civ {loser} — civ {loser} defeated)
   ```
   Now one log line per war, naming both participants and the
   outcome, instead of N lines for N tick-skirmishes.

**Why this respects the project goals.**
- *No new event types.* All three changes consume existing
  payload fields (`axes_q32`, `loser_defeated`) that the sim
  already emits — no protocol churn.
- *Determinism preserved.* The viewport is a pure observer over
  the event stream. Mirrored state (the cosmology snapshot map)
  is keyed on `civ_id` and updated in event order; same NDJSON
  in → same log lines out.
- *Phone-portrait friendly.* Each new legend line stays ≤ 28 cols
  so it slots into the Q149 sidebar without truncation.

**Numerical care.**
- Cosmology delta uses `f64` after `q32_to_f64` decode; the axis
  values are bounded `[-1, 1]` so the absolute difference is at
  most `2.0`. `{:.2}` format gives 0.01 resolution, plenty for
  a viewer-readable label.
- Tie-breaking on equal absolute deltas: `>` (strict greater)
  in the loop means the *earliest* axis wins on a tie, which is
  deterministic.

**Lives in:**
- `sim/report/src/viewport.rs` — `log_message` rewritten to take
  the cosmology snapshot map; `civ_cosmology` field added;
  `apply_state` updates the snapshot post-format and clears it
  on `CivCollapsed`; legend lines moved into
  `build_sidebar_lines` (Q149 surface).

**Follow-ups (not blocking).**
- *Cosmology shift colour cue.* Each axis could colour its name
  in the log line (empirical = blue, mystical = purple, etc.) so
  a sequence of shifts paints a quick character-arc. Pure
  ornament; defer.
- *War-narrative summary in the log.* Could pre-format the
  outcome with year-of-start and casualty fraction
  (`y34→y36 war: civ 1 defeated civ 3, 60% pop loss`). The
  fields exist (`tick`, `loss_fraction_q32`); skipped here for
  log-line brevity.
