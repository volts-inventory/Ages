## Q29 — Modality and manipulation taxonomy
**Status:** Decided
**Context:** The species traits include `communication.modalities` and
`manipulation_modes`, both multi-valued lists drawn from a vocabulary
that needs to be defined and bounded. How many modalities to support,
how to model multi-modal species (additive, dominant, weighted?),
whether new modalities should be authorable.
**Decision:** Bounded enums with per-channel parameter structs and
environment-gated availability. Aggregation across multiple modalities
is *vector* (each modality is independent; downstream consumers query
per channel rather than collapsing to a scalar). YAML-authorable
additions deferred.

**Modalities (15):** `acoustic_air`, `acoustic_water`, `seismic`,
`visual_light`, `visual_polarization`, `bioluminescent`,
`chemical_pheromone`, `chemical_taste`, `tactile`, `electric_field`,
`magnetic_sense`, `infrared_thermal`, `radio_native`, `gestural`,
`postural`. Each modality carries
`(range_m, fidelity, bandwidth, env_gate)` where `env_gate` is a
predicate over planet/region state (e.g. `visual_light` requires
`irradiance > X`; `acoustic_air` requires atmospheric pressure > X;
`acoustic_water` requires liquid-water surface coverage; `electric_field`
requires conductive medium). The env_gate is what makes sub-surface-
ocean and tidally-locked planets seed genuinely different species.

**Manipulation modes (12):** `limb_grasp`, `tentacle`, `mouth_beak`,
`tongue_prehensile`, `trunk`, `mandible`, `fluid_jet`, `tool_extension`,
`web_construct`, `burrow`, `electric_discharge`, `chemical_secretion`.
Each carries `(force_n, precision_m, dexterity_score, dof_count)`. Tier
gating (e.g. T1+ material culture requires `tool_extension`) is the
responsibility of the downstream consumer in `sim/civ`, not encoded in
the taxonomy.

**Aggregation:** Multi-modal species hold the modality vector
unchanged; recognition / transmission code consumes channels
individually. No scalar fidelity collapse — environmental gating per
channel is required for cross-planet differentiation.

**Lives in:** `docs/world.md` Species section (modality + manipulation
taxonomy), `sim/world` species derivation when implemented in M3.
