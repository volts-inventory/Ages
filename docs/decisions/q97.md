## Q97 — Seasonal insolation swing in `Radiation`
**Status:** Decided
**Context:** Q90 shipped a per-row equilibrium temperature
table (`t_eq_per_row: Vec<Real>`) computed once at planet build
from latitude × stellar irradiance × albedo. The table was
*annual mean* — every macro-step relaxed cell temperatures
toward the same fixed gradient, so summer hemispheres never
warmed and winter hemispheres never cooled. The Q90 doc called
this out as a deferred follow-up: "Sub-solar latitude shifts
with `axial_tilt × month-of-year`; the per-row table becomes
per-(row, month). Lands when the sim wants summer/winter
contrast as a recognition target."

Q93's `state.macro_step()` clock plus Q94/Q95 both used as a
proof that the macro-step counter is the right shape for
periodic forcings. Q97 cashes in: feed it into `Radiation` to
make sub-solar latitude actually move with the seasons.

**Decision:** Promote `t_eq_per_row: Vec<Real>` to
`t_eq_per_row_per_season: Vec<Vec<Real>>` — outer index is
the row (latitude band), inner index is one of 12 seasonal
slices over the orbital year. Each season's column is computed
with a different sub-solar row offset:

```text
  raw_offset(season) = max_axial_tilt_rows × triangle_cos(2π × season / 12)
  sub_solar_row(season) = half_h - raw_offset(season)
  pole_dist(row, season) = |row - sub_solar_row(season)|
  T_eq(row, season) = ...same Stefan-Boltzmann formula as Q90,
                       anchored at sub_solar_row instead of half_h...
```

`triangle_cos` is a 4-segment piecewise-linear approximation of
`cos(2π × s/12)` over the integer season indices `s ∈ [0, 12)`.
- `s = 0` → `+max` (sub-solar at northernmost extreme; N summer)
- `s = 3` → `0` (autumn equinox)
- `s = 6` → `-max` (sub-solar at southernmost; S summer)
- `s = 9` → `0` (spring equinox)

Triangular not `cos(...)` because Q32.32 has no `sin`/`cos` and
the order-of-magnitude shape is identical at the per-row
resolution we have (~9 rows, ~1-row tilt range for typical 23°
worlds). Cosine refinement deferred until Q32.32 trig table
arrives.

The integrate path computes the season index from
`state.macro_step()`:

```rust
season = (macro_step × 12) / year_macros) % 12
```

`year_macros = orbital_period_months × 30` for our Q67 1-day-
per-macro-step cadence. Earth defaults to 12 months × 30 days
= 360.

**Why this respects the project goals.**
- *Physics as close to real as possible:* northern-hemisphere
  rows actually warm in N summer and cool in N winter; the
  swing magnitude scales with `axial_tilt_deg`. A 0° tilt world
  collapses cleanly to Q90's annual-mean behaviour. A 45° tilt
  world (extreme but in our worldgen range) flips its hottest
  band from "near equator" to "near 45° N" twice per year.
- *Least hardcoded values:* one new constant
  (`SEASONS_PER_YEAR = 12`); everything else flows from
  `Planet::axial_tilt_deg` and `Planet::orbital_period_months`.
  Per-cell, per-season insolation is computed once at planet
  build — the integrate hot path only does a 2D table lookup.
- *Determinism:* `state.macro_step()` is the only state input;
  table is immutable post-init; pair iteration unchanged.
- *Reusability:* the same `state.macro_step()` clock Q93 tides
  reads now drives Q97 seasons. Future Q-decisions for diurnal
  cycling can use a sub-macro-step clock or read the same
  counter at a different cadence.

**Numerical care.**
- Pre-computed table is `height × 12` entries (~108 for Earth-
  sized 9-row grids). Negligible memory; computed once.
- Integrate path: one table lookup per cell per macro-step,
  zero transcendentals. The pre-Q97 path also did one lookup
  per cell — no perf change in the hot path.
- `axial_tilt_rows = axial_tilt_deg.clamp(0,45) × half_h / 90`
  uses integer arithmetic; for `half_h=4` and tilt=23°,
  `axial_tilt_rows = 1`. Small but nonzero on Earth-sized grids.
- The triangle-wave `raw_offset` calculation uses i64
  arithmetic with `.max(1)` denominators so a degenerate
  `quarter == 0` doesn't divide by zero.

**Lives in:** `sim/physics/src/radiation.rs` (the table + the
`season_index` accessor + integrate-path lookup),
`sim/core/src/lib.rs` (`build_laws` derives `axial_tilt_deg` +
`year_macros` from the planet and threads them into
`Radiation::for_planet`).

**Follow-ups (not blocking).**
- *Cosine bulge profile.* Same Q32.32 trig follow-up the Q93
  / Q94 docs already track. Triangle approximation is fine for
  now.
- *Diurnal cycling.* Sub-macro-step day/night insolation. Real
  ground temperatures swing 10-20 K daily; our resolution
  collapses that into the macro-step. Lands when civs need
  diurnal recognition templates (sunrise warmth, night cooling).
- *Eccentric orbits.* Real orbits aren't circular; insolation
  varies slightly with planet–sun distance (Earth's
  perihelion-aphelion gives ~6 % swing in W/m²). Lands when
  `Planet` carries an eccentricity field.
- *Seasonal recognition templates.* "Polar winter dark", "midnight
  sun", "monsoon onset" — all recognisable now that hemispheres
  swing seasonally. Authoring lands when civs accumulate
  enough seasonal observations to fit form vocabulary.
