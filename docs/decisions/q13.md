## Q13 — Interesting moments surfacing
**Status:** Decided
**Context:** A long run produces thousands of events; the post-run
report's highlight reel and the live CLI `--cli=highlights` mode need
to know which events are meaningful. With LLM narration dropped (Q51),
the criteria serve the deterministic report and the live stream only.
**Decision:** Hybrid (option C in the audit) — structural pin plus
scored long tail.
- **Structural pins** (always highlighted regardless of score): civ
  founding, civ collapse, catastrophe, run-end, persistence-tech tier
  crossings, reach-tech tier crossings, first-of-kind discoveries,
  named-figure promotions, sensorium-extending tech inventions,
  inter-civ artifact comprehension events.
- **Scored long tail** (everything else): score per event =
  `w_n·novelty + w_m·magnitude + w_f·figure-significance +
  w_a·arc-coherence`. Top N% by score added to highlights (default
  N≈5%, configurable). Weights tuned during M6.
A and B were rejected: A (pure scoring) risks false negatives for
major events if weights are off; B (pure whitelist) loses granularity
within a type. C avoids both.
**Lives in:** `docs/architecture.md` Live CLI event stream + Post-run
report sections.
