## Q127 — Themed card layout, descriptive labels, biochem
**Status:** Decided
**Context:** Q126 settled on a 5-line planet card with
abbreviated tier names (`low-soc`, `low-comm`) and the literal
`-- planet --` section divider. User feedback raised four
points:

1. *Descriptive labels.* `aqueous`, `reducing`, `low-soc`,
   `low-comm`, `boiling-off` were technically correct but
   jargon-heavy. A reader scanning the card had to translate
   each token instead of reading it.
2. *Planet name in the divider.* The literal `-- planet --`
   header was uninformative — the section is *about a planet*
   but not *about THIS planet*. Putting the planet name in the
   divider lets the section header *be* the identity.
3. *Carbon-based vs silicon-based biology.* The substrate enum
   already implies the species's biochemistry (silicate →
   silicon, everything else → carbon), but it never surfaced
   on the card. A reader couldn't see at a glance that Earth-
   norm life is on this planet vs. exotic crystalline life.
4. *Field grouping.* The Q122-Q126 card had `substrate · atm ·
   moons` on one line (mixing chemistry + astronomy) and
   `mag · temp · day · tilt · year` on another (mixing climate +
   orbital). Reorganising by theme — "what's the world made
   of, what's the climate, what's the rotation/orbit" — makes
   the card scannable.

**Decision:** Five coordinated changes.

1. **Planet name → section divider.** `render()` calls
   `divider(&p.name)` for the planet section header instead of
   `divider("planet")`. The Q125 `divider()` helper auto-pads
   to `VIEWPORT_WIDTH` so any name length stays centered:
   ```
   ---------------- Lyra-a ----------------
   ---------------- Mira-a ----------------
   --------------- Caelum-a ---------------
   ```
   When `Planet` hasn't been seen yet, falls back to `"planet"`.

2. **Descriptive substrate labels.** New `substrate_descriptor()`
   maps internal substrate names to "what's life made of"
   English:
   | Internal | Display |
   |---|---|
   | aqueous | water-based |
   | ammoniacal | ammonia-based |
   | hydrocarbon | methane-based |
   | silicate | silicon-based |

3. **Descriptive atmosphere labels.** New
   `atmosphere_descriptor()` maps the `Atmosphere` variants to
   composition hints:
   | Internal | Display |
   |---|---|
   | none | vacuum |
   | thin | thin |
   | oxidising | oxygen-rich |
   | reducing | methane-rich |
   | hazy | hazy |

4. **Descriptive sociality + communication labels.** Replace
   Q126's `low-soc` / `low-comm` with semantic words that
   match what the underlying scalar measures:

   - **Sociality** (eco-behaviour scalar):
     `solitary` (< 0.34) / `social` (0.34-0.67) / `eusocial` (> 0.67)
   - **Communication fidelity** (signal-quality scalar):
     `noisy` (< 0.34) / `clear` (0.34-0.67) / `precise` (> 0.67)

5. **Biology line gains biochemistry.** Substrate-implied
   biochem appended as the 4th token:
   ```
   {lifespan}y · {sociality} · {comm} · {biochem}
   ```
   `substrate_biochem("silicate") = "silicon"`, everything else
   = "carbon" — matching the substrate catalogue's only
   silicon-substrate entry.

**Re-themed card layout (5 lines, was 5 lines):**
```
Q126:                                  Q127:
Lyra-a · boiling-off                   ---- Lyra-a ----      (in divider)
aqueous · reducing · 2 moons           boiling-off · water-based
strong mag · 241F · 129h · 17° · 11mo  methane-rich · 241F · strong mag
Cyranites · cent-med · tactile/...     129h · 11mo · 17° · 2 moons
44y life · low-soc · low-comm          Cyranites · cent-med · tactile/tentacle
                                       44y · solitary · noisy · carbon
```

Lines 1-3 of the body now read as themed groups:
- *Line 1 (identity):* "How is this world for its inhabitants,
  and what's life chemistry here?" — badge + substrate.
- *Line 2 (climate):* "What's the air / sky like?" — atmosphere
  + temperature + magnetism.
- *Line 3 (orbital):* "What does the sky cycle look like?" —
  day length + year length + axial tilt + moons.
- *Line 4 (species traits):* the Q120 species line, unchanged.
- *Line 5 (biology):* lifespan + sociality + comm + biochem.

**Why this respects the project goals.**
- *Q47-honest:* sociality/comm tier scalars and substrate type
  are read straight from `Species` / `Planet`. No new sampling,
  no new state, no new event payload — Q127 is purely a
  presentation reorganisation of what was already there.
- *Reader-friendliness:* every label is now an English phrase
  that a non-domain reader can parse without consulting a
  glossary. "Methane-based, oxygen-rich, eusocial, precise,
  carbon" reads like a descriptive sentence; "hydrocarbon,
  oxidising, high-soc, high-comm" reads like a database.
- *Themed grouping:* the four card body lines each answer one
  question — identity / climate / orbit / who lives there /
  what they're like biologically. A viewer can skip-read by
  theme.
- *Determinism preserved:* all changes are pure functions of
  pre-existing protocol fields. Same seed → same output bytes.

**Numerical care.**
- `substrate_descriptor` / `atmosphere_descriptor` /
  `substrate_biochem` return `&'static str` — no allocation,
  no lifetime juggling. Unknown variants fall through to
  `"unknown"` so the viewport stays robust against future
  substrate / atmosphere additions.
- `sociality_label` / `comm_label` use the same `< 0.34 / <
  0.67 / else` thresholds Q120 used for cognition tiers, so
  the three trait scalars all bucket consistently.
- Field reorganisation didn't change line widths past Q125's
  40-col envelope. Worst-case width is line 4 (`Sepharites ·
  cent-high · vis-pol/tentacle` ≈ 41 chars) — same as Q126;
  Q127 didn't widen anything.

**Lives in:** `sim/report/src/viewport.rs` — five new helper
functions (`substrate_descriptor`, `substrate_biochem`,
`atmosphere_descriptor`, `sociality_label`, `comm_label`);
`planet_card()` rewritten with the new line themes;
`render()` writes the planet name as the divider label.

**Follow-ups (not blocking).**
- *Atmosphere → composition mapping accuracy.* `reducing` →
  `methane-rich` is a generalisation; reducing atmospheres
  can also be H2-rich or NH3-rich depending on substrate
  context. A substrate-aware
  `atmosphere_descriptor(substrate, atm)` would be more
  faithful but adds coupling.
- *More body morphology.* `Species` doesn't carry body-plan
  fields (vertebrate / cephalopod / radial-symmetry / etc.).
  A future Q could add an enum and surface it on the species
  line.
- *Configurable label set.* A `--no-friendly-labels` flag would
  let researchers see the raw `aqueous` / `reducing` /
  `cent-low` tokens. Pure preference; deferred.
