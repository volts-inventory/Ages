## Q40 — Pivot to physics engine (Path D)
**Status:** Decided
**Context:** Authored phenomenon catalog (even with Q39's grammar
proposal as Path C) had an authoring ceiling: civs could only discover
truths we wrote down. The user pushed for genuine novelty in what civs
can find. Path D: build a real (simplified) physics engine and let
phenomena emerge from law combinations on planet-specific conditions.
Civs sample real physics state and discover the laws that produce it
via the Q32 fitting pipeline.
**Decision:** Adopt Path D. Encode fundamental laws (mechanics +
gravity, fluid dynamics, heat conduction, simple chemistry, simplified
electromagnetism) on a spatial grid. Phenomena emerge; a recognition-
template library identifies them so the civ-layer can hypothesise
about them. Authoring shifts from phenomena to physics laws +
recognition templates. The catalog model (Q5/Q8/Q15/Q16/Q17) is
superseded; the discovery pipeline (Q6/Q32) is preserved with the
*data* now coming from physics rather than authored true_formulas.
M1 becomes the physics engine; M2 the recognition + civ scaffold; M3
the discovery + species + figures; M4 culture; M5 multi-civ; M6
post-run report + narration. Scope freed by Q39 makes this tractable.
**Lives in:** `PLANNING.md` Milestones, `docs/architecture.md` Physics
engine section, `docs/world.md` Recognized phenomena section.
