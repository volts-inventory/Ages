# Q158 — Belligerence-driven war: pressure + kinship instead of overlap

**Status:** Decided.

## What

Replace "any cell overlap → continuous war" with a per-pair
`belligerence` score that gates whether a 75-tick conflict check
inflicts losses. Overlap remains the *opportunity* (you can only
fight cells you both claim) but no longer is the sole *cause*. War
is now a stateful relationship bracketed by `WarDeclared` /
`PeaceConcluded` events, with hysteresis between declare (0.35) and
end (0.20) thresholds.

```
drive   = 0.45·pressure_a + 0.25·opportunity_b + 0.30·dominance      (∈ [0,1])
kinship = mean(1 − hier_gap, 1 − cosmo_gap, 1 − tech_gap)            (∈ [0,1])

belligerence(a→b) = drive × (1 − KINSHIP_DAMPENER · kinship)
```

Components:
- `pressure_a = pop_a / (capacity_a × 1.25)`, clamped to [0,1]. The
  1.25 headroom factor prevents late-game pinning at 1.0 once
  capacity-driven expansion (a2aa942) saturates every claimed cell.
- `opportunity_b = max(0, capacity_b − pop_b) / pop_a`, clamped to
  [0,1]. How much slack the defender has, normalised against the
  attacker's size.
- `dominance = strength(a) / (strength(a) + strength(b))` using the
  existing `conflict::strength()` function (pop × literacy × hier
  × tools).
- `hier_gap`, `cosmo_gap`, `tech_gap` are L1 differences over the
  hierarchical axis, the four non-hierarchical Cosmology axes
  (averaged), and `literacy_score(tick)` respectively, each clamped
  to [0,1].

Per-check decision rule (each pair with overlap, after contact):

| current state | belligerence  | action |
|---|---|---|
| at peace | ≥ 0.35 | emit `WarDeclared`, run `resolve()` |
| at peace | < 0.35 | border friction, no losses |
| at war   | ≥ 0.20 | run `resolve()` (skirmish) |
| at war   | < 0.20 | emit `PeaceConcluded { BelligerenceDropped }` |
| at war   | overlap empty | `PeaceConcluded { TerritoryResolved }` |
| at war   | resolve → loser_defeated | `PeaceConcluded { Defeated }` |

War also requires prior contact: `emitted_contacts.contains(pair)`
must be true before any war can fire. Civ collapse on either side
prunes the war_state entry with `PeaceConcluded { TerritoryResolved }`.

## Why

Pre-Q158 the conflict resolver was unconditional: any pair with
overlapping `claimed_cells` took losses every 75 ticks until the
overlap drained. That made any touching border a permanent
low-grade war and missed the actual drivers of historical
conflict — resource scarcity, opportunity, ideological distance.

The new model encodes the realist story material conditions
generate the *drive* to fight (pressure on your own land + slack on
the neighbour's + raw strength share), and cultural distance scales
that drive up or down. Two saturated civs that share cosmology and
tech don't fight. A strong civ next to a weak culturally-distinct
neighbor with under-utilised land does. Wars become events with
beginnings and ends, not background radiation.

## Calibration

Initial design called for thresholds 0.55 / 0.35 with
`KINSHIP_DAMPENER = 0.6`. Smoke run at seed 100 / 8000 ticks
produced zero wars: single-species runs have kinship near 1.0 the
whole run (every civ inherits `species.initial_cosmology` per
Q172), and the dampener of 0.6 took drive × 0.4, leaving even
strong-vs-weak imbalances under 0.55.

Tuned to:
- `KINSHIP_DAMPENER = 0.20`
- `WAR_DECLARE_THRESHOLD = 0.35`
- `WAR_END_THRESHOLD = 0.20`

This produces 1–2 wars per 8000-tick run on seeds 100 / 42 — wars
that brackets the existing per-cell skirmish records and end with
explicit `Defeated` reasons rather than fading into silence.

The trade is intentional: the model is now somewhat more peaceful
than the old "overlap = war" rule, but the wars that *do* fire are
meaningful (real drive imbalance) rather than constant low-grade
border attrition. Multi-species seeds (where civs from different
species would have lower kinship) will produce more wars.

## Wire events

Two new variants on `protocol::Event`:

```rust
WarDeclared {
    tick, aggressor_civ_id, defender_civ_id,
    belligerence_q32, drive_q32, kinship_q32,
}

PeaceConcluded {
    tick, civ_a, civ_b,         // normalised: civ_a < civ_b
    reason: PeaceReason,        // Defeated | BelligerenceDropped | TerritoryResolved
    duration_ticks,
}
```

`ConflictResolved` is unchanged it remains the per-skirmish record,
now bracketed by the new events. The post-run report's "Wars"
section reads `digest.wars` (built from the brackets) when present
and falls back to the old "campaign" grouper over
`ConflictResolved` events for older logs.

## What did *not* change

- `conflict::strength()` formula (pop × (1+literacy) × (1+hier/2) × tool_war_multiplier)
- `conflict::resolve()` per-cell skirmish mechanics, `loss_frac`,
  `CELL_FLIP_FLOOR = 25`, `CONFLICT_DEFEAT_FLOOR = 50`
- `CONFLICT_CHECK_TICKS = 75` cadence
- `is_peaceful_pair()` for knowledge-diffusion gating (separate
  concern: governs whether two civs share knowledge, not whether
  they fight)
- Contact event cadence (`CONTACT_CHECK_TICKS = 12`)

## Open follow-ups

- Multi-species runs should produce more wars naturally (lower
  kinship between species). Worth confirming once a multi-species
  seed exists.
- Could add cohesion as an extra drive component: civs in civil-
  war territory (Q180 cohesion < 0.3) might be more raid-prone.
- The 1.25 headroom factor in `pressure` is a rough guess; a
  closer match to historical "subsistence-crisis-drives-war"
  patterns might come from a different functional form (e.g.
  `(pop − 0.5·capacity) / capacity` so half-full reads as zero).
