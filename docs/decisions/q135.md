## Q135 — `RunMetadata` event + `labels.rs` extraction
**Status:** Decided
**Context:** The Q133/Q134 audit surfaced two HIGH findings:

1. **Substrate freeze/boil ranges duplicated in 4 sites.**
   `sim_world::MetabolicSubstrate::temperature_range` (habitable
   window — different concept), `sim_physics::chemistry::substrate_phase_thresholds`
   (canonical phase transition), `sim/report/src/viewport.rs::host_species_status`
   (Q121 hardcode), `narrate.py` (also hardcode). Three of those
   four are *the same data* — drift risk: change one, miss the
   others.
2. **Viewport ↔ narrator label tables duplicated.** `planet_type`,
   `friendly_badge`, `atmosphere_descriptor`, `sociality_label`,
   `comm_label`, `short_modality`, `short_manip` lived
   identically in `viewport.rs` (Rust) and `narrate.py`
   (Python). Same drift risk.

Q135 fixes both with one mechanism + a code-organization split.

**Decision:** Three coordinated changes.

1. **Extract `sim/report/src/labels.rs`.** Pull the label
   functions (`planet_type`, `friendly_badge`,
   `atmosphere_descriptor`, `substrate_biochem`,
   `sociality_label`, `comm_label`, `cog_tier`,
   `short_modality`, `short_manip`, and Q121's
   `host_species_status` — refactored to take freeze/boil as
   args instead of hardcoding them) out of `viewport.rs` into a
   dedicated module. Also expose `KNOWN_*` const arrays
   (substrates, atmospheres, badges, modalities, manipulations)
   that the metadata builder iterates.

   `viewport.rs` shrinks from 1118 lines to ~700; `labels.rs`
   carries the ~250-line label vocabulary.

2. **New `RunMetadata` event in `protocol`.** Single struct
   carrying:
   - substrate freeze/boil (Kelvin) per substrate enum key,
   - planet-type / biochem / atmosphere / friendly-badge /
     modality-short / manipulation-short label maps,
   - tier thresholds and tier-name vectors for cognition,
     sociality, communication-fidelity.

   Keys are the *internal* enum names ("aqueous", "Tactile",
   "frozen-out"), so consumers index directly without a second
   lookup table.

3. **`sim_core` builds + emits `RunMetadata` once per run, right
   after `RunStart`.** The builder (`labels::build_run_metadata`)
   takes a `Fn(&str) -> (f64, f64)` callback for the substrate
   ranges so `sim_core` injects them via
   `sim_physics::chemistry::substrate_phase_thresholds` — no new
   `sim_report → sim_physics` dep is needed (Q121's decision
   stands; the data flows the other direction now).

**Downstream consumer changes:**

- **`sim/report/src/viewport.rs`:** captures `RunMetadata` in
  state. `planet_card`'s host-species badge reads
  `freeze_k`/`boil_k` from the captured metadata (was
  hardcoded). Label functions are imported from `crate::labels`.
- **`narrate.py`:** new `Metadata` class loaded from the
  `run_metadata` event in the NDJSON. All substrate ranges +
  tier thresholds + planet-type/atmosphere/biochem/friendly-badge
  labels come from there. Viewport short-form labels do *not*
  pass through; the narrator keeps its own `PROSE_MOD` /
  `PROSE_MANIP` (longer noun-phrase forms like
  "secreted chemicals" instead of the viewport's `secrete`)
  because the audiences differ — viewport fits a 40-col grid,
  narrator wants English prose.
- **Pre-Q135 NDJSON logs** are still narrated correctly because
  `Metadata` falls back to last-known-good defaults when the
  event is missing.

**Why this respects the project goals.**
- *Single source of truth in Rust.* Labels live in
  `sim_report::labels`; substrate ranges live in
  `sim_physics::chemistry`. Both flow through one builder
  (`build_run_metadata`) into one wire-format event
  (`RunMetadata`) into one downstream consumer
  (`narrate.py::Metadata`).
- *No protocol drift.* New event variant is non-breaking;
  consumers ignore unknown variants. Existing event log
  format is unchanged for everything else.
- *No new direct dep edges.* `sim_core → sim_report` is new
  (sim_core needs `labels::build_run_metadata`), but
  `sim_report` doesn't gain any deps. The dependency graph
  stays a DAG.
- *Narrator audience preserved.* Viewport's `tactile` /
  `secrete` short forms stay terse for the grid card;
  narrator's `touch` / `secreted chemicals` long forms stay
  prose-friendly. Different forms for different surfaces is
  intentional, not duplication.

**Numerical care.**
- `to_f64_for_display()` on `Real` introduces minor float
  artifacts (273.15K serialises as 273.14999999990687) — they
  don't affect tier classification (the ladder thresholds
  span ~25K each).
- `RunMetadata` is `Default`-able so the empty-default `()` is
  populated by the builder; no panic if a substrate is missing
  from chemistry properties.

**Lives in:** `protocol/src/lib.rs` (`RunMetadata` struct +
event variant), `sim/report/src/labels.rs` (new module),
`sim/report/src/lib.rs` (mod declaration), `sim/report/src/viewport.rs`
(imports + state field + planet_card body), `sim/core/Cargo.toml`
(adds `sim-report` dep), `sim/core/src/lib.rs` (build + emit),
`sim/events/src/lib.rs` + `sim/report/src/highlights.rs` +
`sim/report/src/digest.rs` (exhaustive-match coverage),
`narrate.py` (`Metadata` class + `PROSE_*` tables, removes
hardcoded Q127/Q128 mappings).

**Follow-ups (not blocking).**
- *Schema versioning.* Adding `RunMetadata` is non-breaking;
  if a future Q changes a *required* field of an existing event,
  bump `SCHEMA_VERSION` and gate consumers.
- *Exhaustive label test.* A regression test could iterate
  `KNOWN_*` arrays and assert every key has a non-`unknown`
  label, catching new enum variants that haven't been added to
  the label functions.
- *`narrate.py`'s `PROSE_*` could become a metadata field too*
  if a future LLM narrator wants the long forms. For now the
  prose-vs-grid distinction stays Python-side.
