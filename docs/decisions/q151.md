## Q151 — README de-Q-ification
**Status:** Decided
**Context:** A new reader landing on the project's `README.md`
should not need to know about the internal Q-decision system to
understand the project. As the file accumulated features over
many decisions, every paragraph picked up a parenthetical
`(Q##)` tag — useful for contributors who can hop into
`docs/decisions/q##.md`, but jarring noise for first-time
visitors who just want to learn what the simulator does.

The user's request: scrub all Q-decision references from the
top-level README, replacing them with descriptive prose that
conveys the same fact. The contributor-facing decision tracker
at `docs/decisions/INDEX.md` stays where it is — the README is
the user-facing front door, the INDEX is the design-history
archive.

**Decision:** Walk every `Q##` reference in `README.md` and
rephrase the surrounding sentence so the factual content is
preserved without the parenthetical Q-number. The
`docs/decisions/INDEX.md` link stays in the "Where to read
deeper" section, but its description changes from
"Q-decision status pointer (Q1–Q137); restructured (Q136)…"
to "decision-record index for contributors who want to trace
the design history (organised into Latest / Open / sim-core /
viewport / tooling / Superseded sections)".

The Project-structure tree's inline comments lose their
`Q##` prefixes too: `Q133 — one-shot fresh-world launcher`
becomes `one-shot fresh-world launcher`,
`Q90-Q113 climate / atmosphere / tides / …` becomes
`climate / atmosphere / tides / …`,
`emits Q135 RunMetadata at run start` becomes
`emits run metadata at start`, and so on.

The literal string `Q32.32` (fixed-point arithmetic notation)
is preserved — that's a numeric format name, not a decision
reference.

**Why this respects the project goals.**
- *User-facing clarity.* A potential collaborator skimming
  the README on GitHub no longer hits 40+ orphan Q-references
  pointing at a system they've never seen documented.
- *Design history preserved.* `docs/decisions/INDEX.md` and
  every individual `q##.md` file remain untouched. Anyone
  digging into the codebase can still trace why a feature
  exists by cross-referencing commit messages (which retain
  their `Q###:` prefixes) against the decision-record index.
- *No code change.* This is purely a documentation pass.
  Determinism, simulation behaviour, and event schemas are
  unaffected.

**Lives in:**
- `README.md` — every Q-reference rewritten as descriptive
  prose; project-structure tree comments cleaned;
  decisions-index link description softened.
- `docs/decisions/q151.md` — this file.
- `docs/decisions/INDEX.md` — entry added under "Latest
  decisions" (newest first) and under
  "Decided — tooling & narrator".

**Follow-ups (not blocking).**
- *AGENTS.md, PLANNING.md, docs/architecture.md.* These are
  contributor-facing and may legitimately retain Q-references.
  No action unless a future user-facing pass requests it.
- *Per-crate `sim/<crate>/README.md` files.* Same as above —
  these target implementers, who benefit from the Q-anchors.
- *Periodic re-scrub.* New features added to the user-facing
  README should be written without Q-numbers from the start;
  this can be enforced socially during code review.
