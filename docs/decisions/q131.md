## Q131 — Revert Q130's light dividers
**Status:** Decided
**Context:** Q130 introduced two distinct divider styles to make
the map visually dominant: a heavy `------ map ------` for the
map section, and a lighter `· label ·` for non-map sections
(planet, key, species, log). On screenshots that worked, but the
user felt the inconsistency looked off — the heavy `------ label ------`
form had been a recognisable visual rhythm in Q124-Q129, and the
mixed style read as noise rather than hierarchy.

The map can still dominate via the *other* Q130 levers — the
fully-bounded `+----+` box around the grid, blank-line isolation,
and (in color mode) bold-vs-dim brightness — without sacrificing
the consistent divider style.

**Decision:** Revert *only* the divider-style split. All other
Q130 changes stay:

- Full map box (right `|` per row + bottom `+----+`).
- Blank line above and below the map.
- Default grid 32×20.
- ANSI dim wrap on non-map content (when `use_color`).
- Per-terrain 256-color codes + bold civ markers (when
  `use_color`).

Implementation: delete the `light_divider` helper, route every
section header through the existing `divider` helper. The map's
visual prominence now rests on box + isolation + brightness —
all structural / chromatic cues, no divider-style cue.

Every section now has the same `------ {label} ------` pattern;
the map still reads as the visual center thanks to its frame +
breathing room.

**Why this respects the project goals.**
- *Visual rhythm restored.* Five identical-pattern dividers
  read as a series of evenly-spaced section breaks. The eye
  scans them quickly and lands on the map (the only framed,
  isolated artifact between two of those breaks).
- *Map prominence preserved.* The Q130 box + blank lines do
  90% of the focus work. Removing the light divider was the
  one Q130 change the user disliked; the rest stays.
- *Determinism preserved.* Trivial diff: one helper deleted,
  call sites re-routed.

**Lives in:** `sim/report/src/viewport.rs` (`light_divider`
removed; all section headers now call `divider`; log-test
assertion reverted to `-- log --` substring).

**Follow-ups (not blocking).**
- *Dim divider in color mode.* The Q130 ANSI dim wrap already
  applies to non-map dividers when `use_color = true`, so in
  Termius the heavy `------ {label} ------` lines for non-map
  sections will render gray and recede. Same focus story,
  consistent style.
