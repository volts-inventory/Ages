## Q49 — Civ collapse triggers
**Status:** Decided (M4-minimal: food-security + knowledge-plateau
triggers ship now; cultural-lock and conquest deferred)
**Context:** Q47 commits to civ collapse as a normal mid-run event;
the specific triggers (population threshold, cultural-lock,
conquest, plateau + drain) need calibration. Cultural-lock requires
the M4 culture-axis machinery (Q9 / Q24 / Q25 / Q26) to land first;
conquest is M5.

**Decision (v2 — cultural-lock added now that Q9/Q24/Q25/Q26
have wired):**

Three M4 triggers ship; whichever streak crosses threshold first
wins:

1. **Food-security crisis.** A civ collapses when
   `food_security(cohort, carrying_capacity) ≤ 0.3` for 100
   consecutive civ-sim ticks. Food-security is already wired by
   Q22; the streak is per-civ state.
2. **Knowledge plateau.** A civ collapses when it goes 500 ticks
   without a confirmed-relation or refinement-confirmed event.
   Captures the "civ that has stopped learning" failure mode
   without requiring a full cultural-lock model.
3. **Cultural lock.** A civ collapses when `dogmatism > 0.85`
   sustained for 250 ticks AND the same period sees no
   refinement-confirmed events (high-dogma civs that won't
   update their relations are effectively frozen). Reads Q24
   cosmology directly.

Multiple compounding triggers accelerate the streak (e.g.
sustained low food during a plateau both fire) but the v1 model
treats them independently — whichever streak crosses its
threshold first wins. Tuning is under Q28's umbrella; current
defaults (0.3 / 100 / 500) are placeholders.

**On collapse:**
- `Civ.collapsed_tick = Some(tick)`.
- All active named figures' `retired_tick` set to the collapse
  tick (per Q47, surviving figures may join successor civs in M4
  follow-up; for now they all retire).
- The civ's cohort `civ_membership` becomes `None` — population
  becomes *stateless* and is the input pool Q48 reads to found
  the next civ.
- Knowledge graph dissolves with the civ; per-figure `confirmed`
  state stays on the retired figure for the post-run report.
- A `CivCollapsed{tick, civ_id, reason, final_population_q32,
  final_figure_count}` protocol event lands.

**Deferred to v2:**
- Cultural-lock collapse (5 generations cosmology-suppressed
  discovery): waits on Q9 + Q24 + Q25 + Q26.
- Conquest collapse: M5.
- Plateau + drain (net emigration): waits on Q57 settlement-
  level pop tracking + multi-civ migration.

**Q155 covers territory-shrink collapse.** A civ squeezed by a
successor (Q47/Q146) down to a single cell for ≥ 24 ticks
(2 sim-years) now auto-collapses with reason
`territory_too_small`. The remaining open Q49 sub-questions
(conquest collapse, plateau + drain) stay deferred per the v2
notes above; Q155 only addresses the tiny-territory case the user
flagged.
