## Q28 — Loss-formula coefficient tuning
**Status:** Decided (placeholder values; M3-tunable)
**Context:** Q7 settled the *structure* of `loss_per_generation =
species.t0_loss × persistence_tech_modifier × cultural_modifier`, but
the specific coefficients (per-tier multipliers, species-trait →
t0_loss mapping) are tuning work that needs reference runs.
**Decision:** Pin concrete starting values so M3 reference runs can
execute, retune against accumulated runs. Picking flat coefficients
would collapse the persistence-tier hypothesis we want to observe;
asymmetric multipliers preserve testable structure.

**Tier multipliers** (`persistence_tech_modifier`):
- T0 → T1: ×0.5
- T1 → T2: ×0.4
- T2 → T3: ×0.5
- T3 → T4: ×0.4

A T4 civ retains roughly 10× better than T0
(`0.5·0.4·0.5·0.4 = 0.04`). The T1/T3 alternation reflects writing
and printing being qualitatively bigger jumps than bronze→iron-style
material progression.

**Species `t0_loss` formula:**

```
t0_loss = clamp(
    0.50
    − 0.15 · cognition
    − 0.10 · sociality
    − 0.10 · ln(1 + lifespan_norm)
    − 0.15 · communication_fidelity,
    0.05, 0.70
)
```

Each trait is in `[0, 1]`; `lifespan_norm = lifespan / 70` (Earth-
human reference). Range expanded from the original 0.10–0.50 to
0.05–0.70 — degenerate-low species (rapid-turnover insectoid
collectives) and degenerate-high species (no oral tradition at all)
are valid niches; species-extinction handles the bottom end via
Q47's run-end semantics.

**Application cadence:** per civ-generation tick, per Q7.

**Cultural modifier:** Q7 leaves `cultural_modifier` open as a
multiplicative slot for Q23's cultural-influence hooks; pinned at
1.0 until Q23 lands in M4.

**Tuning umbrella.** This Q is the parking spot for M3 numerical
constants that need reference-run calibration. Currently includes:
- All values in this file (tier multipliers, `t0_loss` coefficients,
  the 0.05–0.70 clamp).
- Q35's refinement constants (`λ = 0.02`, switch margin 0.05,
  sustained-trigger 50 ticks, cooldown 100 ticks, probation 200
  ticks).
- Q34's per-form `base_per_form` tolerance multipliers and the
  `intelligence_factor` scale.

Status stays "Decided (placeholder)" rather than "Decided"
unconditionally — the values exist, runs can execute, but the next
real iteration on this Q is calibration, not redesign.

**Lives in:** `sim/civ` knowledge-loss module when implemented;
this file remains the placeholder ledger until M3 reference runs
retune.
