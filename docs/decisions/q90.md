## Q90 — Stellar-driven radiative balance
**Status:** Decided
**Context:** Pre-Q90 the only thing that *changed* a cell's
temperature after `init_planet` was heat conduction. The latitude-
driven gradient was seeded once at world-gen and then diffused
toward the planet-wide mean over thousands of ticks. After ~100
ticks every planet drifted toward an unphysical "lukewarm
everywhere" steady state, because conduction has no source term.
Real planets aren't isolated systems — they sit in a stellar
radiation field, lose heat as blackbodies, and reach equilibrium
where input matches output. Without a source/sink law the climate
gradient was a one-time prop, not an emergent feature of stellar
input × atmospheric optics × latitude geometry.

**Decision:** Add a `Radiation` law that sources cell
temperature from stellar irradiance each macro-step. For each
grid row, pre-compute a latitude-dependent equilibrium
temperature from Stefan-Boltzmann:

```text
absorbed[row] = S_star × cos_lat[row] × (1 - albedo) × 0.25
T_eq[row]     = (absorbed / σ)^(1/4) + greenhouse_offset
```

`cos_lat` is `1 - |row - half_h| / half_h` floored at 0.15 (so
pole cells aren't 0 K — real planets get atmospheric scattering
and longitude variance our 1D banding can't represent). The 0.25
factor distributes a sun's irradiance over the sphere (sub-solar
sees S; spherical-average sees S/4). σ = 5.67×10⁻⁸ W/(m²·K⁴);
greenhouse_offset is atmosphere-derived.

Per tick, each cell relaxes toward its row's `T_eq`:

```text
T[cell] += (T_eq[row] - T[cell]) × relaxation_rate × dt
```

`relaxation_rate = 0.02` (≈50-tick / 4-year e-folding timescale
on monthly cadence — order-of-magnitude match for surface-layer
thermal equilibration on real planets).

Atmosphere class drives albedo and greenhouse:
- `None` → albedo 10, greenhouse 0 K (airless rock)
- `Thin` → albedo 20, greenhouse 10 K
- `Oxidising` → albedo 30, greenhouse 35 K (Earth-like)
- `Reducing` → albedo 35, greenhouse 35 K
- `Hazy` → albedo 50, greenhouse 60 K (Venus-/Titan-ish trap)

These are the *only* per-atmosphere magic numbers in the law;
everything else (per-row T_eq, per-cell relaxation) derives from
stellar luminosity and grid geometry.

**Numerical care.** `S/σ ≈ 10⁹` overflows Q32.32 (±2.1×10⁹
ceiling collapses on intermediate `absorbed/σ`). Compute
`T_eq = exp((ln(absorbed) - ln(σ)) / 4)` instead — both
intermediates stay in range. `ln(σ) ≈ -16.685` is pre-computed
as `Real::from_ratio(-16_685, 1_000)` so the per-row build is
branch-free transcendentals.

**Orchestration.** Radiation runs once per macro-step *before*
heat diffusion: source → diffusion → reaction is the standard
operator-splitting order, so each pass reads the freshest
temperature. `integrate_civ_step` takes
`radiation: Option<&Radiation>` so existing call sites that
don't want a star (deterministic micro-tests) keep working;
`build_laws` now wires one through from `Planet::stellar_w_per_m2`
+ atmosphere class.

**Why this respects the project goals.**
- *Physics as close to real as possible:* the climate gradient
  is now an emergent steady state between a Stefan-Boltzmann
  source and conductive smoothing, not a seeded prop. Hotter
  stars warm planets; thicker atmospheres hold more heat;
  higher-albedo planets run colder — all from the same equation.
- *Least hardcoded values:* the only constants are σ (a real
  physical constant) and the relaxation rate (an arithmetic
  choice with explicit physical justification). The five
  atmosphere → (albedo, greenhouse) tuples are the *only*
  per-class hardcodes; everything else flows from `Planet`
  state and grid geometry.
- *Determinism:* pre-computed per-row table + branch-free
  per-cell relaxation. No state-dependent branching, no f64,
  no per-tick allocation.

**Lives in:** `sim/physics/src/radiation.rs` (the law),
`sim/physics/src/lib.rs` (re-export),
`sim/physics/src/orchestration.rs` (wired into the macro-step
loop before heat), `sim/core/src/lib.rs` (`build_laws` returns
a 5-tuple including `Radiation`; atmosphere → albedo/greenhouse
mapping).

**Follow-ups (not blocking).**
- *Seasonal swing.* Sub-solar latitude shifts with `axial_tilt
  × month-of-year`; the per-row table becomes per-(row, month).
  Lands when the sim wants summer/winter contrast as a
  recognition target.
- *Diurnal cycling.* Per-cell day/night insolation. Requires
  resolving sub-month time inside the macro-step; relevant for
  tidally-locked / fast-rotator recognition templates.
- *Latitude-dependent albedo.* Ice-albedo feedback (cold cells
  gain albedo, warm cells lose it) is the classic positive
  feedback for snowball / hothouse bistability. Lands when
  surface ice fraction becomes a per-cell field.
- *Cloud albedo.* Once Q92's hydrologic cycle puts vapour and
  cloud cover on the grid, cloud fraction modulates albedo
  per-cell instead of per-atmosphere-class.
