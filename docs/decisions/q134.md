## Q134 — `narrate.py` story script
**Status:** Decided
**Context:** A run produces an NDJSON event log at
`runs/{date}-{seed}.ndjson` (Q133). The viewport shows the live
arc, but after the run finishes the user has no easy way to
*review* what happened — they'd have to grep the NDJSON for
`civ_founded` / `civ_collapsed` / `catastrophe_fired` and stitch
the events together themselves.

A "narrate this run" tool fills the gap. The user explicitly
ruled out an LLM-driven narrator earlier ("No forget the llm");
templated prose from the event stream is enough.

**Decision:** Add `narrate.py` at the repo root with one arg —
the NDJSON path — and pure-stdlib Python 3 to render a templated
prose summary.

```bash
$ ./narrate.py runs/2026-05-05-1430-12345.ndjson
```

The script:

1. **Streams the NDJSON.** Skips bad lines, doesn't load the
   whole file into memory.
2. **Pulls foundational events.** `planet`, `planet_map`,
   `species`, `run_start`, `run_end`.
3. **Buckets high-impact events.** `civ_founded`,
   `civ_collapsed`, `civ_contact`, `tech_unlocked`,
   `knowledge_transmitted`, `conflict_resolved`,
   `catastrophe_fired`, `cosmology_shifted`,
   `relation_confirmed`. Ignores per-tick noise (`tick`,
   `recognition`, `civ_territory_changed`, refinements,
   measurements).
4. **Renders four sections** to stdout:
   - **Title bar** — `The Story of {planet}` + seed + sim-year
     count.
   - **THE WORLD** — planet type (water/ammonia/methane/lava),
     atmosphere, mean temperature, magnetosphere, moons, day
     length, year length, axial tilt, biosphere, friendly
     habitability badge.
   - **THE INHABITANTS** — species name, sociality, biochem,
     cognition topology + tier, primary modality, primary
     manipulation, lifespan, communication tier; up to 4
     secondary modalities.
   - **MAJOR ARCS** — events grouped by year, then
     month-of-year inline. Each event reads as a one-line
     prose statement (`Civ 1 unlocked thermal sensor`,
     `Civ 2 transmitted relation 47 to civ 3`, etc.).
   - **THE ENDING** — total civs founded vs collapsed vs
     surviving; aggregate counts of tools, relations,
     transmissions, conflicts, catastrophes.

**Why this respects the project goals.**
- *Out-of-process narrator.* Matches the architectural decision
  the user articulated earlier — narration sits outside the sim
  binary, reads NDJSON, never mutates state. Same model would
  extend cleanly to an LLM narrator later if the user changes
  their mind.
- *Same label vocabulary as the viewport.* Substrate / atmosphere
  / sociality / comm / topology / cognition tier mappings mirror
  Q127/Q128. A reader who watched the live viewport sees the
  same words in the post-run summary; no cognitive translation.
- *Stdlib-only.* No `pip install`, no venv, no Python deps. Works
  on any system with Python 3 already installed (which is
  basically every Linux + macOS today).
- *Templated, not stochastic.* Given the same NDJSON, the script
  produces byte-identical output. Useful for CI / regression /
  reproducible reports.

**Numerical care.**
- Q32.32 conversion (`q32(v) = v / 2^32`) matches `viewport.rs::q32_to_f64`
  and the binary's `Real::to_f64`. Mean temperature K → F via
  `(K - 273.15) * 1.8 + 32`.
- Tick → year via integer divide by `MONTHS_PER_YEAR = 12`,
  matching protocol's invariant.
- Bad / partial JSONL lines are silently skipped — a Ctrl-C'd
  run with a half-written final line still narrates the
  completed portion.
- `event_kind in {…}` whitelist is closed: unrecognized event
  kinds (added by future Qs) are silently dropped from the
  narrative until added explicitly. Fail-soft, not panic.

**Lives in:** `narrate.py` at the repo root, executable
(`chmod +x narrate.py`). 250-ish lines, single file, no project
imports.

**Follow-ups (not blocking).**
- *Era detection.* Currently the script lists events
  chronologically; a future version could detect "the founding
  age" / "the cataclysmic age" by clustering events into
  variance-driven eras.
- *Per-civ subplots.* When N > 1 civs exist, an "arcs by civ"
  section could trace each civ's individual story before the
  global timeline.
- *LLM-driven narrator.* If the user revisits the LLM idea
  later, this script's bucketed event list is a clean prompt
  prefix — feed `events_by_kind` into a model with "tell me
  this world's story" and you get richer prose.
- *Coordinate-aware locations.* Founding cells map to row/col
  via `planet_map.grid_width`; could be enriched with terrain
  context ("on the eastern coast", "in the central mountains")
  using `planet_map.elevation_q32` + `water_depth_q32`.
