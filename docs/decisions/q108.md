## Q108 — Diurnal cycling: day/night insolation modulation
**Status:** Decided
**Context:** Q97 added seasonal swing; Q107 added eccentricity-
driven year-cycle insolation. But within a year, real planets
also have *daily* insolation cycles — sub-solar longitude sweeps
across the planet at the planet's rotation rate, with day cells
seeing full insolation and night cells seeing zero. At our
1-day-per-macro-step cadence (Q67) Earth's diurnal cycle
averages out within a single tick (one full rotation per
macro-step). But for slow-rotator and tidally-locked planets
the day/night asymmetry persists across macro-steps and
becomes a *major* climatic driver — tidally-locked planets have
permanent day and night sides with extreme temperature
asymmetry.

The Q90 / Q97 docs both flagged: "Diurnal cycling. Per-cell
day/night insolation. Requires resolving sub-month time inside
the macro-step; relevant for tidally-locked / fast-rotator
recognition templates."

**Decision:** Add per-cell diurnal modulation to the existing
`Radiation::integrate` pass:

1. **`Radiation` gains two fields**: `day_length_macros: Real`
   (= `day_length_hours / 24`) and `diurnal_amplitude: Real`
   (∈ `[0, 1]`).
2. **Amplitude derivation**: linear ramp from 0 (Earth-like, 1
   macro-step day) to 1 (slow rotator with day length ≥ 10
   macro-steps). Tidally-locked planets (Q86 threshold ≥ 41
   macro-steps) get full amplitude.
3. **Sub-solar longitude sweep**: each macro-step,
   `sub_solar_q[step] = (step / day_length_macros) · width`,
   rounded to the nearest grid cell. For tidally-locked
   planets this advances ~0 cells/tick → permanent sub-solar
   longitude. For Earth-like it advances `width` cells/tick →
   full sweep, but amplitude=0 means the modulation is
   inactive.
4. **Per-cell modulation factor**:
   ```
   q_diff  = (axial.q - sub_solar_q) wrapped to [-width/2, +width/2]
   theta   = 2π · q_diff / width
   factor  = 1 + diurnal_amplitude · cos(theta)
   T_eq    = T_eq_base · factor
   ```
   Sub-solar cell (`q_diff = 0`): factor = `1 + amp` (max
   warming). Antipodal (`q_diff = ±width/2`): factor =
   `1 - amp` (cooling, → zero for tidally-locked). Quarter-
   circle terminators: factor = 1 (mean).

**Why this respects the project goals.**
- *Physics as close to real as possible:* tidally-locked
  exoplanets are a major worldgen archetype; Q108 gives them
  the permanent day/night asymmetry that defines their
  climate. Slow-rotator planets see meaningful diurnal swings
  too. Fast-rotator / Earth-like worlds correctly average out
  the daily cycle at our 1-day resolution.
- *Per-planet rotation drives the swing:* the same
  `day_length_hours` field that Q106 Coriolis reads now also
  drives Q108 diurnal modulation. Tidally-locked planets
  with `Q86::is_tidally_locked()` automatically get full
  diurnal amplitude → permanent day/night.
- *Determinism:* `state.macro_step()` and pre-computed
  amplitude/period are the only inputs; per-cell cos call
  uses Q101's deterministic Q32.32 trig.
- *No table re-derivation:* the existing `[row][season]`
  table is the *baseline*; diurnal modulation is a per-cell
  multiplier applied at integrate time. Cheap to add, cheap
  per-tick.

**Numerical care.**
- Amplitude clamped to `[0, 1]` so factor stays in `[0, 2]`
  for typical `T_eq_base` ≤ 350 K → modulated T_eq ≤ 700 K
  at sub-solar — well inside Q32.32.
- Sub-solar longitude rounded to nearest cell index
  deterministically via `round_ties_even`.
- Per-tick cost: one cos per cell when amplitude > 0.
  Earth-like (amplitude = 0) skips the cos entirely.

**Lives in:** `sim/physics/src/radiation.rs` (new
`day_length_macros` and `diurnal_amplitude` fields,
amplitude derivation in `for_planet`, per-cell modulation in
`integrate`, 2 new tests
`q108_tidally_locked_planet_has_diurnal_amplitude_one` and
`q108_diurnal_modulation_warms_day_side_cools_night_side`),
`sim/core/src/lib.rs` (`build_laws` threads
`planet.day_length_hours`).

**Follow-ups (not blocking).**
- *Sub-macro-step time resolution.* For full real diurnal
  modeling on Earth-like worlds at our cadence, the simulation
  would need to advance `(state, dt)` multiple times per
  macro-step at sub-day dt. Currently the orchestrator runs
  one Radiation call per macro-step; sub-macro-step diurnal
  refinement is a Q-decision in itself.
- *Diurnal recognition templates.* "Sunrise warmth", "midnight
  chill", "dawn-dusk transition" — recognisable now that
  per-cell day/night asymmetry is real. Tidally-locked
  planets' "permanent twilight zone" templates fire on cells
  near the terminator (already covered by
  `tidally_locked_terminator` id 32).
- *Diurnal coupling to wind / vapour.* Real day-side warmth
  drives convection that lifts air; cooling at night drives
  it back down. Q109 vertical atmosphere stack would let
  this emerge.
