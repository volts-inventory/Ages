## Q154 — Poisson-disc peak placement for distributed terrain
**Status:** Decided
**Context:** Q147 dropped the single-cone elevation pyramid in
favour of a 3–5 peak sum-and-cap (primary anchored at
`terrain_centre_q/r` with 80 % of `terrain_peak`, secondaries
split the remaining 20 %). The peak coordinates were drawn as a
single uniform `(q, r)` per secondary off the salted Q147 RNG:

```rust
for _ in 0..n_secondary {
    let q = peak_rng.gen_range(0..w);
    let r = peak_rng.gen_range(0..h);
    peaks.push((q, r, secondary_height));
}
```

Pure uniform `(q, r)` clusters. With 3–5 points on a 36×30 grid
the secondaries landed within a few cells of the primary or each
other on the majority of seeds — the renderer then summed cone
contributions that overlapped, producing one big mountain blob
instead of the separate ranges Q147 promised. User feedback
(seed 1 + 7 + 42 spot checks): "mountains are taking up a lot in
one place." Random clustering is the *expected* behaviour of
uniform sampling — small samples of independent draws have no
notion of repulsion.

**Decision:** Replace the single-draw secondary placement with
**minimum-distance rejection sampling** off the same Q147-salted
`ChaCha20Rng`. Each secondary candidate is re-rolled up to
`q154_max_attempts = 200` times until it sits at least `min_dist`
cells from every already-accepted peak. If 200 attempts fail
(rare on a 36×30 grid with ≤ 5 peaks), the loop falls back to
the last attempted candidate so the RNG sequence is still fully
consumed and seed-determinism survives.

**`min_dist` formula.** `max(3, max(w, h) / (num_peaks × 2))`.
On the 36×30 default grid:

| num_peaks | min_dist |
|-----------|----------|
| 3         | 6 cells  |
| 4         | 4 cells  |
| 5         | 3 cells  |

Half of an equal-partition gap (`max_dim / num_peaks`) is enough
to scatter without forcing a regular grid pattern — fewer peaks
get more breathing room, more peaks pack tighter. The 3-cell
floor keeps the formula safe on the small 8×6 grid used by
`init_planet_is_deterministic` (the raw `max_dim / (num_peaks·2)`
would collapse to 0 or 1, trivially accepting every candidate).

**Distance metric.** Manhattan / axial-sum `|dq| + |dr|`, matching
the `dq + dr` distance the Q147 piecewise-cone uses for falloff.
Keeps "spread out numerically" and "spread out visually" aligned —
two peaks that the cone treats as 4 cells apart will read 4 cells
apart on the rendered map. The Q147 follow-up note about hex-true
distance still applies; that's a Q147 question, not a Q154 one.

**Why rejection-sampling, not Bridson Poisson-disc.** Bridson's
algorithm with an active list + k-attempts per active point gives
better blue-noise distributions and runs in `O(n)`. For ≤ 5
points on a 36×30 grid the simpler rejection-sample approach
checks at most `5 × 200 × 4 = 4000` distance comparisons per
planet init — so cheap that the algorithmic savings don't matter.
The simpler code is also easier to keep deterministic: the
Bridson active-point shuffle order would need its own salted
draw and we'd risk sequence drift between code revisions.

**Why this respects the project goals.**
- *Determinism preserved.* Same RNG, same salt
  (`0xA17E_BEEF_C0DE_0147`), same gen_range sequence — only the
  rejection loop is new. The Q147 multi-peak invariant test
  still passes byte-identical on the seeds it covers (the
  rejection loop happens to accept every secondary on the first
  attempt for those seeds, so no extra RNG draws fire).
- *Q32.32 / Real arithmetic untouched.* The min-distance check
  is `i32` cell coordinates; no fixed-point math anywhere in the
  new path.
- *Q71 imprint regression test still passes.* Primary peak
  remains at `terrain_centre_q/r` with `0.8 × terrain_peak` — the
  Q71 test doesn't care where secondaries land, only that the
  primary anchor still rises above sea level.
- *Visible terrain variation.* Smoke runs across seeds 1, 42,
  100, 495 at the 36×30 default grid show distinct peak clusters
  separated by `~` / `≈` water bands or `▒` plains — the "one
  giant mountain" complaint is gone.

**Numerical care.**
- `num_peaks * 2` casts to `i32` via `try_from(...).expect(...)`;
  worst case is `(1 + 4) × 2 = 10`, well inside `i32`.
- `min_dist_raw = max_dim / (num_peaks × 2)` is integer division
  in `i32`, so we explicitly clamp to a 3-cell floor instead of
  relying on rounding. Negative values are impossible
  (`max_dim ≥ 1` for any non-degenerate grid).
- `q154_max_attempts: u32 = 200` is a literal, not a const, to
  keep the bound visible at the call site. 200 is enough margin
  that the fallback path is unreachable on every seed we've
  exercised in tests (1, 7, 42, 100, 495, 2024, 31337) — but the
  fallback is *correct*, not a panic, so a degenerate grid
  (e.g. `1 × 1` with > 1 peak) still produces a valid planet.

**Lives in:**
- `sim/world/src/lib.rs` —
  - `init_planet` body: `min_dist` computation + nested
    rejection-sampling loop replaces the pre-Q154 single-draw
    `(q, r)` for each secondary. The primary push is unchanged.
  - Test `q154_peaks_respect_minimum_distance` reconstructs the
    peak vector for seven seeds (1, 7, 42, 100, 495, 2024,
    31337) by re-running the same salted RNG against a 36×30
    grid and asserts every pairwise Manhattan distance is
    ≥ `min_dist`. Skips `GaseousShell` planets where
    `terrain_peak == Real::ZERO` and no peaks are placed.
- The Q147 test
  `q147_multi_peak_terrain_has_distinct_maxima_and_shallow_water`
  is unchanged and still passes.

**Follow-ups (not blocking).**
- *Bridson upgrade.* If/when peak counts climb above ~10 (e.g.
  per-cell volcanic-vent placement, or `terrain_complexity` as a
  per-substrate axis), swap rejection sampling for Bridson's
  algorithm. For the current ≤ 5 peaks the simpler code wins on
  readability.
- *Hex-true distance metric.* Same caveat as Q147 — `|dq| + |dr|`
  biases along the axes. True hex distance
  `(|dq| + |dr| + |dq + dr|) / 2` would be a single-line swap;
  defer until a perception bug fires.
- *Per-peak min_dist scaling by height.* Currently every peak
  uses the same `min_dist`. Tall primary cones could justify a
  larger exclusion radius than small foothills. Ornament; the
  uniform radius reads fine on smoke runs.
