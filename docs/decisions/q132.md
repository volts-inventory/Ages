## Q132 — Drop vestigial `--viewport-compact` flag
**Status:** Decided
**Context:** Q122 (Aug 2025) flipped the viewport's compact mode
from opt-in to default. To soften the transition I kept
`--viewport-compact` as an explicit-on flag — "back-compat for
scripts that already pass it." Q122 also added
`--no-viewport-compact` as the meaningful opt-out into the
standard (2-char-per-cell, hex-offset) layout.

Five Qs later, no scripts actually pass `--viewport-compact`.
The flag is a no-op that explicitly sets a value that's already
the default. The user noticed and asked why it existed.

**Decision:** Remove the `--viewport-compact` flag. Keep
`--no-viewport-compact` as the only way to influence compact
mode from the CLI.

**Why this respects the project goals.**
- *No-op flags are technical debt.* They confuse `--help`
  readers ("if it's already the default, why is the flag
  there?") and bloat the CLI surface for no functional gain.
- *Determinism preserved.* The `viewport_compact: bool` config
  default is unchanged (`true`); only the flag plumbing goes
  away.

**Lives in:** `ages/src/main.rs` — three lines deleted from the
arg-parsing match. Help text was already only documenting
`--no-viewport-compact`, so no help-text edit needed.

**Follow-ups (not blocking).**
- *Audit other no-op CLI flags.* Most viewport flags are
  meaningful opt-outs (`--no-color`, `--no-planet-card`,
  `--no-viewport-compact`); none jump out as similarly
  vestigial. If any get added in the future and end up no-ops,
  delete them with the next Q.
