## Q147 — Multi-peak terrain for varied elevation
**Status:** Decided
**Context:** Pre-Q147 `init_planet` sampled elevation as a single
linear cone falling away from `terrain_centre_q/r`:

```rust
let dist = Real::from_int(i64::from(dq + dr));
let drop = (dist * planet.terrain_peak) / max_dim_real;
let elev = planet.terrain_peak - drop;
```

Two consequences fell out of that single-cone shape, both visible
in the live viewport:

1. **One giant pyramid.** Every planet, regardless of seed,
   rendered the same `▲▲▲ / △△△ / ▒▒▒` concentric-ring pattern
   centred on `terrain_centre_q/r`. Seeds didn't *feel*
   different — the mountain was always one mountain, not a
   continent with multiple peaks.

2. **No shallow-water `~` band.** The per-cell drop was
   `terrain_peak / max_dim` ≈ 150–400 m/cell on a default 32-cell
   grid with Earth-class peaks. The renderer's water glyph rule:
   ```
   depth > 100 m → ≈ (deep)
   depth > 0     → ~ (shallow)
   depth = 0     → land
   ```
   With 150–400 m per-cell drops, elevation stepped from "above
   sea level" (`░` coast) straight past 100 m of depth into deep
   water (`≈`) in a single cell. The `~` shallow-water glyph
   never fired — coastlines went `░≈` everywhere.

**Decision:** Replace the single linear cone with a deterministic
3–5 peak composition built from a *piecewise* per-peak cone that
holds the per-cell elevation delta below the renderer's 100 m
shallow-water threshold along the coast band.

**Peaks.** A salted RNG (`planet.seed XOR 0xA17E_BEEF_C0DE_0147`)
draws a count `n_secondary ∈ [2, 4]`. The primary peak is anchored
at `terrain_centre_q/r` so existing callers — the Q71 imprint
regression test and `sim_civ::catastrophe` synthetic planets —
still find a mountain at the hint they planted. Secondary peaks
are uniformly drawn across the grid by the same salted RNG. Peak
heights:

- **Primary:** `0.8 · terrain_peak`. Stays above the renderer's
  `▲ peak` gate (`elev > 0.7 · terrain_peak`) so peak glyphs still
  fire on the primary summit.
- **Secondaries:** equal share of `0.2 · terrain_peak` across the
  2–4 secondaries. Smaller individual peaks read as foothills /
  isles around the primary.

**Piecewise cone.** Each peak's contribution to a cell is:

```text
let pivot_elev = sea_level + 200;     // height where slope flips
let steep_slope   = 200 m/cell;        // summit interior
let shallow_slope = 50  m/cell;        // coast band

if peak_height ≤ pivot_elev:
    contribution = max(0, peak_height − shallow_slope · dist)
else:
    let dist_steep = (peak_height − pivot_elev) / steep_slope
    if dist ≤ dist_steep:
        contribution = peak_height − steep_slope · dist
    else:
        let dist_shallow = dist − dist_steep
        contribution = max(0, pivot_elev − shallow_slope · dist_shallow)
```

The summit interior drops fast (200 m/cell) so a 7 000 m primary
on a 32×20 grid tapers through to the shallow band within a
handful of cells. From the pivot height (200 m above sea level)
downward, the gentle 50 m/cell slope guarantees:

- **Coastal `░`** at the cell where elev crosses sea level.
- **Shallow `~`** in the next 1–2 cells (depth = 50 m, then 100 m).
- **Deep `≈`** beyond.

**Smooth-max sum, capped at `terrain_peak`.** Cell elevation =
sum of per-peak contributions, capped at `terrain_peak`. Summing
(rather than `max()`-ing) lets each secondary peak rise above the
primary's flank as a true local maximum — pure `max()` would
smear small secondaries into the primary's wide cone with no
visible bump, defeating the multi-peak goal. The cap holds the
renderer's relative-elevation glyph ramp (`▲ > 0.7 · peak`,
`△ > 0.4 · peak`) honest at the summit.

**Why this respects the project goals.**
- *Visible terrain variation.* Smoke runs across seeds 1–9 show
  the difference: Lyra-a (seed 1) gets a primary chain across
  the diagonal with a wedge of shallow + coast at one corner;
  seed 7 renders an archipelago of `▲` peaks each ringed by
  `~` shallow band and `≈` open ocean; seed 8 splits into two
  separate islands. Pre-Q147 every map looked the same.
- *Coastal `~` band reliable.* The Q147 regression test asserts
  a temperate Aqueous Rocky seed has at least one cell with
  `0 < depth ≤ 100 m`. Pre-Q147 that band was structurally
  empty.
- *Determinism preserved.* Same `planet.seed` → same peak
  centres + heights → byte-identical NDJSON. The salted RNG is
  drawn from a separate stream than the planet sampler's RNG so
  Q147's peak placement doesn't perturb gravity, atmosphere,
  composition, or any other Q-decided field.
- *Backwards compatibility.* `terrain_centre_q/r` retained as
  the primary peak's anchor. The Q71 imprint regression test
  still finds at least one land cell with the expected charge
  baseline (`primary_height = 0.8 · terrain_peak` ≥ 1 600 m
  comfortably above the synthetic planet's `sea_level = 1 000 m`).

**Numerical care.**
- All arithmetic stays in `Real` (Q32.32). Slopes (`50`, `200`)
  and the buffer (`200`) are `Real::from_int` constants. Per-cell
  contribution is at most `peak_height` ≈ `0.8 · terrain_peak`
  ≤ `0.8 · 15 000` = `12 000` m; sum across 5 peaks ≤ `60 000` m,
  well within the Q32.32 range (±2.1 × 10⁹). The
  `if drop ≥ peak_height { continue; }` early-exit and the
  `if shallow_drop ≥ pivot_elev { 0 }` clamp prevent the
  subtractions from underflowing into the negative-elevation
  regime.
- The salt constant `0xA17E_BEEF_C0DE_0147` is distinct from
  the Q119 planet-name pool (raw `seed`, no salt) and the Q120
  species-name pool (`0xFEED_FACE_BAAD_F00D`) so the three
  streams stay independent.

**Lives in:**
- `sim/world/src/lib.rs` —
  - `init_planet` body: peak vec construction (salted RNG draws
    centres + count) + per-cell piecewise sum-and-cap loop.
  - Test `q147_multi_peak_terrain_has_distinct_maxima_and_shallow_water`
    asserts both invariants on a temperate Aqueous Rocky seed.

**Follow-ups (not blocking).**
- *Lower the primary share for ocean-world seeds.* OceanWorld
  composition gets `terrain_peak ∈ [sea_level + 500, sea_level + 2 500]`
  — a primary at 80 % can still tower above the modest sea_level.
  An ocean-world override that clamps the primary to ~50 % and
  weighs secondaries equally would make ocean worlds feel like
  archipelagos instead of one big island.
- *Hex-true distance.* Currently uses the axial-sum
  `|dq| + |dr|` proxy (cheap, deterministic) — true hex distance
  is `(|dq| + |dr| + |dq + dr|) / 2`. The proxy biases the cone
  along the axes; the renderer's six-neighbour glyph picker
  doesn't notice but a careful eye might. Defer until someone
  files a perception bug.
- *Per-peak slope variation.* All peaks share the same steep /
  shallow slopes today. Sampling a per-peak slope multiplier in
  e.g. `[0.7, 1.3]` would give the planet some "old worn ranges
  vs sharp young uplift" character. Ornament; defer.
