## Q126 — Centered caption, expanded labels, biology line
**Status:** Decided
**Context:** Q125 widened the viewport to 40 cols and centered
the map within that width by computing one body-wide left pad
from the widest line. Three follow-ups landed in user feedback:

1. *Caption was visibly off-centre.* The caption
   (`Y{n} M{n} · {civ} civ · {F}F/{C}C` ≈ 22 cols) inherited the
   same pad as the grid+header (≈ 27 cols), which centered the
   wider lines but left the caption sitting 3 cols left of true
   centre. Reads as "everything in the box is centered except
   the one thing labelled the section title".
2. *Species line had room to grow.* Q123 + Q125 settled on
   `tactile/tentacle`-style abbreviations (`touch/tent`) at a
   time when 32-col width was the budget. The Q125 widening to
   40 cols left the species line with 4-8 chars of trailing
   whitespace on most seeds.
3. *No species biology.* The card surfaced cognition (topology +
   tier), modality, and manipulation, but `SpeciesDerived`
   carries `lifespan_years_q32`, `sociality_q32`, and
   `communication_fidelity_q32` (Q47) that never made it onto
   the viewport. A reader couldn't see at a glance whether the
   inhabitants live 5 years or 200.

**Decision:** Three coordinated changes.

1. **Per-line center the map body.** Replace the single
   body-wide pad with a per-line pad computed from each line's
   own width. The caption gets pad ≈ 9, headers + grid rows get
   pad ≈ 6 — different offsets, but each line sits at its own
   midpoint. Header rows + grid rows still column-align with
   each other because they're equal width by construction.

2. **Expand modality + manipulation short labels.** With
   `VIEWPORT_WIDTH = 40` the species line can carry 13-14 chars
   of `mod/manip` after `{name} · {topo}-{tier} · `. Q126 walks
   the `short_modality` / `short_manip` tables and expands every
   label that fits and reads more clearly:

   | Variant | Q120-Q125 | Q126 |
   |---|---|---|
   | Tactile | touch | tactile |
   | VisualLight | vis | vision |
   | Seismic | seism | seismic |
   | Gestural | gest | gesture |
   | Postural | post | posture |
   | LimbGrasp | limb | limbs |
   | Tentacle | tent | tentacle |
   | Mandible | mand | mandible |
   | ToolExtension | tool | tools |
   | WebConstruct | web | webs |
   | ChemicalSecretion | chem | secrete |

   Compound variants (`ac-air`, `vis-pol`, `biolum`) and
   physically short ones (`beak`, `jet`, `zap`, `mag`, `ir`)
   stay as-is — expanding `MagneticSense` to `magnetic` would
   collide with the planet card's `mag` (magnetosphere) label,
   and `ac-air` is more readable than `acoustic-air`.

3. **Fifth card line: species biology.** A new line under the
   species line surfaces `lifespan_years` + `sociality` +
   `communication_fidelity`:
   ```
   {lifespan_years}y life · {soc_tier}-soc · {comm_tier}-comm
   ```
   Tiers bucket the 0..1 traits the same way Q120 buckets cog:
   `< 0.34` → low, `< 0.67` → med, else high. Lifespan prints
   as integer years (Q47 samples 5..=200). Sample card after
   Q126:
   ```
   ---------------- planet ----------------
   Lyra-a · boiling-off
   aqueous · reducing · 2 moons
   strong mag · 241F · 129h · 17° · 11mo
   Cyranites · cent-med · tactile/tentacle
   44y life · low-soc · low-comm
   ----------------- map ------------------
            Y1 M11 · 1 civ · 1F/0C       ← centered (was off-centre)
            ...
   ```

**Why this respects the project goals.**
- *Q47-honest:* the biology line surfaces three Q47 trait
  scalars that were already emitted in `SpeciesDerived` but
  never displayed. No new state, no new sampling — just
  reading what the species derivation already produced.
- *Information density without overload:* the species + biology
  lines together cover topology, cognition tier, modality,
  manipulation, lifespan, sociality, comm fidelity in two
  rows ≤ 40 cols. A reader can answer "what kind of being
  am I looking at?" in two glances.
- *Determinism preserved:* per-line centering is a pure
  function of each line's char count; tier bucketing is the
  same threshold scheme Q120 used for cognition; lifespan is
  integer-truncated Q32.32.
- *Visual harmony:* every line in the box is now individually
  centered to the box width. No more "the caption is off and
  the grid isn't" cognitive friction.

**Numerical care.**
- `lifespan_years` is Q32.32 with a Q47 sample range of
  `[5, 200]`. Cast to `i64` then displayed; truncation is
  desired (we want `75y` not `75.3y`).
- Per-line centering uses `chars().count()` (not `len()`)
  because the body includes terrain glyphs (`▲ △ ▒ ░ ~ ≈`) and
  bullet `·` that are multi-byte UTF-8 but display 1 col wide.
- `saturating_sub` on the pad guarantees no underflow when a
  line happens to be wider than `VIEWPORT_WIDTH` (e.g.,
  `--grid-width 50` users); the pad falls to 0 and the line
  prints flush-left.

**Lives in:** `sim/report/src/viewport.rs` —
`short_modality` / `short_manip` updated in place;
`planet_card()` writes the 5th biology line; `render()` map
loop switches to per-line centering.

**Follow-ups (not blocking).**
- *Tier names beyond low/med/high.* Sociality has natural
  semantic tiers (solitary / social / eusocial) but their
  string lengths are 8/6/8 — would push the biology line
  past 40 on most seeds. Using `low/med/high` here matches
  the cog-tier convention and keeps the line compact.
- *Body morphology.* "Biology" in the broader sense (carbon
  vs. silicon, exoskeleton, body plan) isn't currently in
  `Species`. The substrate (line 2 of the card) is the closest
  proxy. A future Q could add a `body_plan` enum to
  `SpeciesDerived` and surface it here.
- *Configurable `VIEWPORT_WIDTH`.* Same Q125 follow-up — a
  CLI flag for the box width waits for a user signal.
