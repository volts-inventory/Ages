## Q112 — Per-substrate cell thermal mass + specific heat
**Status:** Decided
**Context:** Q85 added per-substrate `L_fusion` /
`L_vaporisation` / `R_specific` so the *energy released per phase
change* was substrate-correct. But the conversion from "energy
released" to "K of cell-temperature change" still used a hardcoded
`C_P_WATER × CELL_THERMAL_MASS_KG` (4186 × 539). That meant a
methane-cell condensation released the right amount of energy
but spread it across water's specific heat and water's column
mass — wrong by a factor of ~1.5× for methane and ~6× for
silicate.

The Q85 doc explicitly called this out: *"the c_p denominator
stays at water's specific heat as a proxy — per-substrate c_p is
a future Q85 refinement."*

**Decision:** Add `c_p: i64` and `cell_thermal_mass_kg: i64` to
`SubstrateProperties`. Calibrated from real values:

| Substrate | c_p (J/kg·K) | thermal_mass (kg/cell) | Real-world basis |
|---|---|---|---|
| Aqueous | 4186 | 539 | water at STP (unchanged) |
| Hydrocarbon | 3500 | 240 | liquid methane (~445 kg/m³) |
| Ammoniacal | 4700 | 370 | liquid ammonia (~682 kg/m³) |
| Silicate | 1300 | 1450 | molten silicate (~2700 kg/m³) |

The thermal mass scales with substrate density relative to
water (water = 1000 kg/m³, mass = 539). Higher-density substrates
have more mass per cell and so warm/cool more slowly per unit
energy.

Wire `props.c_p` and `props.cell_thermal_mass_kg` into:
1. `Chemistry::for_planet`'s latent-heat denominator
   (replaces `C_P_WATER × CELL_THERMAL_MASS_KG` with
   `props.c_p × props.cell_thermal_mass_kg`).
2. `Hydrology::for_substrate`'s `lh_condense` derivation
   (same change).

The global `C_P_WATER` and `CELL_THERMAL_MASS_KG` constants
remain as the aqueous-default values referenced by
`substrate_properties("aqueous")`, but downstream code now
reads them through `props` instead of importing the globals
directly.

**Why this respects the project goals.**
- *Least hardcoded values:* the hardcoded "every substrate gets
  water's thermal physics" assumption is gone. Each substrate's
  full thermodynamic chain — phase-change energy, specific
  heat, cell mass — is now derived from one substrate tag.
- *Physics as close to real as possible:* methane phase
  changes on Titan now produce the *correctly-scaled* surface-
  temperature swing for liquid methane's actual heat capacity,
  not water's. Silicate worlds (e.g. lava-lake archetypes)
  see molten-rock thermal inertia rather than ocean-style
  smoothing.
- *Determinism:* per-cell behaviour unchanged in shape; only
  the per-substrate calibration constants differ. Same seed,
  same substrate → same numerics.
- *Backwards-compat:* aqueous worlds get bit-identical
  results because `props.c_p == C_P_WATER` and
  `props.cell_thermal_mass_kg == CELL_THERMAL_MASS_KG` when
  substrate_tag is `aqueous` or unrecognised. All existing
  tests pass without modification.

**Numerical care.**
- `props.c_p × props.cell_thermal_mass_kg`:
  - aqueous: 4186 × 539 = 2,256,254
  - hydrocarbon: 3500 × 240 = 840,000
  - ammoniacal: 4700 × 370 = 1,739,000
  - silicate: 1300 × 1450 = 1,885,000
  All in `i64` range, all → `Real::from_int` safely.
- `lh_condense = L_vap / denom` ratios:
  - aqueous: 2,257,000 / 2,256,254 ≈ 1.0 K/kg
  - hydrocarbon: 510,000 / 840,000 ≈ 0.61 K/kg
  - ammoniacal: 1,371,000 / 1,739,000 ≈ 0.79 K/kg
  - silicate: 12,000,000 / 1,885,000 ≈ 6.4 K/kg (silicate
    L_vap is enormous because vaporising rock takes huge
    energy)

**Lives in:** `sim/physics/src/chemistry.rs` (new `c_p` /
`cell_thermal_mass_kg` fields on `SubstrateProperties` + new
constants `C_P_AMMONIA` / `C_P_METHANE` / `C_P_SILICATE` +
per-substrate values + Chemistry::for_planet uses
`props.c_p × props.cell_thermal_mass_kg`),
`sim/physics/src/hydrology.rs` (Hydrology::for_substrate uses
the same).

**Follow-ups (not blocking).**
- *Mixed-substance cells.* Real cells contain multiple
  substances (water + ice + vapour + atmosphere); each has its
  own `c_p`. A weighted-average specific heat per cell would
  be more correct than picking one substrate. Lands when cells
  start tracking multiple solvents simultaneously.
- *Pressure-dependent c_p.* Real specific heats vary slightly
  with pressure (especially gases). Currently we use a single
  reference value per substrate. Refinement.
- *Q113 vacuum guard.* On `Atmosphere::None`, the cell's
  thermal mass is just the surface (no atmospheric contribution).
  Q113 short-circuits affected laws but the thermal-mass
  numbers above assume an atmosphere is present.
