## Q145 — More log events + `run.sh` seed-arg
**Status:** Decided
**Context:** Two small additions the user asked for after the
flicker work landed.

1. The viewport's `-- log --` section surfaces six event kinds
   (Q114): `CivFounded`, `CivCollapsed`, `CatastropheFired`,
   `KnowledgeTransmitted`, `ConflictResolved`, `TechUnlocked`.
   Two narrative-rich events that fire rarely — `CivContact`
   (first encounter between two civs) and `CosmologyShifted`
   (a civ's worldview changes per Q24) — were filtered out.
   Adding them gives the log richer "what just happened on this
   world" beats without flooding (both fire only a handful of
   times per run).
2. `run.sh` always generated a random seed via `$RANDOM$RANDOM`.
   Replaying a memorable world required calling `ages` directly
   with `--seed N`, bypassing the launcher. An optional
   positional arg lets users pass the seed and reuse the
   pacing / NDJSON-archive defaults: `./run.sh 12345` to replay,
   `./run.sh` (no arg) for fresh.

**Decision:** Two small edits.

1. Add two arms to `viewport.rs::log_message`:
   ```rust
   Event::CivContact(c) => Some(format!("civ {} met civ {}", c.civ_a, c.civ_b)),
   Event::CosmologyShifted(s) => Some(format!("civ {} cosmology shift", s.civ_id)),
   ```
2. Add an optional seed arg to `run.sh`:
   ```sh
   if [ -n "${1:-}" ]; then
     SEED="$1"
   else
     SEED=$RANDOM$RANDOM
   fi
   ```

   Usage:
   - `./run.sh` — fresh random seed.
   - `./run.sh 12345` — replay seed 12345.
   - The seed is still echoed before launch and embedded in the
     `runs/{date}-{seed}.ndjson` archive filename, so replay
     paths remain visible.

**Why this respects the project goals.**
- *Narrative-friendly log.* Q47 species-as-persistent-unit
  worlds with multiple civs (succession, contact, conflict)
  now have two more "meeting / paradigm-shift" events visible
  on the live log. `narrate.py` was already capturing both
  for the post-run summary; the live log now matches.
- *Reproducibility-first launcher.* The scripted UX now
  covers both ends of the workflow — fresh runs and replays —
  without forcing the user to remember the full `cargo`/`ages`
  invocation.
- *No protocol or sim-state change.* `CivContact` and
  `CosmologyShifted` events were already in the NDJSON; only
  the viewport's display filter changed.

**Numerical care.**
- `CivContact` fires once per (civ_a, civ_b) ordered pair —
  guarded upstream in `sim_core` so a re-encounter doesn't
  re-emit. The log won't flood from a meeting.
- `CosmologyShifted` fires when a civ's cosmology axes drift
  past Q24's threshold, typically a few times per civ
  lifetime. Manageable log presence.
- `${1:-}` is bash parameter-expansion for "first arg, default
  to empty"; works under `set -u` (which `run.sh` enables via
  `set -euo pipefail`).

**Lives in:**
- `sim/report/src/viewport.rs::log_message` — two new match arms.
- `run.sh` — optional `$1` seed selection.

**Follow-ups (not blocking).**
- *Filter very-frequent cosmology shifts.* If a civ shifts
  its cosmology many times a year, the log could fill. We could
  rate-limit (one per year per civ). Defer until observed.
- *More event kinds.* `FigureBorn` for charismatic figures,
  `KnowledgeDiffused` for cross-cell spread — both noisy by
  default but might be worth surfacing the top-N. Niche.
- *`run.sh --resume runs/foo.ndjson`.* Replaying directly from
  an archive path (extracting the seed automatically) would be
  the most convenient form. Defer.
