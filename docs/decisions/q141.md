## Q141 — Revert Q140's `run.sh` grid auto-shrink
**Status:** Decided
**Context:** Q140 added `tput lines`-driven grid-height
shrinking to `run.sh` so the viewport always fits the visible
terminal, sized down to as little as 32×8 on a phone with the
keyboard up. The user pushed back: they prefer the binary's
full 32×20 grid even when it means the planet section scrolls
off the top.

The trade-off they're choosing: bigger map, sometimes-clipped
top. Reasonable — the map *is* the focus, and the planet card
is static-ish (substrate, climate, orbital, species name, etc.
don't change frame-to-frame), so missing it is less costly
than losing four rows of grid resolution every frame.

**Decision:** Drop the `tput lines` math + `--grid-height`
flag from `run.sh`. The script falls back to the binary's
32×20 default for every run. The atomic frame-paint refactor
from Q140 stays — that was a real flicker fix unrelated to
grid sizing.

If the planet section scrolls off the top on a phone, the
workarounds the user accepts:
- Hide the on-screen keyboard while watching.
- Rotate the device to landscape.
- Zoom the terminal font smaller.
- Or for power users, pass `--grid-height N` to `ages`
  directly.

**Why this respects the project goals.**
- *User preference is canon.* They asked for the bigger grid;
  the trade-off is theirs to make.
- *Q140's atomic-paint stays.* The flicker fix is preserved;
  only the auto-shrink behaviour reverts.
- *Determinism preserved.* `run.sh` no longer varies grid size
  by terminal — every random seed run produces the same grid
  shape regardless of where it's launched.

**Lives in:** `run.sh` only.

**Follow-ups (not blocking).**
- Same as Q140's open ones: a `--grid-height auto` flag on the
  binary itself would let opt-in users still get the auto-fit
  behaviour without paying for it on the script default. Defer
  until someone asks.
