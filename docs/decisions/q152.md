## Q152 — Deterministic civ names
**Status:** Decided
**Context:** Civilizations have been identified only by `civ_id`
since the project's inception — `civ 1`, `civ 2`, … in the
viewport, the post-run report, the narrator, and every
downstream consumer of the NDJSON event log. That works for the
sim's internal bookkeeping, but for the reader it leaves a gap:
the planet has a name (Q119), the species has a name (Q120),
the named figures who do the science have names (Q3), but the
*political entities* on which the entire civ-arc story hangs
read as anonymous integers.

The reader's mental model of the run is "Sumer rose, Akkad
followed, Babylon ruled" — the kingdom-and-empire texture of
real-world history. Without names, the same arc renders as
"civ 1 collapsed, civ 2 took over, civ 3 ruled" — accurate but
flavourless, and structurally indistinguishable from any other
run on any other seed.

Q119 (planet names) and Q120 (species names) demonstrated the
pattern: a curated stem pool, a small ending pool, deterministic
hash from the seed via XOR with a magic constant distinct from
any other naming pool. Q152 applies the same recipe to civs —
keyed on `(seed, civ_id)` rather than `seed` alone, since a
single seed produces many civs over a run.

**Decision:** Three pieces:

1. **`civ_name_from_seed(seed, civ_id) -> String`** in
   `sim/civ/src/lib.rs`. 64 kingdom-feeling stems
   (`Eldor`, `Karn`, `Verum`, `Volkar`, `Aurel`, `Korath`,
   `Thalas`, `Sumer`, `Akkad`, `Babyl`, `Persa`, `Cathar`, …)
   × 6 endings (`ia`, `ath`, `is`, `on`, `an`, `ic`) → 384
   distinct names before collisions. The hash is
   `seed XOR civ_id XOR 0xC1A5_5C0D_E152_BEEF`; modulo against
   stem and ending counts gives the indices. The magic constant
   is distinct from Q119 (no salt, raw seed) and Q120
   (`0xFEED_FACE_BAAD_F00D`) so all three streams pick
   independently — a `Vela-c` planet hosting `Pyrrites` doesn't
   mechanically imply that civ 1 will always be `Eldoria`.

2. **`Civ::name: String`** field added at construction.
   Production callers (the inaugural civ at run start, the
   refound and breakaway paths in `sim/core`) assign the result
   of `civ_name_from_seed(cfg.seed, civ_id)` immediately after
   the constructor returns. Test-only callers (`Civ::new`,
   `Civ::with_intelligence`) leave the field empty — those don't
   emit events, so no protocol consumer ever sees an empty name
   from a real run.

3. **`protocol::CivFounded::name`** new field, marked
   `#[serde(default)]`. Existing NDJSON files written before
   Q152 don't carry the field; the default makes deserialization
   resilient — those events come through with `name = ""` and
   downstream consumers fall back to the legacy `"civ {id}"`
   render path.

The narrator (`narrate.py`) uses the new field when present:
`{name} took root{location}, founded by N figures across M cells`,
falling back to `Civilization {civ_id} took root…` when
`name` is missing or empty.

**Why this respects the project goals.**
- *Reader-friendly identification.* "Eldoria collapsed; Karnath
  rose from its ashes" reads as a kingdom-arc story; "civ 1
  collapsed; civ 2 rose from its ashes" reads as bookkeeping.
  No physics changes, just narrative texture.
- *Pure-function naming.* Same `(seed, civ_id)` → same name,
  byte-for-byte. No mid-run RNG draws; no entanglement with
  physics state. Replays produce identical names.
- *Independent name pool.* The XOR magic
  `0xC1A5_5C0D_E152_BEEF` is distinct from Q119's pool (no
  salt) and Q120's `0xFEED_FACE_BAAD_F00D`, so the three
  streams pick independently. Statistical correlations between
  planet name, species name, and inaugural civ name don't show
  up in a real-world reader's pattern matching.
- *Backward compatible.* `#[serde(default)]` on the protocol
  field means archived NDJSON files from before this Q
  deserialize cleanly. The narrator's fallback path keeps the
  old story renderable.

**Numerical care.** The hash uses raw `u64` XOR + modulo on
the result. Modulo on a `u64` is bit-deterministic across
architectures (no float arithmetic), and the indices land in
`0..STEMS.len()` and `0..ENDINGS.len()` by construction so the
`as usize` cast can never alias or overflow. Pool sizes (64 and
6) are powers-of-two-ish (64 is 2^6, 6 isn't) — the modulo isn't
perfectly uniform on `u64`, but a 0.4% bias on a 64-stem pool
produces no humanly-visible structural skew.

**Lives in:**
- `sim/civ/src/lib.rs` — new `civ_name_from_seed` function;
  new `Civ::name` field on the struct; three tests
  (determinism, low collision rate over civ_ids 1..=16,
  both-inputs-mix sanity).
- `protocol/src/lib.rs` — new `CivFounded::name` field with
  `#[serde(default)]`.
- `sim/core/src/lib.rs` — three founding sites (inaugural,
  refound, breakaway) now sample the name and pass it
  through the `CivFounded` event payload.
- `narrate.py` — reads the optional `name` field; falls back
  to `"Civilization {civ_id}"` when missing.
- `sim/report/src/digest.rs`, `sim/report/src/viewport.rs` —
  test-only struct literals updated to include `name:
  String::new()` (mechanical compile-fix; the viewport
  display itself is untouched and stays for Q150 / Agent B).

**Follow-ups (not blocking).**
- *Viewport sidebar will use the name in Q150.* Agent B's
  follow-up PR threads the civ name through the viewport's
  per-civ sidebar (`Eldoria · 14k · 6 cells` instead of `civ
  1 · 14k · 6 cells`).
- *Post-run report & summary.* Civ chapters in the report
  should pick up the name in their headers and prose. Small
  `sim/report` change once Q150 lands.
- *Per-substrate or per-axis name pools.* All civs draw from
  the same 64-stem pool. Substrate-themed names (`Tholos`-
  feeling for silicate worlds, `Methane`-feeling for hydrocarbon)
  would add narrative texture, similar to how Q120 listed
  per-substrate species pools as a follow-up.
- *Avoid neighbour-id collisions deterministically.* Q152 uses
  `civ_id` as input, so two runs with the same seed produce the
  same civ-1 / civ-2 names. Within a single run, collisions
  across `civ_id`s 1..=16 are tested to be ≤ 4 per seed; a
  follow-up could explicitly skip a stem already assigned to a
  living civ to push collisions closer to zero.
