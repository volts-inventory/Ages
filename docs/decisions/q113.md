## Q113 — Vacuum guards on `Wind` / `Hydrology` / `Coriolis`
**Status:** Decided
**Context:** The `Atmosphere::None` planet class represents a
vacuum world (airless rock — Mercury, the Moon, etc.). Pre-Q113
the fluid-dynamic laws (`Wind`, `Hydrology`, `Coriolis`) ran on
vacuum planets with Earth-like coefficients. Cascading guards
mostly kept the output benign — `surface_pressure_pa = 0`
returned a sub-freeze sentinel from `substrate_boiling_point_k`,
making evaporation never trigger; pressure gradient on a
zero-pressure field gave zero force. But the laws were still
*running*, doing per-cell work and quietly producing zero
deltas. Structurally wrong even when numerically correct.

**Decision:** Add a `has_atmosphere: bool` flag to `Wind`,
`Hydrology`, and `Coriolis`. The `integrate` path short-
circuits with `return` when `has_atmosphere == false`.
`build_laws` sets it from `!matches!(planet.atmosphere,
Atmosphere::None)`.

`Lorentz` does *not* get the same guard — vacuum worlds can
still have charge accumulation (electrostatic build-up on
rocks), and the Lorentz force on those charges (with the Q94
magnetic dipole still present) is real physics. So Lorentz
keeps running unconditionally.

`Magnetism`, `Tides`, `Radiation`, `EM`, `HeatConduction`,
`GravityFlow`, `VerticalConvection` all run on vacuum worlds
fine and should not be guarded.

**Why this respects the project goals.**
- *Physics as close to real as possible:* a vacuum world
  shouldn't have wind, weather, or Coriolis deflection. Q113
  enforces that explicitly rather than relying on
  zero-cascade luck.
- *Least hardcoded values:* `has_atmosphere` is derived from
  the planet's `Atmosphere` class, not hardcoded. Vacuum,
  Thin, Oxidising, Reducing, Hazy all treated correctly.
- *Determinism preserved:* short-circuit returns identically
  on every call, no allocation, no per-cell iteration. Same
  seed + same atmosphere class → same result.
- *Minor perf win:* fluid laws skipped on vacuum worlds save
  per-cell work each macro-step. Not the motivation, but a
  side benefit.

**Numerical care.**
- The short-circuit uses early `return` before any allocation
  or borrow of `state`. No `clone` cost is paid on vacuum
  worlds.
- `Wind::has_atmosphere` and `Coriolis::has_atmosphere` are
  `bool`, mutated only by `build_laws` setting (not
  per-tick), so determinism is guaranteed.

**Lives in:** `sim/physics/src/wind.rs` (new field + early
return + 1 new test `q113_vacuum_planet_wind_short_circuits`),
`sim/physics/src/hydrology.rs` (new field + new
`for_substrate` parameter + early return),
`sim/physics/src/coriolis.rs` (new field + new `for_planet`
parameter + early return),
`sim/core/src/lib.rs` (`build_laws` reads
`Atmosphere::None` and threads `has_atmosphere` through).

**Follow-ups (not blocking).**
- *Sub-vacuum gradations.* `Atmosphere::Thin` (Mars-like) is
  not full vacuum but has very thin air; wind dynamics there
  are real but weaker than Earth. Currently `Thin` runs the
  full Earth-calibrated wind law, which over-estimates the
  effect. Lifts naturally when Q104's `density_x100` is wired
  into `Wind::wind_k` (i.e. `wind_k *= density_x100 / 122`).
- *Atmospheric escape.* On a thin / vacuum world, vapour and
  charge would slowly escape to space. Not modelled.
