## Q57 — Settlement / per-region population
**Status:** Decided (M5 v1 implementation: per-cell cohorts on
each civ, with `claimed_cells` set; aggregate accessors preserve
backwards compat where consumers haven't migrated)
**Context:** PLANNING M3 + Q22 reference per-region pop tracking
("Pop ≥ 100 in contiguous region cluster" for Q48 founding,
"net emigration" plateau for Q49 collapse); M4 catastrophe
events (Q60) would prefer per-region scoping for volcanic eruptions
("only the affected region's population drops"); M5 concurrent
multi-civ requires per-region scoping to know which civ holds
which cells.

**The design dilemma.** Per-region cohorts are a substantial
structural rewrite (Cohort layout, Q22 carrying-capacity
computation, Q48/Q49 triggers, Q60 effects, transmission targets).
A v1 minimum that ships per-region tracking *without* migrating
all consumers leaves a confusing two-API state where some logic
reads aggregate and some reads per-region. A full v1 with
consumers migrated is M5-scale work.

**Decision:** Defer the structural rewrite to **M5 prep**, where
concurrent multi-civ makes per-region scoping unavoidable
(multiple civs can't share a single aggregate cohort). M4 ships
with:

- Civ-aggregate cohort (current shape) for population dynamics,
  Q22 capacity, Q49 collapse triggers.
- Per-cell physics state (current shape) for catastrophe
  detection (Q60 reads cell charge/temp directly) and biosphere
  capacity (sums Substance::Fuel across all cells).
- Q48 v2 triggers operate on the species' total stateless cohort
  (not per-region) — acceptable for the inaugural species + 1
  successor cycle; multi-civ-per-region waits on M5.

**Why this satisfies M4 acceptance:** "A run that spans multiple
civs separated by a dark age, with the second civ partially
recovering knowledge from the first via inherited artifacts."
Q49 → Q48 → Q50 → catastrophe-driven population shocks all work
on the aggregate model. The narrative variety the project goal
demands (different worlds → different sciences) comes from
species sensorium + cosmology drift, not regional migration.

**M5 v1 implementation (this Q):**

- `Civ::region_cohorts: BTreeMap<u32 cell_id, Cohort>` holds
  per-cell population. `Civ::aggregate_population()` sums them.
- `Civ::claimed_cells: BTreeSet<u32>` records which cells this
  civ occupies. Q22 capacity sums fuel only over `claimed_cells`.
- Inaugural civ claims every cell on the grid (no other civ
  competes). Successor civs claim the predecessor's `claimed_cells`
  on refound.
- Volcanic catastrophe (Q60) targets the affected cell's region
  cohort directly; disease still global.
- Population dynamics step per-region with per-region capacity.
- Civ::cohort retained as the aggregate accessor for legacy
  callers; reads `aggregate_population()`.

**Deferred to M5 v2** (concurrent multi-civ + conflict):
- Two civs claiming disjoint subsets of the cell grid.
- Migration between adjacent cells under food-security pressure.
- Q10 conflict when two civs' claimed sets touch.

**Lives in:** `sim/civ/lib.rs` `Civ::region_cohorts` (when M5
lands); deferred consumer migration in Q22 / Q48 / Q49 / Q60 /
M5 conflict.
