## Q121 — Host-species wellbeing badge (was Q119 human survivability)
**Status:** Decided
**Context:** Q119 added a `survivability` badge that asked
"would *unprotected humans* survive on this planet?" Test
results across seeds showed the obvious flaw: **most planets
came back as "alien-chemistry" or "toxic-atm"** because they
weren't aqueous + oxidising at human-survivable temperature
ranges. Mira-a's ammoniacal -85°F world reported
`alien-chemistry` even though it was a perfectly fine ammonia
world for its host species (Quilites).

The user pointed out: the badge should describe the *host
species's* wellbeing, not whether humans would die. Q82
guarantees the host species CAN live there *by construction*
(every seed produces a planet whose substrate window matches
the worldgen-sampled physics). The interesting question is
whether the conditions are *easy* (mid-range of substrate's
habitable window) or *stressed* (near the freeze or boil
edges).

**Decision:** Replace `survivability_badge` (Q119) with
`host_species_status`. The badge reflects substrate-relative
position of mean temperature within the substrate's
freeze→boil window:

| Position | Badge |
|---|---|
| < 0 (below substrate freeze) | `frozen-out` |
| 0 - 0.25 | `near-freezing` |
| 0.25 - 0.75 | `thriving` |
| 0.75 - 1.0 | `near-boiling` |
| > 1.0 (above substrate boil) | `boiling-off` |

Plus an `Atmosphere::None` guard for substrates that need an
atmosphere (vacuum-only world → `vacuum`; silicate is
exempted since silicate substrate tolerates vacuum).

Substrate temperature ranges (from Q83/Q85, hardcoded in
viewport to avoid a `sim_report → sim_physics` dependency):

| Substrate | freeze (K) | boil (K) |
|---|---|---|
| aqueous | 273 | 373 |
| ammoniacal | 195 | 240 |
| hydrocarbon | 91 | 112 |
| silicate | 1687 | 3538 |

Sample badges across 5 seeds:
```
Caelum-a · near-freezing  (aqueous, 55°F → 286K)
Glade-a · near-boiling     (aqueous, 170°F → 350K)
Mira-a · thriving          (ammoniacal, -85°F → 208K)
Aurora-b · boiling-off     (aqueous, 255°F → 397K)
Kraken-a · frozen-out      (aqueous, 3°F → 257K)
```

Mira-a in particular flipped from Q119's `alien-chemistry` to
Q121's `thriving` — the host species lives in its substrate's
sweet spot.

**Why this respects the project goals.**
- *Anthropocentric → host-centric:* Q82 guarantees host species
  habitability; the badge now reads what the *natives*
  experience instead of asking the human-centric question.
- *More information per badge:* `near-freezing` /
  `near-boiling` / `frozen-out` / `boiling-off` give 4× the
  semantic resolution of Q119's binary `survivable / not`.
- *Mira-a is correctly thriving:* the ammoniacal Mira reads
  as a comfortable home for its Quilites, not an alien world.
- *Determinism preserved:* pure function on Q32.32 mean
  temperature × substrate enum. No state, no per-tick branching.

**Numerical care.**
- `(mean_t_k - freeze_k) / range`: range is positive for every
  substrate, so the divide is safe. Negative or > 1.0 results
  are explicitly handled.
- Substrate ranges are hardcoded copies of
  `sim_physics::chemistry::substrate_properties`. If those
  values change, both sites need updating; commented in the
  fn doc.

**Lives in:** `sim/report/src/viewport.rs` — replace
`survivability_badge` with `host_species_status`; planet card
line 1 now reads `{name} · {host_species_status}`.

**Follow-ups (not blocking).**
- *Pressure-aware boil per cell.* The badge uses the substrate's
  reference-pressure boil (1 atm). Q92's per-cell pressure-aware
  boil could give a more localized assessment. Probably overkill
  for a single status badge.
- *Atmosphere stress.* Currently atmosphere is only checked for
  the vacuum case. A reducing atmosphere on an aqueous world
  isn't disqualifying for the host species per Q82 (the species
  evolved for that), but it could matter for stress assessment.
- *Use `host_species_status` in the post-run report.* The
  Markdown report still describes planets in human-relative
  terms; switching to host-relative would be more
  Q82-consistent.
