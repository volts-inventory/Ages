## Q83 — Substrate-relative chemistry physics
**Status:** Decided
**Context:** Q82 added `MetabolicSubstrate` (Aqueous / Ammoniacal /
Hydrocarbon / Silicate) so every seed produces life of *some*
chemistry. But `Chemistry::for_planet` still hardcoded water's
273.15 K freeze and Clausius-Clapeyron-derived boil. A 100 K
hydrocarbon world's solvent (methane) would never freeze in the
sim — the chemistry stayed water-chauvinist even though the
substrate said otherwise.

**Decision:** `Chemistry::for_planet` takes a `substrate_tag`
parameter. New `substrate_phase_thresholds(tag)` returns
`(freeze_point_K, boil_reference_point_K)` per substrate:

| Substrate | Freeze (K) | Boil ref (K) |
|---|---:|---:|
| Aqueous | 273.15 | 373.15 |
| Ammoniacal | 195.4 | 239.8 |
| Hydrocarbon | 90.7 | 111.7 |
| Silicate | 1687 | 3538 |

`Substance::{Water, Ice, Vapour}` carries the canonical "solvent
liquid / solid / gas" semantics for the run; on a methane world
`Water` represents liquid methane; on a silicate world `Water`
represents molten silicate. The substance enum doesn't
proliferate — the *thresholds* dispatch on substrate.

The Aqueous case still routes through `water_boiling_point_k`
(Clausius-Clapeyron from `surface_pressure_pa`) so pressure-
varying water boil is preserved. Other substrates use their
constant reference boil point for now; substrate-specific
Clausius-Clapeyron is a follow-up refinement.

**Why this respects the project goals.**
- *Physical / biological grounding:* solvent dynamics now match
  the substrate. A Titan-style world's methane lakes freeze
  near 91 K, not 273 K.
- *Emergence over authoring:* no per-cell special-casing. The
  one-line `for_planet(... substrate_tag)` signature change
  threads through cleanly.
- *Different worlds, different sciences:* recognition templates
  that key off freeze/boil events (`seasonal_thaw` at 273-290 K
  for Aqueous) still fire correctly; substrate-specific
  templates (Q84) gate on the new substrate-relative
  thresholds.

**Lives in:** `sim/physics/src/chemistry.rs` (`for_planet`
signature + `substrate_phase_thresholds` helper),
`sim/core/src/lib.rs` (passes `planet.metabolic_substrate.tag()`).

**Follow-ups (not blocking).**
- Substrate-specific Clausius-Clapeyron for non-Aqueous
  substrates so boil shifts with pressure.
- Per-substrate latent-heat coefficients (currently all
  substrates use water's `L_FUSION_WATER` / `L_VAPORISATION_WATER`
  as a proxy).
