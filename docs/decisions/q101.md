## Q101 — Q32.32 sin/cos via Taylor series, replacing triangular profiles
**Status:** Decided
**Context:** Pre-Q101 the codebase had no trig in Q32.32: every
place that wanted a cosine bulge — Q90 latitude attenuation, Q93
tidal potential, Q94 dipole magnitude, Q97 seasonal swing —
fell back to a piecewise-linear "triangular" approximation.
Triangles got the +1/0/-1 phase relation right but were the
wrong *shape* (steeper at peaks, flat at zero crossings instead
of the cos shape's flat-at-peaks, steep-at-zero behaviour).

The Q90/Q93/Q94/Q97 docs all listed "cosine bulge profile" as a
follow-up that landed when Q32.32 trig arrived. Q101 ships the
trig and switches the four callers in one wave.

**Decision:** Add `sin` and `cos` to
`sim_arith::transcendental` via Taylor series:

```text
1. Reduce x to [-π, π] via x - 2π · round(x / 2π).
2. If |reduced| > π/2, reflect via sin(π - r) = sin(r) and
   sin(-π - r) = -sin(-r), giving |r| ≤ π/2.
3. Compute sin(r) = r - r³/3! + r⁵/5! - ... using 17 terms.
4. cos(x) = sin(x + π/2).
```

17-term Taylor on `[-π/2, π/2]` is bounded by `(π/2)¹⁷ / 17!`
≈ 5e-13 — well below Q32.32's LSB (~2.3e-10). Verified by
`q101_sin_squared_plus_cos_squared_is_one` (the Pythagorean
identity holds within 1e-7 across nine test angles spanning
multiple periods) and `q101_sin_at_cardinal_angles` /
`q101_cos_at_cardinal_angles` (sin(0), sin(π/2), sin(π),
sin(3π/2), sin(2π) all match within 1e-7).

Public constants `pi()`, `two_pi()`, `half_pi()` exposed for
callers doing angle arithmetic.

**Switching the four callers:**

- *Q90 `cos_lat`:* `cos(half_pi · pole_dist / max_dist)`
  replaces `1 - pole_dist / max_dist`. Steeper attenuation
  near the poles, flatter near the sub-solar row.
- *Q93 tidal potential:* `cos(2θ)` where
  `θ = 2π · q_diff / width` replaces the triangular two-peak
  profile. Real tidal-potential shape: smooth peaks at
  sub-lunar and antipodal, smooth troughs at the quarter-
  circles.
- *Q94 dipole `lat_factor`:* same as Q90 — `cos(half_pi ·
  pole_dist / half_h)` for both `init_field` and `integrate`.
- *Q97 seasonal swing:* `cos(2π · season / 12)` replaces the
  4-segment triangular `triangle_cos`. Smooth seasonal swing
  instead of piecewise-linear segments.

**Why this respects the project goals.**
- *Physics as close to real as possible:* every place that
  modelled a cos bulge now does so with the actual cosine
  shape. The behaviour difference is sub-percent at most
  resolutions but matters when fits try to learn the underlying
  shape — civs hypothesising a "sin / cos / power-law" form
  vocabulary now have a real cosine in the data instead of a
  triangle that no closed-form fits.
- *Determinism:* `q101_sin_is_deterministic` confirms that
  `sin(x)` produces bit-identical Q32.32 raw bits across calls.
  Taylor evaluation is pure arithmetic; argument reduction
  uses `round_ties_even` (deterministic rounding mode).
- *No regression on existing tests:* every Q90/Q93/Q94/Q97
  test still passes after the swap (sim-physics 61/61).
- *Reusable:* future Q-decisions (Coriolis Q106, eccentric
  orbits Q107, diurnal Q108) all want trig; the API is now
  there.

**Numerical care.**
- `sin` evaluation: 17 multiplications, 8 divisions, 8
  additions per call. Roughly 3-4× the cost of `exp`. Per-tick
  hot paths that previously used integer triangle math now
  pay this cost; for the 24,000-tick long test the wall clock
  stayed essentially flat (Q97 already collapsed the per-tick
  trig into a build-time table; the per-tick callers are
  Q93 tides which run once per macro-step — small N).
- `round_ties_even` on the argument-reduction quotient avoids
  the platform-dependent rounding behaviour that
  `as i64`-style truncation would introduce.

**Lives in:** `sim/arith/src/lib.rs` (the `sin` / `cos`
functions + `pi`/`two_pi`/`half_pi` constants + 4 new tests),
`sim/physics/src/radiation.rs` (Q90 cos_lat + Q97 seasonal
swing both swap to real cos),
`sim/physics/src/magnetism.rs` (Q94 dipole lat_factor at both
init_field and integrate),
`sim/physics/src/tides.rs` (Q93 tidal potential becomes
cos(2θ); the now-unused `wrap_dist` helper is removed).

**Follow-ups (not blocking).**
- *Trig table for hot paths.* For applications that want to
  call `sin` or `cos` on every cell every tick (which Q90's
  pre-built table avoids), switching to an interpolated
  256-entry table cuts the cost ~5× at the price of
  ~6-bit precision. Worth doing if a future Q-decision
  needs per-cell-per-tick trig.
- *Atan2 / asin / acos.* Inverse trig follow-ups for
  Coriolis (Q106) heading angles or aurora geometry. Same
  Taylor-series approach.
