## Q133 — `./run.sh` launch script
**Status:** Decided
**Context:** Q122-Q132 settled the viewport's defaults, but a
"watch a fresh world" session still required the user to
remember six args (`--seed`, `--years`, `--cli viewport`,
`--tick-rate-ms`, `--frame-every-ticks`, `--out`) and pick a
random seed manually. A one-shot script removes that friction
and codifies the agreed-on defaults so future sessions can
just `./run.sh`.

**Decision:** Add `run.sh` at the repo root with hardcoded
defaults walked through one-by-one with the user:

| Arg | Value | Rationale |
|---|---|---|
| `--seed` | `$RANDOM$RANDOM` per-launch | Random Bash seed; planet/species names derive from it (Q119/Q120) so memorable seeds are recoverable by re-passing the same value |
| `--years` | `5000` | Full multi-civ-generation arc (foundings, transmissions, catastrophes, collapses) |
| `--cli` | `viewport` | The whole point — live ASCII UI |
| `--tick-rate-ms` | `50` | Pacing arg; @ 50ms tick rate, sim is bottlenecked on Pi5 and watchable on x86 |
| `--frame-every-ticks` | `6` | Half a sim-year per frame; ~3 fps on Pi5, smooth on x86 |
| `--out` | `runs/{date}-{seed}.ndjson` | Archived per-run; sortable by date, identifiable by seed |

Defaults inherited (not passed): `--grid-width 32`, `--grid-height
20` (Q130), `--temperature-unit f` (Q118), `--viewport-log-lines
5` (Q114), color on (Q130), planet card on (Q124), compact mode
(Q122).

The script:
1. Runs `cargo build --release --bin ages` so the latest viewport
   changes ship with each launch (no-op if already current).
2. Generates a random 64-bit-ish seed via `$RANDOM$RANDOM` (two
   concatenated 16-bit Bash builtins).
3. `mkdir -p runs/` for the archive directory.
4. Constructs the timestamped output path.
5. Echoes `seed=… years=… out=…` so the user can note the seed
   for replay.
6. `exec`s `./target/release/ages` with the agreed args.

**Why this respects the project goals.**
- *Codify defaults, don't hardcode them in the binary.* The
  binary's `RunConfig::dev` defaults (Q122) match the script's
  inherited values. If a future Q changes those defaults
  (e.g., bumps the grid), the script picks up the change
  without an edit — only the *script-explicit* args
  (years/pace/cli/out) are pinned.
- *Reproducibility-friendly.* Every run logs its seed to stdout
  before launch, and the NDJSON path embeds the seed. A user
  who notices a fascinating world can re-run it deterministically
  with `./target/release/ages --seed N --years 5000 …` later.
- *Minimal.* Bash builtins only — no `od`, no `awk`, no
  `python -c`. Works on any standard bash shell (Linux, macOS).

**Numerical care.**
- `$RANDOM$RANDOM` is two concatenated 0..32767 values, so the
  seed range is roughly 0..1_073_741_824 (~30 bits). For a
  one-shot launcher this is enough seed entropy; identical
  seeds within a single human's lifetime are vanishingly
  unlikely.
- `set -euo pipefail` — fail-fast if any command errors, no
  unset-var fall-throughs, pipeline failures propagate.
- `exec` replaces the shell process with the sim binary so
  Ctrl-C reaches the sim cleanly (no zombie shell layer).

**Lives in:** `run.sh` at repo root, executable
(`chmod +x run.sh`).

**Follow-ups (not blocking).**
- *Optional `$1` years override.* Decided against — keeps the
  script a one-shot. Users wanting custom years can call the
  binary directly.
- *Auto-detect Pi5 vs x86 and adjust tick-rate.* Not needed
  yet — at 50ms tick rate Pi5 is sim-bottlenecked anyway and
  the x86 wall-clock is fine.
- *`run-replay.sh <seed>` companion.* Could replay a known
  interesting world. Pure quality-of-life; deferred until a
  user asks.
