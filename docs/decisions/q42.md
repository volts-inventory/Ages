## Q42 — Spatial grid resolution
**Status:** Decided
**Context:** Physics runs on a spatial grid covering the planet.
Two coupled choices: lattice geometry (hex vs square) and resolution
(cells per planet). Six options audited (A through F).
**Decision:** **A — hex lattice, ~2500 cells default, configurable.**

- **Lattice**: hexagonal, axial coordinates `(q, r)`. Hex over square
  because we integrate fluid dynamics in M1a; square's grid-axis bias
  produces visible artefacts in tides, atmospheric flow, and ocean
  currents — phenomena civs are meant to observe and derive laws
  from. Worth the slightly more complex coordinate system.
- **Default resolution**: 2500 cells (~200 000 km²/cell on Earth-
  sized). Middle of the 1000–5000 range — compelling fluid behaviour
  without prohibitive per-tick cost.
- **Configurable**: 500–10 000 cells per run via config. Tests can
  use coarser; richer runs can use finer. No code change needed.
- **Crate choice**: `hex2d` or equivalent, or hand-rolled per the
  `sim/arith`-routed-only discipline. Picked at scaffold time.

**Escape hatches (deferred but tracked)**:
- **E — adaptive mesh refinement**: finer cells near coasts,
  tectonic boundaries, charge-accumulation zones. Real research
  problem under determinism (deterministic AMR is hard). Add only
  if specific phenomena demand it during M1a/M1b/M2.
- **F — nested grids per physics layer** (atmosphere coarse, ocean
  fine, etc.). Coupling between layers is a research problem.
  Reserved as a possible direction if uniform hex hits scaling
  walls.

**Note to revisit:** Verify during M1a that 2500 cells produces
recognisable fluid and thermal behaviour. If it doesn't, first move
is widening to ~5000 (still option A); E/F only if uniform-hex hits
real walls.
