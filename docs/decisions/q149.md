## Q149 — Side-by-side viewport layout
**Status:** Decided
**Context:** The pre-Q149 viewport was a vertical stack: planet
section → blank → map section → key → species → log. On a phone-
portrait Termius window (~25 visible rows once the keyboard is
up), the legend + species sections + log together pushed half the
map off-screen any time the user scrolled the keyboard up to type.
The visual hierarchy also fought itself — the map was the focus,
but the key/species/log sections were stacked underneath it
demanding attention with full-width dividers.

A side-by-side layout fixes both: the map sits on the left, every
non-map element shares a sidebar on the right, and the per-civ
panels (Q150) get a sensible home where they don't compete with
the map for vertical space.

**Decision:** Replace the vertical-stack `render()` body in
`sim/report/src/viewport.rs` with a three-section layout:

1. **Top — planet section, full-width.** Unchanged from pre-Q149:
   `--- Lumen-h ---` divider, 4 centred card lines (badge+type,
   climate, orbital, caption). Width bumped from 40 to 70 cols so
   the divider matches the new total width.

2. **Middle — side-by-side map + sidebar.** The defining change.
   - **Map zone** (cols 0–35, `MAP_WIDTH = 36`). The compact-mode
     map renders as `{r:>2}|` (3-char row prefix) + 32 cells +
     `|` right border = 36 cols.
   - **Gap** (cols 36–38, 3 cols). Single space, vertical `|`
     rule, single space.
   - **Sidebar zone** (cols 39–68, `SIDEBAR_WIDTH = 30`). Three
     stacked sub-blocks separated by blank rows:
     1. **Legend** (Q148, 4 lines): every glyph the renderer can
        emit, including Q139's `·` and `≡` fallbacks.
     2. **Species** (3 lines): re-uses `species_card()`'s body
        (cognition, sense+manip, biology) under a
        `─── {species_name} ───` mini-header.
     3. **Per-civ panels** (Q150, 3 lines per active civ): one
        block per civ in `self.civs`, auto-added on `CivFounded`
        and auto-dropped on `CivCollapsed`.
   - **Top divider** of this section: `--- map ---+--- key ---`
     with `+` corner at `RULE_COL = MAP_WIDTH + 1 = 37`. The `+`
     is where the vertical `|` rule meets the dashed horizontal
     header.
   - **Per-row** of this section: `[map_padded] + " | " +
     [sidebar_padded]`. Map rows pad to `MAP_WIDTH` with trailing
     spaces; sidebar rows pad to `SIDEBAR_WIDTH`. Either side
     becomes blank when the other has more rows — the row count
     is `max(map_rows, sidebar_rows)`, so a long per-civ panel
     list extends the section below the map's bottom border.

3. **Bottom — log section, full-width.** Divider:
   `--- log ---+---` (split form, `+` corner at `RULE_COL` so the
   vertical `|` rule terminates cleanly), then N centred log
   lines spanning the full 70-col width. Same content as pre-Q149.

**Width math:**
```
MAP_WIDTH      = 36       (3-char row prefix + 32 cells + 1 right |)
gap            =  3       (space + | + space)
SIDEBAR_WIDTH  = 30       (legend + per-civ headers fit, with margin)
                ----
total          = 69 → rounded up to VIEWPORT_WIDTH = 70
```

The +1 from rounding is absorbed in the divider math (`---` runs
slightly longer on the right).

**Implementation outline (in render()):**
1. Render planet section (full-width, unchanged path).
2. Build map body via existing `render_world_frame_*` variants,
   strip ` ```text ` fences (Q124), split on `\n` → `Vec<String>`.
3. Call `build_sidebar_lines()` (new helper) to assemble the
   sidebar `Vec<String>` (legend + species + per-civ panels).
4. Emit `split_divider("map", "key")` (new helper) which formats
   `--- map ---+--- key ---` with the `+` at `RULE_COL`.
5. For `i in 0..max(map_rows, sidebar_rows)`:
   `pad_to(map_lines[i], MAP_WIDTH) + " | " +
    pad_to(sidebar_lines[i], SIDEBAR_WIDTH)`.
6. Emit `split_divider("log", "")` (right label empty → pure
   dashes on the right side, but the `+` corner still anchors
   the vertical rule at `RULE_COL`).
7. Emit log lines (centred within the full 70-col width).
8. Q146 atomic clear-and-paint preserved — the whole frame is
   built into a `Vec<u8>` and prepended with `\x1b[H\x1b[2J`
   before a single `write_all`, so the terminal sees clear and
   content as one chunk.

**Why this respects the project goals.**
- *Map is the focus.* The full-brightness map sits on the left;
  the sidebar (under ANSI dim when `use_color=true`) recedes.
  Same dim-vs-bright contrast as Q130 used for the section
  layout, just rotated 90°.
- *Phone-portrait friendly.* Map zone width unchanged at 36 cols
  → map still fits portrait. The sidebar adds 33 cols (gap +
  width); a phone in landscape (or a desktop terminal) sees
  everything; portrait users see the map and as much of the
  sidebar as the column count allows. The user already runs
  the viewport in a code-block-rendered chat surface that wraps
  fine.
- *Determinism preserved.* Every layout helper is a pure function
  of its inputs. Same NDJSON event sequence → same rendered
  bytes.
- *Backwards compatibility for tests.* The bare-map test path
  (`show_planet_card = false, log_lines = 0`) still works:
  `build_sidebar_lines()` returns empty, the per-row loop falls
  back to `write_centered_line(map_row)`, and the dividers use
  the unsplit form.

**Numerical care.**
- `pad_to` uses `visible_width` (Q138) so ANSI escapes don't
  inflate the column count when civ markers carry colour codes.
- The `+` corner on split dividers lands at exactly
  `RULE_COL = MAP_WIDTH + 1 = 37` regardless of label length —
  the left half is padded to `MAP_WIDTH + 1` chars
  (`fmt_half(label, MAP_WIDTH + 1)`), which means even an empty
  label gets the right number of dashes.
- Map zone always emits `MAP_WIDTH` visible chars per row
  (the compact renderer enforces this via the `+----+` borders);
  no need to truncate.

**Lives in:**
- `sim/report/src/viewport.rs` — `VIEWPORT_WIDTH` bumped to 70;
  `MAP_WIDTH` / `SIDEBAR_WIDTH` constants added; `split_divider`
  + `pad_to` helpers; `build_sidebar_lines` helper;
  `render()` rewritten to the three-section layout.
- `sim/report/src/frame.rs` — `centroid_symbol` /
  `claim_symbol` switched from private to `pub(crate)` so
  `viewport::build_sidebar_lines` can call them for the per-civ
  panels (Q150).

**Follow-ups (not blocking).**
- *Sidebar overflow scroll.* If a run produces > ~6 active civs,
  the per-civ panels overrun the map height and extend the
  middle section below the map's bottom border. Current behaviour
  pads the missing-map rows with spaces and continues — readable
  but wastes vertical space. Could collapse panels to one line
  each (`A=cap, 1=civ, y0, 7c`) when row pressure is high.
- *Sidebar resize on narrow terminals.* On a true 30-col phone
  terminal, the sidebar zone runs past the visible window and
  wraps. Pre-Q149 the vertical-stack layout fit; post-Q149
  needs ≥ 70 cols. The user's chat surface and Termius
  landscape both supply that comfortably.
