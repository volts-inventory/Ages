## Q118 — Viewport temperature unit (Fahrenheit default)
**Status:** Decided
**Context:** Q114's planet card displayed mean surface
temperature in Kelvin (`350K`). Most readers — especially in
the US — find Fahrenheit more intuitive for surface-temperature
intuition (350 K ≈ 170 °F = "very hot" is much clearer than
"350" Kelvin). Internal physics is always Kelvin (the SI unit
all latent-heat / Stefan-Boltzmann math runs on); the question
is what the *display* shows.

**Decision:** Add a `temperature_unit` field to `ViewportConfig`
defaulting to `Fahrenheit`. New CLI flag `--temperature-unit
f|c|k` lets the user pick:
- `f` (default) → "170F"
- `c` → "77C"
- `k` → "350K" (the original behaviour)

Internal physics is unaffected — only the planet card's display
formatting.

**Why this respects the project goals.**
- *Reader-friendly default:* surface-temperature intuition
  is the most useful thing the planet card communicates;
  Fahrenheit gives that intuition immediately for the largest
  reader base.
- *Universal switchability:* one CLI flag covers all three
  common units. Physics-minded readers get Kelvin with one flag.
- *No physics impact:* the conversion is a pure display-side
  function of the existing `mean_temperature_q32` value.
  Determinism preserved.
- *Same architecture pattern:* `TempUnit` enum follows the
  established `ChannelKind` / `ClimateBand` shape.

**Lives in:** `sim/report/src/viewport.rs` (new `TempUnit` enum
+ `temperature_unit` field on `ViewportConfig` + display
formatting in `planet_card`), `sim/report/src/lib.rs`
(re-export `TempUnit`), `ages/src/main.rs` (new
`--temperature-unit` CLI flag + plumbing).

**Follow-ups (not blocking).**
- *Per-unit gradient display.* Q114 dropped the `±N K` gradient
  to fit width. If users want it back, adding `±NF` (or `±NC`)
  with the same unit conversion is trivial.
- *Other unit displays.* Day length is in hours (universal),
  axial tilt in degrees (universal), year length in months
  (sim-internal). All three are reader-friendly already; no
  pressure to change.
- *Post-run-report unit option.* The Markdown / HTML report
  also shows temperatures in K. Adding the same flag to
  `ages-report` is an obvious follow-up if a consumer asks.
