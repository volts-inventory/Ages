## Q137 — Refresh stale `--help` text after Q130's grid bump
**Status:** Decided
**Context:** Q130 bumped the default grid from 24×16 to 32×20,
but the `--help` text in `ages/src/main.rs` still described the
old defaults — "default 24" / "default 16" / "fits a 24×16 grid
in ~27 cols". A user running `ages --help` would get
mis-information about what they'd actually get.

This is the LOW finding from the Q133/Q134 audit. Trivial fix;
batched alongside Q136 because both are pure docs touchups.

**Decision:** Update three sentences in `print_help`:

| Before | After |
|---|---|
| `fits a 24×16 grid in ~27 cols` | `fits the 32×20 grid in ~36 cols` |
| `default 24` (grid-width) | `default 32` (grid-width) |
| `default 16` (grid-height) | `default 20` (grid-height) |

The arg defaults themselves (in `parse_args`) were already
correct since Q130; only the help text was stale.

**Lives in:** `ages/src/main.rs` — three lines in `print_help`.

**Follow-ups (not blocking).**
- *Periodic help-text audit.* Comments and `--help` text drift
  whenever code constants change. A future test could parse
  `--help` output and assert it mentions the values from the
  CLI defaults — but it's overkill for a help string. Manual
  audit on grid/pacing changes is fine.
