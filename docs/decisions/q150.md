## Q150 — Per-civ sidebar panels
**Status:** Decided
**Context:** Q149's side-by-side layout opened up the right-hand
sidebar; the obvious thing to put there — once the legend + species
sub-blocks are placed — is one block per currently-active civ
showing identity at a glance. Q152's deterministic civ names
(`Karnan`, `Veruman`, etc.) gave each panel a memorable header,
and the existing `centroid_symbol(civ_id)` / `claim_symbol(civ_id)`
helpers in `frame.rs` already mapped civ_id to the visible glyphs
on the map, so the sidebar panel just needed to round up:
- the civ's letter on the map (`A`, `B`, …),
- the civ's digit (`1`, `2`, … `*`),
- when it founded,
- how big it is right now.

**Decision:** Each currently-active civ in `self.civs` gets a
3-line sidebar block under the heading `─── {Civ name} ───`:

```
─── Karnan ───
A=cap · 1=civ
y0 · 10 cells
```

Layout per block:
- **Header line** — `─── {name} ───`. `name` comes from the
  `CivFounded::name` field (Q152). When an older NDJSON event
  predates Q152 and `name` is empty, fall back to `civ {civ_id}`
  so the header is always non-empty.
- **Identity line** — `{centroid_letter}=cap · {claim_digit}=civ`.
  Centroid letter from `frame::centroid_symbol(civ_id)` (uppercase
  A..Z mod 26); claim digit from `frame::claim_symbol(civ_id)`
  (1..9, then `*` for civ_id ≥ 10). Both helpers were private
  before Q150; bumped to `pub(crate)` so `viewport.rs` can use
  them.
- **Stats line** — `y{founded_year} · {N} cells`. Founded year
  derived from the `CivFounded.tick` divided by
  `protocol::MONTHS_PER_YEAR` (Q67's tick=month convention). Cell
  count is `claim.claimed_cells.len()` from the live `CivClaim`
  (already maintained for the map renderer).

**State plumbing in `ViewportEmitter`:**
- New field `civ_names: BTreeMap<u32, String>` — captures
  `CivFounded::name` per civ so the sidebar can read it without
  re-scanning the event stream.
- New field `civ_founded_year: BTreeMap<u32, u64>` — captures
  `tick / MONTHS_PER_YEAR` at `CivFounded` time.
- Both maps populated in the `Event::CivFounded` arm of
  `apply_state` and *removed* in the `Event::CivCollapsed` arm so
  the panels auto-disappear when a civ falls. The existing
  `civs` BTreeMap is the source-of-truth for "which civs render
  now"; the new maps are sidebar-only ornaments keyed off the
  same civ_id.

**Sidebar assembly (`build_sidebar_lines`):**
Order: legend (4 lines) → blank → species sub-block (`─── {sp} ───`
+ 3 lines) → blank → per-civ panels (each preceded by a blank
row). Iteration is `for (civ_id, claim) in &self.civs` —
`BTreeMap`'s ascending civ_id order is deterministic, so panels
list oldest-civ-first.

**Why this respects the project goals.**
- *Map is the focus, sidebar is glanceable.* Each per-civ block
  is 3 lines + 1 separator = 4 rows; with 2 civs that's 8 rows
  of sidebar after the legend + species, well within the 22-row
  middle section a 32×20 grid produces.
- *Q149 absorbs overflow gracefully.* The
  `max(map_rows, sidebar_rows)` row count means a 6-civ run
  extends the middle section a few rows below the map's
  `+----+` bottom border instead of truncating panel data.
- *Names are readable.* Q152 keyed names by `(seed, civ_id)`
  with kingdom-flavoured stems × endings; the user gets to
  associate a memorable string (`Karnan`) with the letter on
  the map (`A`) instead of watching numeric `civ 1` render.
- *Determinism preserved.* All sidebar state is keyed on civ_id
  and updated in event order; same NDJSON in → same panels out.

**Numerical care.**
- `civ_names.insert` on `CivFounded` and `civ_names.remove` on
  `CivCollapsed`: O(log n) on civ count, n ≤ ~20 in practice.
- `claim.claimed_cells.len()` is O(1) on `BTreeSet`.
- `MONTHS_PER_YEAR` is a u64 const (Q67); the year division is
  exact integer arithmetic.

**Lives in:**
- `sim/report/src/viewport.rs` — three new fields on
  `ViewportEmitter` (`civ_names`, `civ_founded_year`,
  `civ_cosmology` — the third is Q148's), populated in
  `apply_state`'s `CivFounded` / `CivCollapsed` arms;
  `build_sidebar_lines()` builds the legend + species + per-civ
  blocks; `render()` zips it row-by-row with the map body.
- `sim/report/src/frame.rs` — `centroid_symbol` and
  `claim_symbol` now `pub(crate)`.

**Follow-ups (not blocking).**
- *Population in the panel.* The `CivClaim` already carries
  `cell_populations_q32`; could add `pop {N}` line. Skipped here
  to keep the panel under 4 lines for layout safety; defer.
- *Tech-tier indicator.* The narrator surfaces tech tier
  crossings (`paleolithic` → `neolithic` → …); a single-letter
  badge on the panel header (`─── Karnan [N] ───`) would tell
  the viewer how advanced each civ is at a glance. Defer until
  the tier-tracking state is added to the viewport.
- *Collapsed-civ tombstones.* Currently the panel disappears the
  tick a civ falls. Could keep a faded `─── Karnan (fell y124)
  ───` for the next ~5 frames so the viewer notices the loss
  before the slot clears. Defer.
