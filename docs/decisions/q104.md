## Q104 — Per-atmosphere numeric scale height + density
**Status:** Decided
**Context:** The `Atmosphere` enum (`None`, `Thin`, `Oxidising`,
`Reducing`, `Hazy`) was a symbolic tag — recognition templates
keyed off it, build-laws picked albedo / greenhouse / ignition
thresholds from it, but the underlying *numeric* properties
(scale height, density) were absent. Q96 had to hardcode
`SCALE_HEIGHT_M = 8400` (Earth-like) for every world; Q91 wind
and Q98 Lorentz coefficients were similarly Earth-calibrated
across all atmospheres.

The Q92, Q96, Q91, Q98, and Q104 docs all flagged
"per-atmosphere scaling" as a follow-up.

**Decision:** Add two numeric methods to `Atmosphere`:

```rust
impl Atmosphere {
    pub fn scale_height_m(self) -> i64 { ... }
    pub fn density_x100(self) -> i64 { ... }
}
```

Calibrated against real-planet values:

| Class | scale_height_m | density (kg/m³) | Real-planet analogue |
|-------|---------------:|----------------:|----------------------|
| `None` | 1 (sentinel) | 0 | airless rock |
| `Thin` | 11 000 | 0.02 | Mars |
| `Oxidising` | 8 400 | 1.22 | Earth |
| `Reducing` | 15 000 | 67 | Venus |
| `Hazy` | 21 000 | 5.4 | Titan |

Update `Hydrology::for_substrate` to take `scale_height_m`, and
thread `planet.atmosphere.scale_height_m()` through
`build_laws`. Pre-Q104 every world used 8400; post-Q104 a
Mars-thin world's altitude-pressure profile is shallower (you
have to climb higher for the same pressure drop), Venus-thick is
deeper, Titan-cold-methane is the most gradual.

The `density_x100` method is exposed for downstream
Q-decisions to scale wind / Lorentz / advection coefficients;
Q104 itself doesn't yet wire it (left as deferred follow-ups
for Q-decisions that focus on those laws).

**Why this respects the project goals.**
- *Physics as close to real as possible:* atmospheres now have
  the *correct* altitude-pressure profile for their type.
  Earth-like cells boil at ~373 K at sea level, ~357 K at 5 km
  (`exp(-5000/8400)·101325 = 55 kPa`); Mars-like cells boil at
  ~373 K at sea level, ~242 K at 5 km
  (`exp(-5000/11000)·6 kPa = 3.8 kPa`).
- *Lifts a hardcoded constant:* `SCALE_HEIGHT_M = 8400` was
  the last hardcoded planetary constant in Hydrology. Now
  derived from `Planet::atmosphere`.
- *Density numeric available:* future Q-decisions for
  per-atmosphere wind scaling (Q91 follow-up) read
  `density_x100` directly without re-deriving it.
- *Determinism:* the new methods are pure / `const`-ish
  matches; no state-dependent branching.

**Numerical care.**
- For `Atmosphere::None` (`scale_height_m = 1`), the
  barometric formula `exp(-elev/H)` would overflow Q32.32 for
  any non-trivial elevation. Hydrology now special-cases
  `scale_height_m <= 10` to return zero pressure (vacuum
  physics: every cell evaporates instantly, matching the
  sub-freeze sentinel `substrate_boiling_point_k(_, 0)`
  returns).
- For other atmospheres, the exp argument is clamped to
  `[-20, 1]` before passing to `exp`. Negative-elevation
  basins (Death Valley, Hellas Planitia) get up to
  `exp(1) = 2.7` × surface pressure, well above any plausible
  basin's actual pressure but harmless as an upper bound.
- Bonus fix: `Form::ExpDecay` and `Form::ExpGrowth` in sim-civ
  now clamp their exp arguments to `[-20, 20]` to prevent
  panics when fits drift into ranges where the `params[1] · x`
  product exceeds Q32.32's exp range. Q104's per-atmosphere
  pressure variation pushed some seed-42 fits into that
  regime; the clamp returns finite (saturated) values instead
  of panicking.

**Lives in:** `sim/world/src/lib.rs` (new
`Atmosphere::scale_height_m` and `::density_x100` methods),
`sim/physics/src/hydrology.rs` (new `scale_height_m` field +
vacuum special case + exp-arg clamp),
`sim/core/src/lib.rs` (`build_laws` threads
`planet.atmosphere.scale_height_m()` to Hydrology),
`sim/civ/src/forms.rs` (exp-arg clamp on `ExpDecay` /
`ExpGrowth`).

**Follow-ups (not blocking).**
- *Per-atmosphere wind scaling.* Q91 `Wind`'s `wind_k` and
  `advect_k` should scale with `1/density` and `density`
  respectively (real Newtonian momentum × heat-content). Lifts
  the Earth-like wind dynamics into per-planet shapes.
- *Per-atmosphere Lorentz scaling.* Q98 `Lorentz`'s
  `lorentz_k` could scale with density to reflect the
  charge-carrier mass per cell. Smaller for thin atmospheres,
  larger for thick.
- *Per-atmosphere c_p / specific heat.* Currently the cell
  thermal mass × c_p denominator in latent heat is water-
  calibrated. Atmosphere-aware c_p would change how much
  temperature swings per unit mass-phase-change.
