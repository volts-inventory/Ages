## Q34 — Fit metric and tolerance formula
**Status:** Decided
**Context:** Hypothesis fitting needs a fit metric and a tolerance for
when residual is "low enough." Original sketch was
`confidence = 1 / (1 + residual / tolerance)` and
`tolerance = base_tolerance / (intelligence × (1 + supporting_density))`.
**Decision:** RMSE as the fit metric, exponential-decay confidence,
intelligence applied to both tolerance and minimum-sample requirement,
per-form tolerance multipliers.

- **Metric:** `residual = sqrt(sum((y_i - f(x_i))^2) / n)` (RMSE).
  Closed-form for linear / polynomial templates, well-known for
  non-linear, deterministic under fixed-point arithmetic.
- **Tolerance:** `tolerance = base_per_form × (1 / intelligence_factor)
  / sqrt(n)` — base multiplier is per-form (e.g. `constant` tight,
  `periodic_sine` loose); intelligence shrinks tolerance so smarter
  species demand tighter fits; `1/sqrt(n)` shrinks tolerance as more
  data accumulates.
- **Confidence:** `confidence = exp(-residual / tolerance)`. Bounded
  (0, 1], no asymptote at residual=0, degrades smoothly as residual
  grows. Replaces the original `1 / (1 + r/t)` sketch.
- **Minimum-sample requirement:** `n_min = ceil(k_form /
  intelligence_factor)` — smart species can attempt a fit with fewer
  points; per-form floor `k_form` reflects how many observations a
  given template needs to be identifiable (e.g. `polynomial_3` needs
  more than `linear`).

**Why not reduced χ² now (Alt B):** the principled answer is to
compare residuals to expected measurement noise, not to a fudge
tolerance. But recognition firings today carry no σ. Migrating to
reduced χ² requires recognition channels to emit measurement noise,
which is meaningful work — split out as Q54.
