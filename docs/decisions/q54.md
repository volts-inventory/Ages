## Q54 — Recognition-channel measurement noise + reduced-χ² migration
**Status:** Open (blocks on recognition-layer per-channel σ; not
M3-acceptance-blocking, lands whenever recognition gets that
capability — could be a M3 tail commit, M4, or later)
**Context:** Q34 chose RMSE + tunable tolerance for the M3 fit
pipeline because recognition firings today carry no measurement σ.
The principled metric is reduced χ² — residuals compared to expected
noise rather than a fudge constant — but it requires every recognition
channel to emit a noise estimate alongside its value.
**Decision:** Pending. Two coupled questions:
1. Should recognition channels carry per-firing measurement σ?
   (If yes: how is σ derived — per-template constant, per-region
   environmental, per-instrument once sensorium tech lands?)
2. If σ is available, switch the fit pipeline from RMSE+tolerance to
   reduced χ² (`χ²_red = (1/(n-p)) × sum(((y_i - f(x_i))/σ_i)^2)`),
   accept fits at `χ²_red ≤ 1+ε`?
**Recommended starting position:** Add per-template constant σ in M3
follow-up work; defer per-instrument σ to the sensorium-tech
milestone. Migrate to reduced χ² once σ is in the firing payload.
Keep RMSE path as fallback for any channel that hasn't declared σ.
**Lives in:** `sim/recognition` channel schema, `sim/civ` fit module.
