## Q128 — Species section, friendly badges, axis separators
**Status:** Decided
**Context:** Q127 settled the planet card on a 5-line themed
layout (identity / climate / orbital / species / biology), but
five follow-ups landed in user feedback:

1. *`boiling-off` is opaque.* The Q121 host-species wellbeing
   badges are precise but jargon-hyphenated. A reader wants a
   single-word adjective ("scorching") next to the planet type,
   not a compound term they have to parse.
2. *Planet type wanted before the badge.* The card opened with
   `{badge} · {substrate-descriptor}`; flipping to
   `{planet_type} · {badge}` reads like a noun-adjective phrase
   that an astronomy reader recognises ("ocean world · scorching"
   vs "boiling-off · water-based").
3. *Species deserves its own section.* The species traits + bio
   lines were the bottom two rows of the planet card, where
   they competed with planet info. With a `Vec<Species>` future
   in mind (Q47 is currently single-species but the user
   anticipates growth), splitting the species into its own
   section lets the layout extend to N species without
   restructuring.
4. *`cent-med` is too cryptic for its own section.* Once
   species had a dedicated section, the abbreviation no longer
   needed to fit shared 40-col-with-name budget. Expanding to
   `centralized medium cognition` reads as a noun phrase.
5. *Center every line; add ASCII axis separators.* The map body
   was per-line centered (Q126); the rest of the viewport
   (planet card, legend, log) was left-aligned. Visual harmony
   wanted everything centered. Plus a `+------` corner + per-row
   `|` between the row index and grid cells gives the map a
   conventional ASCII-table feel.

**Decision:** Six coordinated changes.

1. **Friendly badge labels.** New `friendly_badge()` helper maps
   Q121's hyphenated names to single-word adjectives:

   | Q121 | Q128 |
   |---|---|
   | frozen-out | frozen |
   | near-freezing | cold |
   | thriving | habitable |
   | near-boiling | hot |
   | boiling-off | scorching |
   | vacuum | vacuum |

2. **Planet type before badge.** New `planet_type()` helper
   maps substrate to an evocative archetype noun:

   | Substrate | Planet type |
   |---|---|
   | aqueous | ocean world |
   | ammoniacal | ammonia world |
   | hydrocarbon | methane world |
   | silicate | lava world |

   Card line 1 becomes `{planet_type} · {badge_friendly}`
   (e.g. `ocean world · scorching`). Q127's
   `substrate_descriptor` is removed; the substrate stays
   implied by the type and surfaces explicitly as the biochem
   token on the species biology line.

3. **Dedicated species section.** New `species_card()` returns
   a 3-line block:
   - Cognition: `{topology} {tier} cognition` (e.g.
     `centralized medium cognition`).
   - Senses + manip: `sense: {modality} · manip: {manipulation}`
     — labeled so the reader doesn't need to know the order
     convention.
   - Biology: `{lifespan}y · {sociality} · {comm} · {biochem}`
     (Q126 + Q127's biology line, unchanged).

   The species name moves to the section's divider label
   (`---- Cyranites ----`), matching how the planet section
   uses the planet name (Q127). With a future `Vec<Species>`
   the loop can emit one section per species.

4. **Universal centering.** Two new helpers,
   `write_centered_line` and `write_centered_block`, center
   every body line within `VIEWPORT_WIDTH = 40`. Applied to:
   the planet card, the species card, the map body, the
   legend, and the log. The whole viewport now reads as a
   single centered column.

5. **Axis separators in the compact grid.** Two changes in
   `frame.rs::render_world_frame_inner`:
   - After the column header, write a horizontal axis divider
     (`{2 spaces}+{n dashes}`) that aligns column-wise with
     the grid cells.
   - Each grid row's prefix becomes `{r:>2}|` (was `{r:>2} `),
     so the `|` sits at the same column as the `+` corner.
     Cells stay at column 3 — same alignment as Q117's dense
     2-row header above.

   Standard (non-compact) mode is unchanged — the divider +
   `|` are compact-only since the post-run report still uses
   non-compact and shouldn't grow box-drawing ornament.

6. **Card body lines reduce to 3.** With species moved out, the
   planet card carries just identity / climate / orbital, and
   the species section carries cognition / senses-manip /
   biology. The total line count stays at 6 lines (3 + 3) plus
   the two dividers.

**Why this respects the project goals.**
- *Reader-first:* every label that a non-domain reader would
  have to translate (`boiling-off`, `cent-med`, `aqueous`,
  hyphenated badges) gets replaced with English. The whole
  viewport reads as descriptive prose.
- *Future-proof:* the species section structurally supports
  multiple species (Q47 is currently single but the design
  anticipates growth). The `render()` body can iterate
  `Vec<Species>` and emit one section per entry without
  changing the visual contract.
- *Visual harmony:* universal centering + axis separators give
  the viewport a single coherent style. Pre-Q128 mixed centered
  dividers, centered map, left-aligned card / legend / log;
  Q128 makes everything one centered column with consistent
  ASCII-table grid markers.
- *Determinism preserved:* every helper is a pure function
  (`planet_type`, `friendly_badge`, `write_centered_line`,
  axis separators are static); same seed → same output bytes.
- *Q121 internal model preserved:* `host_species_status`
  still returns the technical Q121 names; `friendly_badge` is
  a presentation-layer translation only. Anything that reads
  the underlying scalar / event payload gets unchanged data.

**Numerical care.**
- `friendly_badge` / `planet_type` return `&'static str` —
  no allocation. Unknown variants fall through to `"unknown"`
  / `"unknown world"` so the viewport stays robust.
- `write_centered_line` uses `chars().count()` for line width
  to handle multi-byte UTF-8 glyphs (▲ △ ▒ ░ ~ ≈ · |) that
  display 1 column.
- The horizontal axis divider's prefix length (`row_prefix_pad
  - 1`) uses `saturating_sub` so a future zero-prefix
  config doesn't underflow.

**Lives in:** `sim/report/src/viewport.rs`
(`planet_type`, `friendly_badge`, `write_centered_line`,
`write_centered_block` helpers; `species_card()`; `planet_card()`
trimmed to 3 lines; `render()` emits species section + centers
all body blocks). `sim/report/src/frame.rs`
(`render_world_frame_inner` adds the `+`/`|` axis separators
in compact mode).

**Follow-ups (not blocking).**
- *Multi-species rendering.* When Q47 grows to N species per
  planet, `render()` should iterate `&self.species_list` and
  emit one species section per entry. The Q128 structure is
  ready; just needs `species: Option<SpeciesDerived>` →
  `species_list: Vec<SpeciesDerived>`.
- *Bottom + right axis separators.* Q128 only adds top + left
  separators. A `+` at the bottom-left corner + dashes below
  the grid would make a fully closed box. Pure ornament;
  deferred.
- *Configurable label set.* A `--no-friendly-labels` flag
  would let researchers see raw Q121 / Q47 names. Same Q127
  follow-up.
