## Q99 — Charge advection along the wind velocity field
**Status:** Decided
**Context:** Q91 advected temperature along velocity. Q98 added
the Lorentz coupling so charged regions deflect velocity by
`q · v × B`. But pre-Q99 the *charge* itself didn't ride the
wind — `Electromagnetism` only diffused charge between
neighbours via molecular conductivity. Charged regions stayed
sealed in their cell of origin even when the atmosphere around
them was actively flowing. The M1b coupling triangle (charge ↔
velocity ↔ magnetic field) had only two of three sides.

**Decision:** Add a `charge_advect_k` coefficient to
`Electromagnetism` and a corresponding `advect_charge` step that
runs alongside `diffuse_charge` each `integrate` call. The
advection uses the same pair-flux upwind scheme `Wind` uses for
temperature and `Hydrology` uses for vapour:

```text
For pair (i, j) with i < j:
  v_along = midpoint(v[i], v[j]) · dir(i → j)
  upwind  = if v_along > 0 { charge[i] } else { charge[j] }
  flux    = charge_advect_k · dt · v_along · upwind
  charge[i] -= flux
  charge[j] += flux
```

Pair-flux structure preserves total charge bit-exactly
(`q99_charge_advection_conserves_total` test).

`charge_advect_k = 0.01` for the Earth-like default — same scale
as Q91's heat advection and Q92's vapour advection.

**Why this respects the project goals.**
- *Physics as close to real as possible:* the M1b coupling
  triangle now closes. Charge rides with the wind (Q99),
  charged regions deflect velocity (Q98), velocity transports
  heat and vapour (Q91/Q92). Three previously-isolated
  M1b/M1a fields form a single consistent dynamics. Drifting
  electric storms, wind-driven plasma, geographically-shifted
  electrostatic build-up all become emergent.
- *Least hardcoded values:* one new coefficient
  (`charge_advect_k`); same form, same calibration scale as
  the existing pair-flux advections in Q91 and Q92.
- *Determinism:* pair iteration in canonical order; no
  state-dependent branching beyond the `v_along > 0` upwind
  selection (exact Q32.32 comparison); pair-flux is
  bit-exactly mass-conserving.

**Numerical care.**
- The pair-flux flux `charge_advect_k · dt · v_along · upwind`
  is the largest hot-path product. With `charge_advect_k =
  0.01`, `dt ≤ 1`, `v_along ≤ 10`, `upwind ≤ 50` (the
  discharge threshold), the per-pair flux stays under 5 — far
  inside Q32.32's ±2.1×10⁹.
- The new step doesn't change discharge semantics; the
  discharge pass still runs after both diffusion and advection,
  preserving Q43's "discharge fires on post-transport state"
  ordering.

**Lives in:** `sim/physics/src/em.rs` (new `charge_advect_k`
field on `Electromagnetism`, new `advect_charge` step in
`integrate`, plus a hex-direction constant shared with
`wind.rs` / `hydrology.rs` / `lorentz.rs`),
`sim/core/src/lib.rs` (`build_laws` passes `charge_advect_k =
0.01`).

**Bonus:** the three multi-minute sim-core tests
(`collapse_fires_in_long_enough_run_with_no_discoveries`,
`knowledge_transmits_across_collapse_boundary`,
`successor_civs_found_after_collapse`) are now `#[ignore]`d so
routine `cargo test` finishes in ~30 s instead of ~20 min.
A new `long_run_loop_smoke_short` (50 sim-years) catches
panic / overflow / plumbing regressions in seconds; the slow
tests still run with
`cargo test --release -- --include-ignored` for shipping
checks. No coverage lost — only dev iteration time recovered.

**Follow-ups (not blocking).**
- *Charge-density bound on advection.* Currently the advection
  has no upper bound on charge per cell other than the
  discharge threshold (which dissipates rather than capping).
  If the wind drives charge accumulation faster than discharge
  can clear it, charge could grow large. Real plasma physics
  has Debye-screening limits; lands when local
  electrostatic-energy budgets matter for recognition templates.
- *Coupling charge advection to the Lorentz kick.* Currently
  Q98 reads pre-advection charge for the velocity kick.
  Reading post-advection charge would tighten the
  charge → velocity feedback. Order-of-operations question
  more than a physics one.
