## Q91 — Atmospheric advection: pressure-gradient winds
**Status:** Decided
**Context:** Q90 sources cell temperature from stellar irradiance,
giving an emergent equator-to-pole gradient. Q90 + heat conduction
alone, however, is the wrong transport story: molecular diffusion
(`HeatConduction::alpha ≈ 0.1`) moves heat ~100-1000× slower than
real atmospheric circulation, so even with radiation sourcing the
gradient correctly, the steady-state climatology is unrealistically
sharp (poles too cold, equator too hot, no Hadley-cell-style
redistribution). The `pressure`, `fluid_v_q`, `fluid_v_r` fields
have existed in `PhysicsState` since the M1a foundation but no
law actually wrote them — the whole atmospheric-dynamics layer was
a stub.

**Decision:** Add a `Wind` law that (1) derives per-cell pressure
from temperature, (2) accelerates a per-cell velocity field from
pressure gradient with a friction term, and (3) advects temperature
along that velocity using pair-flux upwind differencing.

```text
P[i]    = c · T[i]                     // ideal-gas proxy
∇P[i]   = Σ_nb (P[nb] - P[i]) · dir(i → nb)
v[i]   += -dt · ∇P[i] · wind_k         // Newtonian acceleration
v[i]   *= (1 - friction_per_tick · dt) // surface drag

// Pair (i, j), j > i, advection of T:
v_along = midpoint(v[i], v[j]) · dir(i → j)
T_up    = if v_along > 0 { T[i] } else { T[j] }   // upwind
flux    = advect_k · dt · v_along · T_up
T[i]   -= flux
T[j]   += flux
```

`dir(i → nb)` is the axial-coordinate offset to that neighbour
(see `NEIGHBOUR_DIRECTIONS`); it isn't a Euclidean unit vector
(axial coords aren't orthonormal) but it's the natural
"direction toward neighbour" basis for pair-flux on a hex grid.

Earth-like defaults: `c = 1` (only the gradient matters), `wind_k
= 0.001`, `friction_per_tick = 0.3` (~3-tick velocity memory),
`advect_k = 0.01` so a 50 K equator-to-pole gradient transports
heat ~50× faster than `HeatConduction`'s `alpha = 0.1` default.

**Why this respects the project goals.**
- *Physics as close to real as possible:* atmospheric circulation
  is the dominant horizontal heat-transport mechanism on every
  real planet with an atmosphere. Q91 lets that emerge from
  pressure gradients (which emerge from temperature gradients,
  which emerge from Q90's stellar sourcing) instead of being a
  hardcoded "make the equator-to-pole gradient less extreme"
  fudge factor.
- *Least hardcoded values:* the only constants are `wind_k`,
  `friction_per_tick`, `advect_k`, and `pressure_per_kelvin`.
  All four are dimensional coefficients with explicit physical
  meaning; none are per-planet or per-substrate. Per-atmosphere
  scaling (Hazy atmosphere → more friction × thicker air →
  different `wind_k`) is a deferred follow-up that wires through
  `Atmosphere`'s eventual numeric scale-height, not new hardcodes.
- *Determinism:* pair-flux preserves total temperature
  bit-exactly (same `delta` applied with opposite signs to both
  cells); pair iteration is canonical-order; pressure and
  velocity computed from previous-tick snapshots; only
  state-dependent branch is `v_along > 0` (an exact Q32.32
  comparison, deterministic).
- *Reusability:* the velocity field `Wind` writes is genuine
  state. Q92 reuses it to advect water vapour (closing the
  hydrologic cycle: evaporate over warm cells → wind transport →
  condense over cold cells). Q94 magnetic field, future ash
  dispersal, and recognition templates that key on "downwind
  warmth" all read the same `(v_q, v_r)`.

**Numerical care.**
- Friction is *required* — without `friction_per_tick > 0`,
  velocity grows unboundedly under any sustained pressure
  gradient and overflows Q32.32 within tens of ticks. The 0.3
  default damps each component to ~5% of its accumulated value
  after 10 ticks, leaving a steady-state where pressure
  acceleration balances drag.
- Upwind selection (`v_along > 0`) is asymmetric in heat
  conservation but symmetric in mass conservation under the
  pair-flux structure — total temperature is bit-exactly
  preserved (verified by `pair_flux_conserves_total_temperature`).
- `advect_k · dt · v_along · upwind_t` is the largest hot-path
  product. With `advect_k = 0.01`, `v_along ≤ ~10`, and
  `upwind_t ≤ ~400`, the per-pair flux stays well under 100,
  far inside Q32.32's ±2.1×10⁹ range.

**Lives in:** `sim/physics/src/wind.rs` (the law),
`sim/physics/src/lib.rs` (re-export),
`sim/physics/src/orchestration.rs` (wired into the macro-step
loop after radiation, before molecular conduction;
`integrate_civ_step` takes `wind: Option<&Wind>` so existing
deterministic micro-tests can pass `None`),
`sim/core/src/lib.rs` (`build_laws` returns a 6-tuple including
`Wind::earth_like()`).

**Follow-ups (not blocking).**
- *Per-atmosphere wind scaling.* Hazy atmospheres (Venus, Titan)
  have higher mass per column → more momentum carried per gradient
  → larger `advect_k`. Wires through once `Atmosphere` carries
  numeric scale-height + density rather than just a tag.
- *Coriolis deflection.* Real winds curve right (N) / left (S)
  due to planetary rotation. Lands when rotation period becomes
  a queryable per-planet field that orchestration can read.
- *Thermal vs mechanical wind separation.* Currently one wind
  field; real atmospheres have geostrophic + ageostrophic
  components. Worth splitting once vapour (Q92) couples back
  into pressure via humidity.
- *Vertical advection.* Currently 2D only. A height-stack would
  let warm air rise / cold air sink instead of just moving
  horizontally — the missing dimension of real Hadley circulation.
- *Q91-aware recognition templates.* "Downwind warmth", "windward
  rain shadow", "trade-wind belt" all become recognisable now
  that the velocity field is real. Authoring lands when civs
  start accumulating wind-related observations.
