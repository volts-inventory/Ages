## Q116 — Configurable grid size + compact viewport rendering
**Status:** Decided
**Context:** Grid size was hardcoded at 12×8 (96 cells) in
`RunConfig::dev`. The viewport rendered each cell as `symbol +
space` (~28 cols at default width), with a hex-row offset every
other row. That fits any phone, but a user wanting *more*
detail (24×16 = 384 cells, 4× resolution) hit two walls:

1. No CLI override for `grid_width` / `grid_height`. Bigger
   grids required code changes.
2. A 24-wide grid at 2-char-per-cell = 52 cols, which doesn't
   fit portrait phone terminals.

**Decision:** Two new opt-in features:

1. **`--grid-width <N>` and `--grid-height <N>`** — override the
   sim grid dimensions. Smaller grids run *faster* (each law
   iterates O(N_cells)); bigger grids give more spatial detail
   at proportional cost.
2. **`--viewport-compact`** — render each cell as a single
   character with no hex-row offset. Halves the on-screen grid
   width (default 12-wide grid: 28 cols → 16 cols; 24-wide:
   52 cols → 27 cols). Pure visual change; sim state and
   determinism unaffected.

**Concrete tradeoffs**:

| Grid | Cells | Smoke (10y) | Long test (24,000t) | Compact width |
|---|---|---|---|---|
| 8×6 | 48 | ~0.15 s | ~550 s (~9 min) | 11 cols |
| 12×8 | 96 (default) | ~0.28 s | ~1100 s (~18 min) | 16 cols |
| 16×12 | 192 | ~0.55 s | ~2200 s (~37 min) | 20 cols |
| 24×16 | 384 | ~1.1 s | ~4400 s (~73 min) | 27 cols |
| 32×20 | 640 | ~1.8 s | ~7300 s (~2 h) | 35 cols |

All still inside the project's "hours not days" runtime budget.
The 24×16 + `--viewport-compact` combo is the sweet spot for
"more detail, still fits phone": 4× the spatial resolution, ~4×
slower long runs, fits 27-col portrait terminals.

**Why this respects the project goals.**
- *More detail when wanted, fast default:* the user picks the
  tradeoff. Default 12×8 keeps the project's existing tuning;
  `--grid-width 24 --grid-height 16` opts into more detail.
- *Mobile-friendly density:* compact rendering lets phone users
  see twice as many cells horizontally without changing the
  sim. Square-grid look (no hex offset) is also easier to read
  on a small screen.
- *Runtime is honest:* documentation states the measured cost
  per grid size so users know what they're getting.
- *Determinism preserved:* same seed × same grid size → same
  events. Render variant doesn't affect the event stream.

**Lives in:** `ages/src/main.rs` (new `--grid-width`,
`--grid-height`, `--viewport-compact` CLI flags + plumbing),
`sim/report/src/frame.rs` (new `render_world_frame_compact` /
`render_world_frame_colored_compact` exports + `compact: bool`
parameter on the inner renderer; existing 2-char-per-cell mode
unchanged), `sim/report/src/viewport.rs` (`ViewportConfig.compact`
field; render-path picks the compact variant when set).

**Follow-ups (not blocking).**
- *Per-grid-size physics calibration.* Some constants (e.g.
  `lat_floor` in radiation, friction in wind) are tuned for
  ~10-row grids. On a 32×20 grid the climate-band resolution
  is finer; per-row constants might want re-tuning.
- *Default grid-size CLI argument.* If the project standardises
  on a non-12×8 default later, `RunConfig::dev` would change.
  Currently the CLI defaults to whatever `RunConfig::dev` sets.
