## Q45 — Pattern recognition mechanism
**Status:** Decided
**Context:** Continuous physics state needs to be chunked into discrete
"recognized phenomena" before the discovery pipeline can hypothesise
about relations. How the recognition layer detects signatures (tides,
lightning, fire, fermentation, etc.) from raw state determines which
phenomena fire when.
**Decision:** Template-driven signature matching. Each template
specifies a `Signature` over physics fields (`Above`, `Below`,
`AbsAbove`, `All`-of-list); the recognition layer scans every cell
each tick and emits one firing per matched (template, cell) pair.
Iteration is template-major then cell-major sorted, deterministic
per Q31.

M2 ships per-cell signatures only — threshold crossings and
co-occurrence (`All`). Spatial-temporal clustering for multi-cell
or multi-tick phenomena (tides as periodic oscillation, weather
systems as cell clusters) lands as recognition matures (M2
follow-up or M3 if the discovery pipeline demands it).

Initial M2 template set: fire (hot + fuel + oxidiser), lightning
buildup (|charge| above threshold), ice present, vapour present,
surface water. Catalog grows as authored.
**Lives in:** `sim/recognition/src/lib.rs`,
`docs/world.md` Recognized phenomena section.
