## Q44 — Seeded law variation
**Status:** Decided (revisit calibration during M2/M3)
**Context:** Same laws across all runs (Earth-civ and Run #847 derive
the same gravitational inverse-square) vs sampling law parameters per
run (Run #847 has 1/r^1.97 gravity, all consequences cascade). The
latter produces alternate-physics worlds. Audit distinguished
*structural* constants (gravity exponent, conservation forms — wide
variation can break self-consistency) from *parametric* constants
(rate coefficients, couplings, viscosities, chemistry coefficients —
wide variation is safe and self-consistent).
**Decision:** C2-phased — wide seeded variation in **parametric**
constants only; structural constants stay fixed. Phased rollout:
- **M1**: parametric constants wired in via config. Defaults are
  Earth-physics-equivalent. No sampling. Behaves identically to a
  fixed-laws build but the architecture is config-driven.
- **M2**: seeded-sampling layer lands alongside the recognition
  layer. Sampling defaults off (`sample_laws: false`); opt-in per
  run via config. Initial sample-space ranges are narrow.
- **Post-M3**: once fixed-default runs are validated as stable and
  discovery works, flip sampling default to on (or per-planet-type).
  Calibrate ranges against accumulated run data. Widen ranges as
  confidence grows.

C2 → D is trivial (just don't sample / always use defaults), so this
isn't a one-way door. Sample-range calibration is the ongoing tuning
work.

**Note to revisit:** This decision should be re-examined during
M2/M3. Specifically — verify (a) that fixed-default runs are stable
and produce coherent civs, (b) that initial narrow sampling produces
recognisably different but still coherent runs, (c) that calibration
ranges aren't producing degenerate worlds. If any of those fail,
back off to fixed defaults (D-behaviour) and revisit the strategy.
