## Q119 — Planet name + survivability badge + "no mag"
**Status:** Decided
**Context:** Three small viewport polish items requested
together:

1. **Planet has no name.** The card shows `aqueous · reducing
   · 4 moons` with no human-readable identifier. Long sessions
   and screenshots become hard to refer back to ("the planet
   from seed 42" is awkward).
2. **`none mag` reads awkwardly.** "None mag" is grammatically
   off — should be "no mag" or "no magnetosphere".
3. **No survivability indicator.** A reader can't tell at a
   glance whether the planet would kill an unprotected human.
   The temperature, atmosphere, pressure, and substrate all
   matter and are spread across two card lines.

**Decision:** Three additions, one Q-decision.

1. **`Planet::name: String`** generated deterministically from
   the seed via `planet_name_from_seed(seed)`. 64 evocative
   stem names (`Vela`, `Lyra`, `Cygnus`, `Aleph`, …) × 16
   suffix letters (`a`–`p`) gives ~1024 distinct names before
   collisions. Same seed → same name. Threaded through to
   `PlanetDerived` so all downstream consumers (viewport,
   report, narrator) can use it.
2. **`none` → `no`** for the magnetosphere display.
   `magnetosphere == "none"` renders as `no mag` instead of
   `none mag`. Pure formatting.
3. **`survivability_badge` helper** producing one of:
   - `survivable` — aqueous + oxidising + temp 250-330 K +
     pressure 50-200 kPa
   - `alien-chemistry` — non-aqueous substrate
   - `vacuum` — `Atmosphere::None`
   - `toxic-atm` — non-oxidising atmosphere (reducing, hazy,
     thin)
   - `too-hot` / `too-cold` — temp out of range
   - `low-pressure` / `crushing-pressure` — pressure out of
     range
   The badge appears on the new top line of the planet card
   alongside the name: `Vela-c · survivable`.

**Final viewport layout** (3 planet card lines + caption,
default `--temperature-unit f`):
```
Glade-a · toxic-atm
aqueous · reducing · 4 moons
no mag · 170F · 32h · 45° · 13mo
Y4 M11 · 1 civ · 1F/0C
```

vs a survivable seed:
```
Caelum-a · survivable
aqueous · oxidising · 2 moons
no mag · 55F · 200h · 27° · 14mo
```

**Why this respects the project goals.**
- *Reader-friendly identification:* names give a memorable
  handle for screenshots and conversations. "Caelum-a from
  seed 12" beats "the planet from seed 12 with the slow day"
  for cross-session continuity.
- *Honest assessment:* the survivability badge tells the truth
  about each world's habitability for *humans* without
  pretending the sim is more accurate than it is. Q82 already
  guarantees every planet is habitable for *some* chemistry;
  the badge just answers "could *we* live there".
- *Determinism preserved:* name generation is pure-function
  on the seed. Survivability is pure-function on the planet
  fields. No state mutation, no per-tick branching.
- *Backward compatible:* existing tests pass. The synthetic
  planet fixtures get a `name: "TestPlanet".to_string()` field
  — one line each.

**Numerical care.**
- `planet_name_from_seed` uses `usize::try_from(seed % N)` to
  index into the static name arrays — bounded by `STEMS.len()
  = 64` and `SUFFIX.len() = 16`, both well within `usize` on
  any target.
- `survivability_badge` is a pure function on f64 values
  derived from Q32.32 fields. The thresholds (250 K, 330 K,
  50 kPa, 200 kPa) are conservative human-survival ranges.

**Lives in:** `sim/world/src/lib.rs` (new
`planet_name_from_seed` + `Planet::name` field + sample-
planet wiring + synthetic-fixture update),
`protocol/src/lib.rs` (`PlanetDerived::name` field),
`sim/core/src/lib.rs` (`planet_to_event` copies `name`),
`sim/civ/src/catastrophe.rs` (test-fixture `name` field),
`sim/report/src/viewport.rs` (planet card now 3 lines, top
line is `name · badge`; `survivability_badge` helper at
module scope; `none mag` → `no mag`).

**Follow-ups (not blocking).**
- *Per-substrate name pools.* All seeds currently draw from
  the same name pool. Substrate-themed names (silicate gets
  crystal-flavor names, hydrocarbon gets oil-flavor) would
  add narrative texture.
- *Use the planet name in the post-run report.* The Markdown
  render still says "the planet (seed 42)". Switching to
  `Caelum-a (seed 12)` would be a small report-side change.
- *More-nuanced survivability tiers.* Currently binary
  (survivable / something-else). A `marginal` tier between
  could capture borderline-but-not-immediately-fatal worlds.
