## Q96 — Per-cell pressure-aware boil in `Hydrology`
**Status:** Decided
**Context:** Q92 shipped `Hydrology` with a planet-wide constant
`boil_threshold_k`. Real planets have *altitude-dependent*
boiling — water boils at ~70 °C atop Mt. Everest and at ~100 °C
at sea level. With a single threshold, mountain cells couldn't
evaporate at temperatures that would drive a perfectly normal
upland mist on Earth, and tropopause-altitude clouds had no
basis for forming. The Q92 follow-up list called this out
explicitly: "Per-cell pressure-aware boil. Q91 already updates
the per-cell `pressure` field each tick; Hydrology could feed
each cell's pressure through `substrate_boiling_point_k` for
per-cell altitude-aware boil. Cheap follow-up."

The implementation isn't quite that direct. Q91's `state.pressure`
is a normalised value (`P = c · T` with `c = 1`), not in Pa, so
it can't feed the Clausius-Clapeyron formula directly. And Q91's
pressure is *temperature-driven* (warm cells low-P), not
altitude-driven, so even if it were in Pa it wouldn't represent
the relevant atmospheric column.

**Decision:** Compute the per-cell pressure inside `Hydrology`
from `state.elevation()` and the planet's `surface_pressure_pa`
using the standard barometric formula:

```text
  P(h) = P_0 · exp(-h / H)
```

with `H = 8400 m` (Earth's nominal atmospheric scale height; a
reasonable approximation for all `Atmosphere` classes in our
taxonomy until per-atmosphere scale heights become a separate
field). Then feed `P(h)` to `substrate_boiling_point_k(tag, p)`
per cell to get the elevation-aware boil threshold.

Concretely:

```rust
fn cell_pressure_pa(&self, state, cell) -> Real {
    let elev = state.elevation()[cell];
    self.surface_pressure_pa * exp(-elev / SCALE_HEIGHT_M)
}

fn cell_boil_threshold_k(&self, state, cell) -> Real {
    let p = self.cell_pressure_pa(state, cell);
    substrate_boiling_point_k(&self.substrate_tag, p)
}
```

`integrate` computes the per-cell threshold table once per
macro-step (elevation is static post-init, surface pressure is
constant) and reuses it for both the evaporation and
precipitation passes.

`Hydrology::for_substrate` now takes `(substrate_tag,
surface_pressure_pa)` instead of a precomputed
`boil_threshold_k`. `core::build_laws` passes
`planet.metabolic_substrate.tag()` and `planet.surface_pressure`
(both already on `Planet` since worldgen).

**Why this respects the project goals.**
- *Physics as close to real as possible:* a 5 km mountain cell
  on Earth boils at ~357 K instead of 373 K — exactly the
  Clausius-Clapeyron prediction. Mountain cells now evaporate
  at temperatures that would barely warm a coastal cell. With
  Q91 winds and Q92 advection in place, this gives orographic
  evaporation patterns: warm slope cells dry out, cool peaks
  precipitate the rising vapour. Sub-sea-level basins get
  *higher* boil points (slightly more than 1 atm), matching
  Death-Valley physics.
- *Least hardcoded values:* one constant
  (`SCALE_HEIGHT_M = 8400`) plus the existing substrate-aware
  Clausius-Clapeyron parameters from Q85. No per-cell tables;
  per-cell behaviour falls out of `state.elevation()` and the
  planet's `surface_pressure_pa`, both of which are already
  `Planet` state.
- *Determinism:* per-cell threshold derived from static state
  (`elevation`) and constant config (`surface_pressure_pa`,
  `substrate_tag`); no state-dependent branching beyond the
  existing `T > boil ? evap : condense` pair.

**Numerical care.**
- `exp(-elev / 8400)` for `elev ∈ [0, 15000]` stays in
  `[exp(-1.79), 1] ≈ [0.17, 1]` — well-behaved in Q32.32.
- `surface_pressure_pa = 101_325` (Earth) for an aqueous
  planet; thinner-atmosphere worlds could surface as low as
  `Atmosphere::None` → 0 Pa, in which case
  `substrate_boiling_point_k` returns the sub-freeze sentinel
  (`freeze_point - 50`) and every cell evaporates. That's
  the correct vacuum behaviour: any liquid is gone instantly.
- `substrate_boiling_point_k` uses `ln` and Q32.32 reciprocal —
  both deterministic, both finite for all in-range inputs.

**Lives in:** `sim/physics/src/hydrology.rs` (the
`cell_pressure_pa` / `cell_boil_threshold_k` helpers + per-cell
threshold table + `for_substrate` signature change),
`sim/core/src/lib.rs` (`build_laws` calls
`Hydrology::for_substrate(substrate_tag, surface_pressure)`).

**Follow-ups (not blocking).**
- *Per-atmosphere scale height.* `H = 8400` is Earth's value.
  Hazy / Reducing / Thin atmospheres should each get their own
  scale height, lifting alongside Q91's per-atmosphere wind
  scaling. Both await `Atmosphere` growing numeric
  scale-height + density fields.
- *Saturation curve below boil.* Real evaporation occurs below
  the boiling point at a rate set by saturation vapour
  pressure (a Clausius-Clapeyron exponential in `T`, not just
  a step at `T_boil`). Lands when per-cell humidity becomes a
  tracked field.
- *Latitude-correlated atmospheric thickness.* Real Earth's
  atmosphere is slightly thicker at the equator (centrifugal
  bulge). Tiny effect; deferred indefinitely.
